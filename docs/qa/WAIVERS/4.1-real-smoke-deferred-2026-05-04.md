## Risco aceito

**O que pode dar errado em produção / dataset por aceitar este WAIVER:**

1. **Caminho real CLI multi-symbol → broker → 4 workers DLL real → catálogo único**
   não foi exercitado E2E. Mocks unit + integration cobrem o caminho funcional
   usando MockProfitDLL + factories injetadas, mas a integração com a DLL
   Nelogica real pode revelar:
   - Comportamento de licença ao iniciar 2+ instâncias na mesma máquina
     (Q17-OPEN — pode falhar com NL_LICENSE_BUSY ou similar).
   - Q06-V (callback nesta thread): cada worker tem sua própria
     ConnectorThread; sem cross-process race esperado.
   - Q08-E (init não-idempotente): cada worker init UMA vez (R3 expandida);
     pool persistente reusa workers entre jobs sem re-init.
   - Performance real do broker vs IPC overhead em produção (banda + lat
     SQLite) pode diferir do bench mock.

2. **Speedup target ≥ 3.2x** (Pyro D4) é projetado em mock baseline (pool
   persistente elimina spawn cost). Real pode ser menor por contention de
   rede entre workers (4 conexões DLL simultâneas dividem banda) — Pyro
   reavalia em 4.1-followup com bench real.

3. **AC8 §"Worker crash simulado"** (broker detecta crash + reinicia worker)
   NÃO foi implementado — adiado para 4.1-followup com smoke real (mock
   crash adicionaria complexidade sem valor real).

**Mitigação:** O smoke real multi-symbol é **bloqueante de release V1**.
Nenhum binário V1 será publicado por @devops sem este smoke PASS arquivado em
`docs/qa/SMOKE_EVIDENCE/`. Risco residual = zero para release; risco
operacional = um eventual descoberta-tarde de quirk DLL multi-process antes de V1
(probabilidade baixa porque single-symbol smoke já valida cada worker
isoladamente; multi-process só adiciona contention).
