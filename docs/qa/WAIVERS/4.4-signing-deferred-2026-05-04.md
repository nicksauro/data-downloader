# WAIVER — `4.4` — `2026-05-04` (Code signing deferred V1.1)

## Justificativa

A AC5 da Story 4.4 prevê **Caminho A vs Caminho B** para code signing
(ADR-016):

- **Caminho A (cert disponível):** integrar signtool no pipeline release
  (Azure Trusted Signing OU EV cert local + HSM token) — instalador
  ganha selo "Verified Publisher" no SmartScreen.
- **Caminho B (cert não disponível V1):** documentar workaround
  SmartScreen + plano de upgrade V1.1 com signing.

V1.0 segue **Caminho B** porque:

1. **EV cert ainda não adquirido** — Sectigo / DigiCert EV Code Signing
   custa ~$300-700/ano + 5-10 dias úteis de validação organizacional.
   A timeline da Story 4.4 (2 dias) não comporta aquisição.
2. **HSM token físico** (mandate Microsoft pós-2023 para EV) precisa
   ser shipped + setup por humano. Fora do escopo agente.
3. **Audiência V1.0 = squad + early adopters técnicos** — mesma
   política do MVP (Epic 1-3) já aceita em ADR-016 §"Plano".
   Estende-se para V1.0 sem mudança de risk profile (squad de uso
   interno).
4. **DigiCert KeyLocker (cloud HSM)** seria alternativa CI-friendly,
   mas exige cert EV adquirido + setup vendor + integration release.yml
   — ainda fora do escopo desta story.

Caminho B é **aceitável V1.0** conforme ADR-016 §"Decisão" — a decisão
arquitetural já prevê deferred até EV cert ser adquirido.

## Mitigação V1.0 (Caminho B implementado)

`docs/release/INSTALL.md` §5 ("SmartScreen workaround") documenta
explicitamente:

1. O que o usuário vai ver (texto exato do SmartScreen warning).
2. Por que aparece (sem assinatura digital + audiência inicial).
3. Como executar mesmo assim (Mais informações → Executar mesmo assim).
4. Comportamento esperado em releases subsequentes (cada update reseta
   reputação até cert EV ser adquirido).

`build/WINDOWS_DEFENDER_NOTES.md` (Felix prep, já existente) documenta:

- Recomendações de exclusão Defender (pasta de instalação + pasta de dados)
  para mitigar performance scan overhead (R-P3 / Pyro).
- Política false-positive reporting.
- Por que UPX é proibido (confunde com malware + invalida signing futuro).

`README.md` aponta para INSTALL.md como entry point oficial.

## Cobertura compensatória

V1.0 publica **SHA256 verificável** em 3 lugares como substituto parcial
de signing:

1. `dist/build-manifest-v{version}.json` — sha256 de cada arquivo do
   onedir + sha256 do zip + git_sha + builder_hostname_sanitized.
2. GitHub Release notes body (gerado por `scripts/github_release.py`) —
   inclui SHA256 do zip + comando PowerShell para usuário verificar.
3. README.md — append "Latest release: vX.Y.Z" link após cada publicação
   bem-sucedida (via `scripts/github_release.py`).

Tampering detection é **manual** (usuário compara hashes) vs **automatic
via signature** (Caminho A). Aceitável V1.0 para audiência técnica
(usuário sabe rodar `Get-FileHash`).

## Risco aceito

**O que pode dar errado em produção / dataset por aceitar este WAIVER:**

1. **SmartScreen warning persiste em todas as releases V1.x** até cert
   EV ser adquirido + integrado. UX ruim para usuários novos (alguns
   recusam executar app não-assinado por princípio corporativo).
   Mitigação: INSTALL.md guia explícito; cobertura compensatória SHA256.

2. **AV false positives** podem aumentar — ADR-016 §"AV false
   positives" cita "alto" para `none` vs "mínimo" para `EV`.
   Mitigação: WINDOWS_DEFENDER_NOTES.md §1 (exclusões); seção §4
   (reportar falsos positivos) em INSTALL.md / WINDOWS_DEFENDER_NOTES.md.

3. **Tampering detection manual** — usuário precisa rodar
   `Get-FileHash` proativamente. Mitigação: SHA256 publicado em 3
   lugares + INSTALL.md §3.2 documenta o processo.

4. **Re-distribuição não-autorizada** — sem signing, ator malicioso
   pode redistribuir binário modificado com mesmo nome. Mitigação:
   GitHub Release oficial é única fonte verdade; sha256 publicado
   permite verificação cruzada.

**Mitigação principal:** V1.1 fecha definitivamente — `docs/stories/4.4-followup.story.md`
tem AC dedicado a code signing Caminho A com EV cert. Probabilidade de
aprovação humana orçamentária ($400/ano): alta (custo modesto +
profissionalismo industria).
