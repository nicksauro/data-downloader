## Risco aceito

**O que pode dar errado em produção / dataset por aceitar este WAIVER:**

1. **Caminho real CLI → DLL → write Parquet → catalog → DuckDB** não foi
   exercitado E2E. Mocks integration (`tests/integration/test_cli_download.py`
   11 testes) e unit (`tests/unit/test_public_api_download.py` 15 testes)
   cobrem o caminho funcional usando factories injetadas (`dll_factory`,
   `catalog_factory`, `writer_factory`), mas a integração com a DLL Nelogica
   real pode revelar:
   - Quirks de inicialização da DLL não cobertos pelo mock (ex: timing de
     `wait_market_connected`, callbacks fora de ordem).
   - Quirks de codificação de string (Q06-S — UTF-16 vs WideChar).
   - Comportamento real do quirk Q11-99 (99% reconnect) em download longo.
   - Interação real com chunking 5d (WDO) sob carga real.

2. **Idempotência cross-run** (re-run = no-op) é validada por property test
   (`test_orchestrator_idempotent.py`) sob mocks, mas não contra estado
   persistido por DLL real.

3. **Range esperado** (PASS-2 do SMOKE_PROTOCOL.md: 100k-50M trades em 30 dias)
   não foi calibrado contra dados reais.

**Mitigação:** O smoke real é **bloqueante de release V1**. Nenhum binário V1
será publicado por @devops sem este smoke PASS arquivado em
`docs/qa/SMOKE_EVIDENCE/`. Risco residual = zero para release; risco operacional
= um eventual descoberta-tarde de quirk DLL antes de V1 (mas com Stories 1.7a
audit Aria APPROVED + Story 1.3 audit Nelo APPROVED, a probabilidade é baixa).
