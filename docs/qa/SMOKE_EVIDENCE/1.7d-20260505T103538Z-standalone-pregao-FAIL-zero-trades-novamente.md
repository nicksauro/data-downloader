## Hipóteses sucessoras (a investigar — Q-DRIFT-27/28/29/30)

| ID | Hipótese | Como discriminar |
|----|----------|------------------|
| Q-DRIFT-27 | Exchange `'F'` errado para WDOJ26 | tentar `'BMF'`, `'B3'`; comparar com `profitdll/Exemplo Python/main.py` |
| Q-DRIFT-28 | Race ou ordem incorreta `set_history_trade_callback_v2` antes/depois `get_history_trades` | revisar `download_chunk` L640-642 — verificar se DLL exige register PRÉ-subscribe |
| Q-DRIFT-29 | `TConnectorTrade` struct mismatch — `translate_trade` retorna None silenciosamente | counter `translate_failures=0` indica que callback **NUNCA foi chamado**, não que falhou no translate; refuta para esta sub-run específica mas pode aplicar em outros cenários |
| Q-DRIFT-30 | `make_history_trade_callback_v2` factory + lifetime CFUNCTYPE — ref garbage-collected entre register e fire | inspecionar `data_downloader.dll.callbacks._cb_refs` ao final da run; comparar com main.py exemplo Nelogica |

**Indício forte para Q-DRIFT-27 ou Q-DRIFT-30:**
`translate_failures=0` + `trade_edits=0` + `progress_history_len=0`
→ DLL aceita `subscribe_ticker(code=0)` + aceita `get_history_trades(code=0)`
mas **nada** retorna pelo callback. Isso é consistente com:
1. callback registrado para slot ERRADO ou
2. callback ref morta (GC'd antes do fire) ou
3. exchange code semanticamente inválido (server retorna sucesso mas
   silenciosamente descarta query).
