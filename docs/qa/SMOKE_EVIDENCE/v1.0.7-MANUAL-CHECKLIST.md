## Setup

- [ ] Rodar `dist/data_downloader/data_downloader.exe` (double-click). NÃO abrir CLI.
- [ ] App abre em ~2-3s, mostra DownloadScreen com botão "BAIXAR HISTÓRICO".
- [ ] Status bar mostra "DLL desconectada" inicialmente (esperado).

## Test 1 — Progress bar atualiza durante download

- [ ] Preencher symbol = `WDOJ26`, exchange = `F`, período = últimos 5 dias úteis.
- [ ] Clicar "BAIXAR HISTÓRICO".
- [ ] **Em ~5s:** transição para tela "loading" (form some, ProgressCard aparece).
- [ ] **Verificar logs visíveis no painel** (NOVO v1.0.7) — ProgressCard mostra área expansível "▾ Detalhes" JÁ ABERTA com linhas tipo `[HH:MM:SS] Iniciando download...`, `[HH:MM:SS] Inicializando ProfitDLL...`, `[HH:MM:SS] DLL pronta...`.
- [ ] **Após ~30-90s** (DLL handshake + 1º chunk): barra azul move de `0%` → `~20%` (5 chunks de 1d cada → +20% por chunk).
- [ ] Subtitle exibe `"1/5 chunks (NNN trades) — 20.0%"` após 1º chunk.
- [ ] Barra continua progredindo até `100%` (subtitle: `"5/5 chunks (XXX trades) — 100.0%"`).
- [ ] **Toast verde** aparece com `"Pronto! WDOJ26 (XXX trades em 5 arquivos) baixado"` após completar.
- [ ] Volta para DownloadScreen normal (form visível de novo).

## Test 2 — Cancelar download mid-progress

- [ ] Iniciar novo download (mesmo símbolo, período maior — ~1 mês = 5 chunks).
- [ ] Após 1-2 chunks (barra ~20-40%), clicar "CANCELAR".
- [ ] Modal de confirmação aparece com texto literal Uma.
- [ ] Confirmar cancelamento → estado vira "Cancelando..." (botão amarelo, disabled).
- [ ] Após drain (~5-30s) → toast `"Download cancelado"` + volta para tela normal.
- [ ] Logs no painel mostram `[HH:MM:SS] WARN ui.cancel_clicked` ou similar.

## Test 3 — Verificar arquivos parquet salvos (paridade com CLI)

- [ ] Após Test 1 completar com sucesso, abrir `Catálogo` (Ctrl+B na sidebar).
- [ ] Catalog mostra 1 linha para `WDOJ26 / F / [data range]` com `trades > 500.000` (paridade com CLI smoke v1.0.6 = 524.191).
- [ ] Verificar arquivos parquet em `~/data/history/WDOJ26/F/` (ou `data_dir` configurado).

## Test 4 — Erro flow (DLL credentials missing)

- [ ] Abrir Settings (Ctrl+,), apagar valor de "Chave de ativação", salvar.
- [ ] Voltar para Download, clicar "BAIXAR".
- [ ] Após ~3s → estado "error" com card vermelho `"Sem licença"`.
- [ ] Logs no painel mostram `[HH:MM:SS] ERROR NL_NO_LICENSE` ou similar.

## Test 5 — Logs do download.start visíveis (Bug 2 — regressão crítica)

- [ ] Iniciar download (Test 1 pode ser reaproveitado).
- [ ] **DENTRO DOS PRIMEIROS 5 SEGUNDOS** após clicar "BAIXAR", verificar que o painel de logs do ProgressCard mostra:
  - [ ] `[HH:MM:SS] Iniciando download...` (de `INF_STARTING_DOWNLOAD`)
  - [ ] `[HH:MM:SS] Inicializando ProfitDLL...` (de `INF_STARTING_DLL`)
- [ ] Se NENHUM log aparecer = **regressão Bug 2** (QtLogHandler quebrado).

## Test 6 — Navegação entre telas durante download

- [ ] Iniciar download.
- [ ] Clicar Catálogo (Ctrl+B) — DownloadScreen continua rodando em background.
- [ ] Voltar para Download (Ctrl+D) — barra continua de onde estava.
- [ ] Estado preservado.

## Test 7 — Quit during download

- [ ] Com download em progresso, fechar app (Ctrl+Q ou X).
- [ ] Modal "Sair durante download?" aparece.
- [ ] Confirmar "Sair e cancelar" → app fecha.
- [ ] Reabrir app → Catalog mostra `cancelled` para o job.

## Critérios de PASS

- **Bug 1 fix confirmado** se Test 1 mostra barra progredindo de 0% → 100%.
- **Bug 2 fix confirmado** se Test 1 + Test 5 mostram logs no painel do ProgressCard dentro dos primeiros 5s.
- **Sem regressão funcional**: Tests 2-7 passam.
- Final: trades count na UI bate com CLI smoke v1.0.6 (~524.191 para WDOJ26 1 mês).

## Critérios de FAIL — abrir issue + assignar Felix

- Barra fica em 0% mesmo após 90s+ → @Slot(object) não está fazendo efeito; investigar PySide6 frozen build.
- Logs vazios mesmo após 5s → QtLogHandler não foi instalado; verificar `app.py::main`.
- Toast de sucesso não aparece → `_on_finished` slot não disparou (cross-thread issue persiste).
