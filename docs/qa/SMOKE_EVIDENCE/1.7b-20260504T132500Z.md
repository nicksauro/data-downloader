## Smoke 3 — `tests/smoke/test_mvp_gate.py` (30 dias WDOJ26 — gate Epic 1)

### Tentativa A — pytest

- Status: `FAIL` (rc=1, elapsed 2.33s — falhou cedo)
- Subprocess CLI dentro do pytest crasha com `UnicodeDecodeError: 'charmap'
  codec can't decode byte 0x90` na thread `_readerthread` que captura stdout.
- **Hotfix B4 (utf-8 reconfigure no topo de cli.py) não cobre este caso**: o
  pytest spawning subprocess via `subprocess.run(text=True, capture_output=True)`
  default em Windows usa cp1252 para **decodificar bytes recebidos**, não para
  o encode interno do CLI. CLI emite UTF-8 corretamente, mas pytest reader
  thread crasha decoding. **Bug B4 hotfix incompleto** para o caminho
  test_mvp_gate.py — precisa setar `encoding='utf-8'` no
  `subprocess.run` do test.
- CLI por trás retornou rc=3 (mesma causa que tentativa B), mas stdout/stderr
  ficaram parcialmente perdidos por crash do reader.
