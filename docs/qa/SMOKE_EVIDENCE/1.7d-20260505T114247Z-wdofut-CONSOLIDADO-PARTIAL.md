Esta sessão validou a diretiva via 3 execuções e descobriu que o bug
"zero trades" tem **duas camadas**:

1. **Camada externa (DLL/permissão/contrato/janela)** — RESOLVIDA pela diretiva
   do usuário. Probe `ctypes` puro com WDOFUT/F + janela 4 dias recebeu

   (per `probe_history_minimal.py`): "Bug é no NOSSO wrapper/orchestrator".

2. **Camada interna (orchestrator/wrapper)** — NÃO RESOLVIDA. Standalone
   `run_smoke_real_standalone.py` com WDOFUT recebeu trades do servidor
   (callback V2 disparou) mas **morreu na primeira tradução** —
   `IngestorThread` quebrou silenciosamente em duas modalidades distintas:
   - `minimal_handshake=True` → `OverflowError: int too long to convert`
     em `TranslateTrade` (Q-DRIFT-12 incompleto — `_configure_dll_signatures`
     skipped removeu `argtypes` de `TranslateTrade` também, fazendo
     ctypes truncar `c_size_t` handle em `c_int`).
   - `minimal_handshake=False` → `ValueError: timestamp_ns must be >= 0;
     got -2209161600000000000` em `format_brt_timestamp`. Struct
     `TConnectorTrade` aparentemente NÃO está sendo populado pela DLL
     em alguns casos (talvez sentinel inicial), mas `_process_trade` não
     tem try/except → `IngestorThread` morre, callback V2 segue
     enfileirando handles, ninguém drena, `trades_count=0`.

- **Causa real (extraída do traceback):**

  ```text
  File "...wrapper.py", line 1683, in _translate_trade_raw
      rc: int = self._dll.TranslateTrade(p_trade_handle, byref(trade_struct))
  ctypes.ArgumentError: argument 1: OverflowError: int too long to convert
  ```

- **Bug:** `minimal_handshake=True` skip `_configure_dll_signatures`
  (pulado registrar `TranslateTrade.argtypes = [c_size_t, POINTER(TConnectorTrade)]`).
  Sem argtypes, ctypes default para `c_int` → handle V2 (size_t = 64 bits)
  overflow no marshalling.

- **Log:** `docs/qa/SMOKE_EVIDENCE/logs/standalone-wdofut-fullconf-20260505T113850Z.log`
- **Verdict:** processo morreu antes de SUMMARY (log truncado em 56 linhas).
- **Causa real (traceback no log):**

  ```text
  File "...orchestrator/download_primitive.py", line 344, in _process_trade
      timestamp_str = format_brt_timestamp(timestamp_ns)
  File "...orchestrator/timestamp.py", line 142, in format_brt_timestamp
      raise ValueError(f"timestamp_ns must be >= 0; got {ns}")
  ValueError: timestamp_ns must be >= 0; got -2209161600000000000
  ```

- **Bug:** `_translate_trade_raw` retorna `rc=0` (DLL contente), mas
  `TConnectorTrade.TradeDate` permanece zerado (struct sentinela ou
  primeira invocação antes do dado real). `_system_time_to_ns_local`
  converte SystemTime zero → ano 1601 → ns negativo. `_process_trade`
  não envolve `format_brt_timestamp` em try/except → `IngestorThread`
  morre. Callback V2 segue empilhando handles, nada drena.
