## Análise técnica (Dex)

### Refutação clara de 3 famílias de hipóteses

**Q-DRIFT-19 — pytest fd-capture / DLL escreve em stderr → pipe satura → ConnectorThread bloqueia em write:**
**REFUTADA** empiricamente. Argumentos:

1. `-s` desativa fd-capture default; FD 1/2 mantêm-se nos handles do console host. Mesmo assim, trava.
2. `PYTHONUNBUFFERED=1` equipara buffering. Mesmo assim, trava.
3. **Argumento decisivo:** se ConnectorThread C estivesse bloqueada em `write()` por pipe saturado, o callback de state PARARIA imediatamente após o pipe encher. **Mas o log mostra o callback `MARKET_DATA/1 result=1` disparando a cada 2s, regularmente, durante 150s.** Logo a thread C está VIVA e ATIVA — não há bloqueio em I/O. Ela só não promove o estado. Q-DRIFT-19 fica refutada para o cenário pytest atual.

**Q-DRIFT-20 — pytest-cov instala `sys.settrace` global → ConnectorThread herda tracer → tracer overhead enfileira eventos:**
**REFUTADA**. `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` impede pytest-cov de carregar (e a tentativa anterior com `--timeout=120` de fato falhou pelo plugin nem estar no PATH). Mesmo sem cov, trava.

**Q-DRIFT-18 — pytest-qt autoload chama CoInitializeEx(MTA) no MainThread → DLL espera STA:**
**REFUTADA** pelo mesmo motivo: `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` impede o autoload. Mesmo assim, trava.

### Sintoma de fato observado

O loop é estável: `(2,1)` a cada 2s, com `dll.waiting_market_data` heartbeats em 30/60/90/120s. **A ConnectorThread C está ativa, dispara callback de state regularmente, mas o servidor Nelogica nunca emite o evento de promoção `(2,2)` MARKET_WAITING ou `(2,4)` MARKET_CONNECTED.** Isto significa que a ConnectorThread está enviando o handshake e recebendo "MARKET_DATA conexão tentando" (result=1) repetidamente — sem nunca convergir.

### Implicação grave

Refutadas Q-DRIFT-19/20/18 isoladamente, **e** com plugins desligados / capture desligado / buffering equiparado, o ambiente Python do pytest **ainda** difere o suficiente do script standalone para o servidor Nelogica não promover MARKET_DATA. O delta deve ser **algo que o pytest faz mesmo sem plugins externos** — i.e. parte do *core* do pytest, ou diferença residual no processo Python (sys.argv, sys.path, signal handlers, atexit, sys.modules pré-carregados via collection).
