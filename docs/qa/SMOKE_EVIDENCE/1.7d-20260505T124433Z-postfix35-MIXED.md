## Q-DRIFT-35 — diagnóstico

### Sintoma (postfix-34, commit 8cc2b38)
Smoke standalone WDOFUT morria em ~35s **sem traceback Python**. Log mostrava:
```
agent_resolver.unknown_id length=-2147483636  (×4)
```
`-2147483636` decimal = `0x80000004` reinterpretado como `c_int` signed.

### Root cause
No path `minimal_handshake=True`, `_configure_dll_signatures` é skipado integralmente (Q-DRIFT-12). Q-DRIFT-33 já registrava `TranslateTrade.argtypes/restype` cirurgicamente. Mas `GetAgentNameLength` e `GetAgentName` ficavam com defaults ctypes (argtypes None, restype c_int signed). Sem isso:
- Args sem coerção stdcall em x64.
- Retorno `c_uint32` (length positivo) interpretado como negativo gigantesco.
- Passar esse "length" como tamanho de buffer para `GetAgentName` poderia causar access violation nativa silenciosa.

### Fix aplicado (mesmo padrão Q-DRIFT-33)
`src/data_downloader/dll/wrapper.py` no path `minimal_handshake=True`:
```python
self._dll.GetAgentNameLength.argtypes = [c_int, c_int]
self._dll.GetAgentNameLength.restype = c_int
self._dll.GetAgentName.argtypes = [c_int, c_int, c_wchar_p, c_int]
self._dll.GetAgentName.restype = c_int
```
Mesmas signatures registradas no path full em `_configure_dll_signatures` (linhas 406-407 do wrapper.py).

### Faulthandler
`scripts/run_smoke_real_standalone.py` e `scripts/probe_history_minimal.py` agora chamam `faulthandler.enable(file=sys.stderr, all_threads=True)` antes de carregar a DLL. Isso captura access violations nativas com stack trace. Confirmado: no smoke postfix-35, faulthandler capturou múltiplos AVs em `_translate_trade_raw` que são RECUPERADOS (count `translate_failures=961` em `download.complete`).
