## Decisões do council (caminho v1.2.0)

1. **Mudança estrutural central — parquet-por-dia (ADR-025):** `{exchange}/{symbol}/{YYYY}/{MM}/{DD}.parquet`. Resolve 3 problemas: elimina O(N²) por mês, elimina o blocker `_PARTITION_ROW_LIMIT=5M` (cada dia ~500k << 5M), dá resume granular trivial (arquivo existe = dia feito). Migração one-shot dos parquets mensais existentes. **Se não couber na janela autônoma:** paliativo `_PARTITION_ROW_LIMIT` 5M→50M (aceita O(N²) ~3h extra/7anos) + parquet-per-day vira P0 da v1.2.1.
2. **translate_failures → 0:** traduzir o handle DENTRO do callback V2 (`make_history_trade_callback_v2` chama `TranslateTrade` e enfileira `TradeFields` copiado, não o handle). Tensão com R3 ("callback só put_nowait") — `TranslateTrade` é ~µs, não bloqueia a ConnectorThread perceptivelmente. + retry de chunk se `nl_errors/total > 0.001%` + métrica `completeness_pct` por chunk.
3. **Resume real:** `download(resume_job_id=...)` exposto + CLI `--resume <job_id>` funcional + auto-resume (se job `partial`/`in_progress` existe pro mesmo `(symbol,exchange,range)`, retomar) + fresh-path consulta partições já existentes e pula (não re-baixa o que já tem).
4. **Multi-symbol footgun:** deprecar `-p N>1` → cai no loop sequencial single-symbol com aviso; remover `orchestrator/broker/` (−2034 LOC) + os 4 test files do broker. (Multi-symbol real = Epic futuro: N processos, não N threads — Q08-E.)
5. **Reconnect-without-reinit (P1):** state monitor lê `_state_queue` durante o download; ao ver `MARKET_DATA != MARKET_CONNECTED`, pausa o orchestrator (não mata threads), re-handshake na DLL já inicializada (sem re-init — Q08-E), retoma do chunk corrente. Validar empiricamente que re-handshake sem re-init volta.
6. **UX long-haul:** estender `DownloadProgress` com `elapsed_s`/`eta_s`/`trades_failed`/`retries`; ProgressCard mostra ETA/throughput/tempo/trades-baixados/trades-perdidos/chunks-com-retry + log view com `setMaximumBlockCount(2000)`; preset de período "Desde 2018" / "Ano completo" + aviso `WAR_LARGE_PERIOD` ("~1750 dias, pode levar 30-90h, você pode pausar/retomar"); ao reabrir o app, detectar job incompleto no catalog → banner "Retomar download de WDOFUT? (parou em X/Y)". Pause/Resume na UI (depende do orchestrator — P1).
7. **Gate fixes:** `test_smoke_imports` derivar de `importlib.metadata` (não enrijecer literal); ruff UP047 → migrar p/ type params OU `ignore=["UP047"]` no pyproject.
8. **Cleanup:** `PRAGMA busy_timeout=5000` no SQLite; `get_pending_chunks` `OR`→UNION; deletar `ui/shortcuts.py` folclore; remover `BTN_REPEAT_LAST` órfão; retirar waiver `test_pool_lifecycle_broker_dead_code.md`; `KNOWN-TEST-FAILURES` → v1.2.0; `reconcile` lazy no boot (P1).
9. **Defer p/ v1.2.1+ (P2):** contextvars cross-thread real; CatalogScreen heatmap de cobertura + filtro por ano; `Path("data")` lazy nos comandos de manutenção (`integrity`/`validate`/`doctor`/`migrate`); `aiox compact` mensal offline; benchmarks novos (`bench_long_download`, `bench_parquet_append_cost`, implementar `bench_boot_cleanup`).
