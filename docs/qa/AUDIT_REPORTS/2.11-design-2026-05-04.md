## 2. Escopo

- **Em escopo:**
  - Confirmar implementação fiel das **3 camadas ADR-011 §"Decisão" Opção A**:
    L1 (`_internal/exceptions.py`) com `_InternalError` + 8 subclasses;
    L2 (`_internal/exception_adapter.py`) com `translate_to_public()` +
    `@translate_internal`; L3 (`public_api/exceptions.py`) com hierarquia
    pública `DataDownloaderError` + `OperationCancelled` + `ConnectionLost`
    adicionados.
  - Verificar **invariante de fronteira** "internals NÃO vazam em
    `public_api/`" via property test Hypothesis (`tests/property/test_no_internal_leak.py`).
  - Confirmar **`DownloadHandle.cancel()` H10 closure** conforme
    ADR-007a §"Cancel atômico" — graceful drain entre chunks (preserva
    INV-12), idempotência R5, cooperative cancellation.
  - Verificar **Microcopy IDs** (Uma authority R17) — novos `error.cancelled.*`,
    `error.connection_lost.*`, aliases `ERR_CONNECTION_LOST`,
    `SUC_CANCEL_DONE`, `ERR_CHUNK_FAILED`, `ERR_CATALOG_DRIFT`.
  - Classificar **SemVer impact** da mudança em `public_api/`.
  - Verificar **cooperative cancel**: `cancel_event` chegando ao
    orchestrator + check no boundary entre chunks (NÃO interrompe chunk
    em andamento).

- **Fora de escopo:**
  - Implementação interna de cada subclass (testada em
    `test_exception_hierarchy.py` — 30 unit tests).
  - Adapter pattern em isolamento (testado em `test_exception_adapter.py`
    — 29 unit tests).
  - Microcopy textual em PT-BR (Uma authority — review separado em
    COUNCIL-17 §D3).

- **Arquivos tocados (validação manual):**
  - `_internal/exceptions.py` (~160 linhas) — 9 classes (`_InternalError`
    + 8 subclasses); marker `_internal: ClassVar[Literal[True]]`.
  - `_internal/exception_adapter.py` (~190 linhas) — `translate_to_public`
    + `translate_internal` decorator; late-import para evitar ciclos.
  - `public_api/exceptions.py` (~265 linhas) — adicionados
    `OperationCancelled`, `ConnectionLost`, `humanized_message` property,
    `_PUBLIC_ERROR_MICROCOPY_ID` map (8 entradas).
  - `public_api/handle.py` (~365 linhas) — `cancel()`, `cancelled()`,
    `is_cancelled`, `is_cancelling()`, `peek_result()`, `result()` raise
    `OperationCancelled` quando status final = `"cancelled"`.
  - `public_api/download.py` (~535 linhas) — passa `cancel_event` ao
    `orchestrator.run()`; mapeia `JobResult` → `DownloadResult(status="cancelled")`.
  - `orchestrator/orchestrator.py` (~150 linhas modificadas) — `run(...,
    cancel_event=None)` opcional; check `cancel_event.is_set()` ENTRE
    chunks (linhas 410-419); `break` graceful.

- **Mudou public_api?** **SIM (aditivo).** Adicionados 2 tipos novos
  (`OperationCancelled`, `ConnectionLost`) + 1 property (`humanized_message`)
  + comportamento alterado em `result()` (raise vs return — soft breaking
  para callers que usavam `status == "cancelled"`).

- **Mudou fronteira interna?** **SIM (NEW).** `_internal/` agora é
  pacote oficial; antes não existia. Marker `_internal=True` permite
  auditoria mecânica.

## 4. SemVer Impact Analysis

| Mudança                                     | SemVer impact | Justificativa                                                                                            |
|---------------------------------------------|---------------|----------------------------------------------------------------------------------------------------------|
| `OperationCancelled` (NEW class)            | MINOR         | Aditivo — não renomeia/remove tipo existente.                                                            |
| `ConnectionLost` (NEW class)                | MINOR         | Aditivo — Q02-E exposto como tipo público dedicado.                                                      |
| `humanized_message` property (NEW)          | MINOR         | Aditivo em `DataDownloaderError` base. Subclasses não-mapeadas fallback `ERR_DLL_GENERIC`.               |
| `peek_result()` em `DownloadHandle` (NEW)   | MINOR         | Aditivo — utilitário non-blocking sem alterar API existente.                                             |
| `cancel(*, timeout=30.0) -> bool` (NEW)     | MINOR         | Substitui placeholder anterior (que retornava `None`); contrato aditivo (retorna `bool`).                |
| `cancelled()`, `is_cancelled` (NEW)         | MINOR         | Aditivo — non-blocking probes.                                                                           |
| `result()` agora raise `OperationCancelled` | **SOFT BREAKING** | Antes: retornava `DownloadResult(status="cancelled")`. Agora: raise. Caller que pegava `Exception` continua funcionando; caller que checava `status == "cancelled"` precisa atualizar para try/except. **Aceito** porque cancel real era findings H10 (não funcionava antes); breaking-soft é desejável (Pythonic — força tratar). |

**Veredicto SemVer:** **MINOR aditivo** (com soft-break documentado em
`result()`). Recomendar bump 0.3.0 → 0.4.0 antes de Epic 3 release. NÃO
exige MAJOR porque tipos existentes não foram renomeados/removidos.
