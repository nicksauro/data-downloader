  por run.

  `ParquetWriter.write` canônico (não `pq.write_table` raw). Per
  `parquet_writer.py:_build_metadata`, todo Parquet escrito carrega
  `b"schema_version" = b"1.0.0"`. Validado indiretamente: re-leitura via
  DuckDB pelos consumidores downstream (não exercitada no bench, mas garantida
  pelo path canônico).

  1.8 não introduz testes funcionais novos; valida apenas baselines de perf.
- [x] **`fsync(parent_dir)` pós-replace (durabilidade Linux/Windows)?** ✅ —
  inalterado em `ParquetWriter` (Story 1.4 audit Sol APPROVED).

  (Story 1.4 + tracking F-L-1 lá → 2.X).
