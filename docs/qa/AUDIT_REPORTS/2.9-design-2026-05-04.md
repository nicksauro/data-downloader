## 2. Escopo

- **Em escopo:**
  - Confirmar implementação fiel de **ADR-010 §Configuração** — pipeline
    structlog formal com contextvars merge + redaction + JSON renderer
    + cross-thread propagation.
  - Verificar **R21 hot path discipline** — `configure_logging` chamado
    UMA vez por boot; processors em cool path apenas; `redact_secrets`
    NÃO no hot path.
  - Confirmar **cross-thread propagation** via
    `contextvars.copy_context()` em IngestorThread + ProgressMonitor +
    public_api/download worker.
  - Verificar **redaction de secrets** recursivo + allow-list +
    case-insensitive substring match.
  - Confirmar **CLI flags** `--log-level` / `--log-format` + env vars
    `DATA_DOWNLOADER_LOG_*` + heurística TTY default.
  - Verificar **bind contextvars** em todos os entry points orchestrator
    (job + chunk separados, lifecycle limpo via try/finally).

- **Fora de escopo:**
  - `docs/dev/LOGGING.md` ops doc (deferred Story 2.12 docs sweep).
  - `docs/dev/LOG_SCHEMA.json` JSONSchema (deferred Story 2.12 docs sweep).
  - Linter custom para enforce R21 (Story futura).
  - PII home folder redaction (`redact_userprofile` ADR-010 deferred V2).

- **Arquivos tocados (validação manual):**
  - `observability/logging_config.py` (~530 linhas) — pipeline canônico
    + helpers contextvars + redaction recursive + cross-thread helpers
    + factory canônico.
  - `observability/__init__.py` — re-exports
    `configure_logging`/`bind_context`/`clear_context`/etc.
  - `orchestrator/orchestrator.py` — `bind_context(job_id, correlation_id,
    symbol, exchange)` no início + `clear_context()` no `finally` global;
    `bind_context(chunk_id=...)` em `_process_chunk` +
    `unbind_context("chunk_id")` no `finally`.
  - `orchestrator/download_primitive.py` — bind chunk_id REAL (uuid hex);
    IngestorThread + ProgressMonitor capturam contextvars do parent via
    `contextvars.copy_context()` no `__init__` (snapshot); `run()`
    executa via `ctx.run(self._run_inner)`.
  - `public_api/download.py` — worker thread executa via
    `parent_ctx.run(...)` (cross-thread propagation; preserva qualquer
    bind feito antes de `download()`).
  - `cli.py` — `@app.callback()` global instala `configure_logging` UMA
    vez por boot; flags `--log-level` / `--log-format` + heurística TTY.

- **Mudou public_api?** **NÃO.** Apenas re-exports adicionados em
  `observability/__init__.py` (canal NOVO opcional). `public_api/`
  intacto.

- **Mudou storage / DLL?** **NÃO.** Apenas dependência leitura de
  contextvars (sem mudança em wrapper).

  afetarem next. Documentado em COUNCIL-19 §2.

  em `*qa-gate` (HOT_PATH_RULES.md §Auditoria). Linter custom seria
