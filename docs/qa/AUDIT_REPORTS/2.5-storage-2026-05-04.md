**Justificativa:**

A Story 2.5 entrega exatamente o que prometeu — **substituir tabela
hardcoded de feriados B3 por leitura do arquivo oficial Nelogica
`holidays.dat`** com fallback graceful — sem **NENHUMA** violação das
garantias storage que Sol custodia (calendar_b3 é responsabilidade Sol
compartilhada com Quinn por gap detection):

1. **API pública preservada.** `is_holiday`, `is_b3_business_day`,
   `b3_business_days_range` mantêm assinatura. Tests existentes Story
   2.1 PASS sem regressão. Apenas adição de export `b3_holidays`
   (helper público).
2. **Graceful fallback (AC8) funcional.** `holidays.dat` ausente → log
   INFO + `_B3_HOLIDAYS_2020_2030` puro. CI sem ProfitDLL roda OK.
   Contributors externos sem licença não bloqueados.
3. **Cache mtime-based** em ambas as camadas (parser + calendar) —
   custo amortizado O(1) por chamada após boot. Lock thread-safe em
   primeiro uso simultâneo. Re-parse automático quando Nelogica
   atualizar DAT.
4. **União parser ∪ hardcoded captura completa.** Tabela `HOLIDAYS_DAT_FORMAT.md`
   §7 valida ground truth 2025 (15 feriados full-day + pontos facultativos
   + feriados FDS). Hardcoded recupera FDS (Nelogica omite); parser
   recupera 24/12 e 31/12 (hardcoded omite). Decisão D1 COUNCIL-16
   formalizada.
5. **Cobertura hardcoded estendida 2020-2030 (AC4)** — 143 entradas
   atestadas via tests parametrizados (`test_calendar_b3_extended.py`).
   Páscoa calculada via Gauss/Meeus para feriados móveis. Consciência
   Negra a partir de 2024 (Lei 14.759/2023). Range respeitando Sol
   policy ≥ 2020 (M17 DST).
6. **Validation source declarado.** `reverse_engineered` em
   `HOLIDAYS_DAT_FORMAT.md` (Nelo authority). Q16-OPEN ratificado como
   **resolved-com-caveat** (parser funcional via reverse engineering +
   tracking aberto para confirmação oficial Nelogica futura).
7. **DST boundary preservada.** Hardcoded começa em 2020; gap detection
   (Story 2.1 IntegrityChecker) continua respeitando Sol policy `>= 2020-01-01`.
8. **76+ novos tests, 0 regressão.** Suite completa 622 PASS + 5 SKIP
   (vs 622 PASS + 1 SKIP em HEAD anterior — os 4 SKIPs novos são
   condicionais ao real DAT estar presente, esperado em CI). Property
   tests Hypothesis (5 tests) garantem invariantes parser-vs-hardcoded.

A Story 2.5 fecha o **finding F-S-1** do Sol audit Story 2.1
(calendário hardcoded como caveat) e o **caveat COUNCIL-04** (pandas
para business-days com calendário ainda placeholder). Resolvido com a
estratégia união conservadora que minimiza falso positivo de gap.

**Próxima ação:** Nelo prossegue para `*audit` formato `holidays.dat`
(artefato dedicado em `2.5-dll-2026-05-04.md`); Quinn para `*qa-gate 2.5`.
