## 2. Escopo

- **Em escopo:**
  - Threading model: separação física `IngestorThread` + `ProgressMonitor` +
    OrchestratorThread (caller de `download_chunk`) + `ConnectorThread` (DLL).
  - Bounded queues: `_trade_queue` (`maxsize=100_000`, finding Pyro 1.4.5)
    + `_progress_queue` (`maxsize=1000`).
  - Política de back-pressure: `put_nowait` com `suppress(Full)` no callback;
    `get(timeout=...)` no consumer com `stop_event` para shutdown.
  - Fronteira `orchestrator/` ↔ `dll/`: primeira travessia formal — DLL
    consumida via `ProfitDLL` instance + tipos de `dll/types.py`.
  - Public API surface: `download_chunk`, `TradeRecord`, `ChunkResult`,
    `parse_brt_timestamp`, `format_brt_timestamp` re-exportados em
    `orchestrator/__init__.py` (mas NÃO em `public_api/` — Story 1.7a/b
    cuida da fronteira pública oficial).
  - Error propagation conforme ADR-011 (`ValueError` para entradas
    inválidas, `DLLInitError` para DLL não-inicializada, `nl_error_code`
    em `ChunkResult` para erros NL_*).
  - Hot path R21 (callback NÃO loga, NÃO aloca além do par).
  - Shutdown: `stop_event` + `join(timeout=10.0)` para IngestorThread,
    `join(timeout=5.0)` para ProgressMonitor.
- **Fora de escopo:**
  - Detalhes ctypes (Nelo).
  - Schema `TradeRecord` 17 campos (Sol — Story 1.4 já validou).
  - Microcopy de erro user-facing (Uma — Story 0.3 / 1.7a/b).
  - Multi-symbol orchestration (Story 1.7a/b).
- **Componentes tocados:**
  - `orchestrator/` — `download_primitive.py` (497L NOVO), `timestamp.py` (157L NOVO), `__init__.py` (re-exports)
  - `dll/` — extensoes em `types.py`, `callbacks.py`, `wrapper.py` (auditadas por Nelo)
- **Cruza fronteiras?** **SIM** — primeira travessia formal `orchestrator/` ↔
  `dll/`. Direção: `orchestrator/download_primitive.py` importa de
  `data_downloader.dll.callbacks` (factories) + `data_downloader.dll.types`
  (TConnectorTrade, TC_*) + `data_downloader.dll.wrapper` (ProfitDLL via
  `TYPE_CHECKING` + runtime injetado). DLL **NÃO importa** de orchestrator
  (correto — internals to public direction; ADR-007a respeitado).

**Adicionais (relevantes da `core_principles` do agente Aria):**

- **ADR-FIRST:** Decisões da story (V2 callback, TranslateTrade fora do
  callback, queue 100k, timeout 1800s, política 99% reconnect) já têm
  registro em COUNCIL-03 + QUIRKS.md. Sem necessidade de novo ADR — todas
  são detalhes de comportamento DLL ou parâmetros bem documentados.
  Pyro 1.4.5 finding sobre queue 100k já endossado em ADR-005 amendment.
- **DOMAIN É DOS ESPECIALISTAS:** Aria não opina sobre signatures
  `WINFUNCTYPE` (Nelo) nem sobre nullability dos 17 campos do schema (Sol).
- **BACK-PRESSURE OBRIGATÓRIA:** Para `trade_queue` no callback, "block
  back-pressure" da ADR-005 NÃO se aplica — callback NUNCA pode bloquear
  ConnectorThread (R3 absoluta). Política `put_nowait + suppress(Full)` é
  a única opção. Mitigação: queue **enorme** (100k) + IngestorThread
  drenando rápido (`get(timeout=0.1)`). Em produção, se Full virar real
  (perda de trades), counter externo (ADR-013 — futuro) detectaria.
- **PÚBLICO ≠ INTERNO:** `_IngestorThread`, `_ProgressMonitor`,
  `_INGESTOR_GET_TIMEOUT`, `_VALID_EXCHANGES`, `_system_time_to_ns`,
  `_ns_to_datetime` todos com prefixo `_` (privados ao módulo).
  `download_chunk`, `TradeRecord`, `ChunkResult`, `DEFAULT_TIMEOUT_SECONDS`,
  `TRADE_QUEUE_MAXSIZE`, `PROGRESS_QUEUE_MAXSIZE` em `__all__` (públicos).
  Fronteira clara

## 4. Achados (Findings)

### F-1 — INFO — `_trade_queue` overflow é silencioso (cinto-e-suspensório com queue 100k é correto, mas observabilidade futura é necessária)

- **Arquivo:** `dll/callbacks.py:215-216` (callback `suppress(Full)`); `orchestrator/download_primitive.py:459` (queue maxsize=100k)
- **Descrição:** Se IngestorThread sofrer pause (GC, OS) longa o suficiente
  para encher 100k slots, `put_nowait` no callback engole silenciosamente.
  Este é o **único caminho seguro** no callback (lançar = bloqueio
  ConnectorThread = R3 violado). 100k é confortável para 1.4.5 baselines
  (~1s buffer). Mas **detecção** via metric externa não existe ainda — Story
  2.1/2.X (perf optimization) deve adicionar counter
  `dll_trade_queue_full_total` (proposto em ADR-013 amendment).
- **ADR/INV ref:** ADR-005 (back-pressure), ADR-013 (observabilidade —
  proposto), Q15-OPEN (queue saturation behavior).
- **Sugestão de fix:** Story 2.1 — adicionar counter Prometheus-like
  (`dll_trade_queue_full_total`) incrementado quando `qsize() > 0.8 *
  maxsize` (warning) ou Full disparado (error). Endereçar Q15-OPEN com
  probe (resposta H4 em `OPEN_QUESTIONS_RESPONSES.md`). Sem ação para 1.3.
- **Trade-off alternativo:** Aumentar queue maxsize para `1_000_000`
  (Pyro provou que callback path é cheap). Pro: mais buffer. Con: memory
  spike — 1M tuples (`int, int`) = ~80MB. Decisão Pyro 1.4.5 = 100k está
  no ponto certo. Aria concorda.

### F-2 — INFO — Validação de exchange duplicada (orchestrator + dll wrapper)

- **Arquivo:** `orchestrator/download_primitive.py:448-452` + `dll/wrapper.py:641-646`
- **Descrição:** Tanto `download_chunk` (fronteira pública do orchestrator)
  quanto `ProfitDLL.get_history_trades` (fronteira pública do wrapper)
  validam `exchange in ('F', 'B')`. Defesa em profundidade — cada camada
  protege seus invariantes. Aria endossa (Nelo já anotou como F-1 LOW).
  Sem violação de design — apenas observação de DRY potencial.
- **ADR/INV ref:** R8/Q05-V.
- **Sugestão de fix:** N/A — design correto. Em refactor amplo (Story 2.X),
  poderia centralizar em `dll/types.py:validate_exchange(s)`. Sem ação para 1.3.

### F-3 — INFO — Public API surface diferida para Story 1.7a/b

- **Arquivo:** `orchestrator/__init__.py` (re-exports) + `public_api/__init__.py` (não modificado)
- **Descrição:** `download_chunk`, `TradeRecord`, `ChunkResult`,
  `parse_brt_timestamp`, `format_brt_timestamp` re-exportados em
  `orchestrator/__init__.py` mas NÃO em `public_api/`. Decisão correta:
  primitiva interna (1 chunk) não deve ser API pública — usuário deve
  consumir o `Downloader` facade (Story 1.7a/b) que orquestra chunking,
  retry, catalog. `__api_version__` permanece "0.1.0" (sem bump).
- **ADR/INV ref:** ADR-007a §SemVer + §public_api facade pattern.
- **Sugestão de fix:** Story 1.7a/b deve criar `data_downloader.public_api.downloader.Downloader`
  expondo `download(symbol, dt_start, dt_end) -> DownloadResult` como
  facade — `download_chunk` permanece como helper interno. Sem ação para 1.3.

### F-4 — INFO — `DEFAULT_TIMEOUT_SECONDS=1800` aplica-se per-chunk; orchestrator total deve aplicar timeout adicional em Story 1.7

- **Arquivo:** `orchestrator/download_primitive.py:84`
- **Descrição:** Timeout de 1800s/chunk é correto para Q02-E (99% reconnect).
  Em Story 1.7 quando orchestrator emitir múltiplos chunks (e.g. 30 dias
  WDO = 6 chunks de 5d), timeout total seria 6×1800s = 3 horas. Story 1.7
  precisa decidir: (a) timeout por chunk (default), (b) timeout total
  configurável por download, ou (c) timeout adaptativo por symbol (Q12-E).
  Sem ação para 1.3 — apenas anotação para Story 1.7 design.
- **ADR/INV ref:** ADR-005 amendment, Q02-E, Q12-E.

**Nenhum finding HIGH ou CRITICAL.**

## 5. Decisão

| Verdict             | Marcar |
|---------------------|--------|
| APPROVED            |  [x]   |
| CHANGES_REQUESTED   |  [ ]   |
| BLOCKED             |  [ ]   |

**Justificativa:** Design respeita thread model (ADR-005 + INV-1 + INV-11
preparação para 1.7a) com **separação física** real de 4 threads (1
ConnectorThread DLL + 1 OrchestratorThread caller + 1 IngestorThread + 1
ProgressMonitor). Filas bounded com política bem documentada (`100k` para
trade — finding Pyro 1.4.5; `1k` para progress) e back-pressure correta para
callback context (`put_nowait + suppress(Full)` — única opção segura
respeitando R3). Fronteiras de camada respeitadas (`orchestrator/` consome
`dll/`, sem importação reversa). Public API surface diferida adequadamente
para Story 1.7a/b (`Downloader` facade). Erro propaga em 4 níveis (ValueError
input, DLLInitError DLL não-init, ChunkResult.status para erros runtime,
translate_failures count) conforme ADR-011. Hot path R21 preservado
(callback sem log, IngestorThread sem alloc supérflua, struct reusado).
Shutdown LOCAL ao chunk via `stop_event` + `join(timeout=...)` com drenagem
final preserva TC_LAST_PACKET tardio. Findings F-1/F-2/F-3/F-4 são todos
INFO — apontam tracking para Story 1.7a/b/2.1, não bloqueantes para 1.3.
