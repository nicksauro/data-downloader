### F-P-5 — LOW — Heurística `_LOGGER_NAME_HINTS` pode ter falsos negativos

- **Arquivo:** `scripts/audit_hot_path.py:100-101 + 153-173`
- **Descrição:** `_is_logger_attr_call` identifica logger calls via
  match de substring (`log/logger/_log/_logger`) no receiver name.
  Falsos negativos teóricos:
  - Variável renomeada para algo como `tracker.info(...)` ou
    `metrics.warning(...)` (nomes não-canônicos para logger).
  - Logger acessado via subscript (`loggers[name].info(...)`) — não
    captura.
  - Logger retornado por função (`get_log().info(...)`) — pode capturar
    se função tem "log" no nome, senão não.
  Em prática, projeto usa convenção `log = structlog.get_logger(__name__)`
  consistentemente — heurística cobre ~99% dos casos. Cleanup futuro
  pode adicionar AST tracking de bindings se necessário (mais
  complexo, possivelmente over-engineering para este projeto).
- **ADR/INV ref:** N/A (heurística de tooling).
- **Sugestão de fix:** Tracking informal — se Story 2.X-cleanup
  encontrar falso negativo, expandir heurística OU mudar convenção
  para enforcement por nome (`# @hot_path_logger_alias` ou similar).
  Por agora, aceitar cobertura 99% como suficiente.
