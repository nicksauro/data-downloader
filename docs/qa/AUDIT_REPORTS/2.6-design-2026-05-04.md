## 2. Escopo

- **Em escopo (Aria authority — fronteiras + thread model + exception hierarchy):**
  - Confirmar que **state machine 3-state canônica** (CLOSED → OPEN
    → HALF_OPEN → CLOSED|OPEN) está implementada corretamente em
    `circuit_breaker.py` com transições documentadas.
  - Confirmar **thread-safety** via `threading.Lock` interno + invariante
    "função wrapped roda **FORA** do lock" para evitar deadlock se `fn`
    re-entrar no breaker.
  - Confirmar **cooldown amplificado capped** em `base_cooldown × 8`
    (anti-DoS — evita cooldown explosivo após probe failure consecutiva).
  - Confirmar **`RetryPolicy` é frozen dataclass** (imutável,

    + `breaker.call(fn)` API explícita ambas funcionando.
  - Confirmar **fronteira `public_api/` PRESERVADA** — `CircuitOpenError`
    herda `DataDownloaderError` (ADR-011) e já era tipo público em
    `public_api/exceptions.py`; semântica de `download(...)` para
    chamadores externos continua "raise se erro fatal".
  - Confirmar **integração orchestrator NÃO-QUEBRADORA** — `with_retry(fn)`
    sem `policy=` mantém comportamento Story 1.7a; `with_retry(fn, policy=...)`

    (`download_primitive` Story 1.3 + breaker NÃO conta como falha

    sem retry.

    `_get_breaker` — multi-symbol futuro (Epic 3) preparado.

## 3. Checklists

### 3.1 design_review (Aria checklist canônico — `agents/architect.md` `checklists.design_review`)

- [x] **Respeita thread model (ADR-005)?** **SIM.** `CircuitBreaker._lock`
  é `threading.Lock` interno; toda mutação de estado (`_state`,
  `_failures`, `_opened_at`, `_current_cooldown`) ocorre sob lock
  (`record_failure`, `record_success`, `state` property,
  `cooldown_remaining_seconds`). **CRÍTICO**: a função wrapped (`fn()`
  em `call()`) executa **FORA do lock** (linha 320) — evita deadlock
  se `fn` re-entra no breaker (ex.: wrap aninhado) e não bloqueia
  outros threads observando state durante longo download_chunk
  (1800s). Padrão clássico bem implementado. RetryPolicy é frozen
  (imutável) — naturalmente thread-safe.

- [x] **Respeita fronteiras de camada?** **SIM.** `circuit_breaker.py`
  importa **apenas** `public_api.exceptions.DataDownloaderError`
  (fronteira pública SemVer-tracked já usada por outras exceptions).
  `retry_policy.py` importa **apenas** `dll.error_taxonomy.ErrorCategory`
  + `categorize_nl` (consumer da taxonomia Nelo). Nenhum import de
  `storage/`, `ui/`, `cli.py`. **Sem dependências circulares**
  verificado por inspeção de imports.

- [x] **Toda fila tem `maxsize` E política de back-pressure?** **N/A
  direto.** Story 2.6 NÃO introduz novas filas. As filas existentes
  (DLL, write, ui_progress — ADR-005) mantêm `maxsize` e políticas.
  `CircuitBreaker._failures` é `deque[float]` mas com **eviction
  baseada em janela sliding** (`_evict_old_failures_locked` remove
  timestamps fora de `window_seconds`). Crescimento limitado por
  failure rate; em estado OPEN, novo failure é appended-and-evicted
  (não unbounded). Aria endossa: deque com sliding window é o
  padrão correto para circuit breaker (não precisa de maxsize formal
  porque eviction garante boundedness em regime estacionário).

- [x] **É idempotente onde precisa ser?** **SIM.**
  - `breaker.reset()` é idempotente — força CLOSED + clear failures
    + reset cooldown ao base; chamadas subsequentes retornam no-op.
  - `record_success(was_half_open=True)` é idempotente em estado
    CLOSED (apenas evicta janela).
  - `record_failure` em estado OPEN é idempotente (já bloqueando;
    apenas append em audit log).
  - `RetryPolicy(frozen=True)` é estruturalmente imutável — qualquer
    `==` é estável; pode ser instanciada N vezes com mesma semantics.
  - `policy_from_env()` é idempotente per env var snapshot — chamadas
    subsequentes com mesma env produzem mesma policy.
  - **Idempotência de `download(...)` para chamadores externos
    PRESERVADA** — circuit breaker é layer interna; chamador externo
    sempre vê "raise se fatal, sucesso se completed". Re-rodar
    `download(...)` não duplica trades (Story 1.4 dedup já garantiu).

- [x] **Introduz dep nova? Tem ADR?** **NÃO.** R10 (minimal deps)
  honrado: implementação dependency-free (`collections.deque`,
  `threading.Lock`, `time.monotonic`, `enum.StrEnum`). Trade-off
  documentado em COUNCIL-20 §D3: alternativa `pybreaker` (~3KB +
  dep transversal) rejeitada porque nossa state machine é simples
  (3 estados, ~150 linhas de lógica). Aria endossa: NÃO precisa de
  novo ADR — decisão é "preservar R10", não "adicionar dep".

- [x] **Schema Parquet alterado? Sol foi consultada?** **N/A.** Story
  2.6 NÃO toca `storage/`. Sol não precisa auditar.

- [x] **Wrapper DLL alterado? Nelo foi consultado?** **SIM, taxonomia
  NL_* (Nelo authority).** `dll/error_taxonomy.py` é nova (Nelo
  audita em `2.6-dll-2026-05-04.md` — APPROVED). `RetryPolicy`
  consome `categorize_nl` (Aria reusa, não duplica semântica).
  Consultoria explícita em COUNCIL-20 §D1 + §D4 com Nelo sign-off.

- [x] **Public API alterada? Bump de versão planejado?** **NÃO há
  breaking change na public_api.** `CircuitOpenError` JÁ ERA tipo
  público em `public_api/exceptions.py` (ADR-011 — pré-Story 2.6).
  Story 2.6 apenas IMPORTA + INSTANCIA de novo módulo interno
  `circuit_breaker.py`; semântica para chamadores externos
  inalterada. `with_retry(fn)` sem `policy=` mantém comportamento
  Story 1.7a (max_attempts=3, OSError/TimeoutError retryable, raise
  `RetryError` em exhaustion). NÃO requer SemVer bump.

## 4. Achados (Findings)

### F-A-1 — INFO — State machine canônica 3-state implementada com rigor

- **Arquivo:** `src/data_downloader/orchestrator/circuit_breaker.py:96-466`
- **Descrição:** Implementação fielmente alinhada com pattern Martin
  Fowler ("Circuit Breaker", 2014) — 3 estados (`CLOSED`/`HALF_OPEN`/`OPEN`)
  com transições documentadas em docstring + comentários. Trip via
  threshold em sliding window (não simples counter), recovery via
  cooldown + probe, backoff em probe failure (cooldown × 2 capped em
  ×8). Aria reconhece como **canônica e correta**. Test
  `test_amplified_cooldown_capped_at_8x_base` valida cap.
- **ADR/INV ref:** Pattern canônico circuit breaker (Fowler 2014 +
  Hystrix lineage); ADR-005 thread model.
- **Sugestão de fix:** Sem ação. Recomendado manter este código como
  **referência interna** para futuras state machines do projeto
  (orchestrator state machine Epic 3 pode espelhar o padrão).

### F-A-2 — INFO — Decisão "fn FORA do lock" é não-óbvia mas crítica

- **Arquivo:** `src/data_downloader/orchestrator/circuit_breaker.py:316-326`
- **Descrição:** Decisão de executar `fn()` fora do `with self._lock`
  é **não-óbvia** para revisores futuros — código alternativo
  (`with self._lock: result = fn(); record_success(...)`) PARECE mais
  simples mas introduz deadlock se `fn` re-entra no breaker (ex.:
  download_chunk usa threads internas) E congela outros threads
  observando state durante chunk longo (1800s timeout). Decisão
  documentada em COUNCIL-20 §D3 + comentário inline (linhas 316-318)
  EXPLÍCITO. Aria endossa.
- **ADR/INV ref:** ADR-005 (thread model — fn não bloqueia outras
  threads em estado read).
- **Sugestão de fix:** Sem ação. Comentário inline + COUNCIL-20
  documentam suficientemente.

### F-A-3 — INFO — Q02-E tratado em layer correto (não vaza para breaker)

- **Arquivo:** `src/data_downloader/orchestrator/circuit_breaker.py:41-45`
  (docstring Q02-E) + `download_primitive` (Story 1.3) + COUNCIL-20 §D4
- **Descrição:** Decisão arquitetural Q02-E (progress=99% reconnect)
  é tratada em **2 layers separadas com responsabilidade clara**:
  (a) `download_primitive` (Story 1.3) trata transparente — timeout
  duro 1800s sem progresso real promove para `status='timeout'`;
  retorna `status='completed'` se TC_LAST_PACKET chegou mesmo após N
  callbacks 99%. (b) Breaker NÃO recebe NL_* error code para 99%
  repeats (porque NÃO É error code), apenas para `status='timeout'`
  (TimeoutError) ou `status='failed'` (OSError com nl_code). Logo
  Q02-E aparece como SUCESSO para o breaker — exatamente como
  deveria (download completou). Test `test_circuit_breaker_does_not_count_q02e_progress_99_as_failure`
  valida. Aria endossa: separação correta de responsabilidades.
- **ADR/INV ref:** Q02-E `validated` (Story 2.6 close); ADR-005
  thread model.
- **Sugestão de fix:** Sem ação.

### F-A-4 — INFO — Decorator `@with_circuit_breaker(breaker)` é API

## 5. Decisão

| Verdict             | Marcar |
|---------------------|--------|
| ✅ **APPROVED**     | **[x]** |
| 🟡 CHANGES_REQUESTED | [ ]    |
| 🔴 BLOCKED          | [ ]    |

**Justificativa:**

A Story 2.6 entrega **state machine canônica de circuit breaker
3-state + RetryPolicy frozen dataclass** com qualidade arquitetural
exemplar:

1. **State machine canônica** (CLOSED ↔ OPEN ↔ HALF_OPEN) implementada
   fielmente ao pattern Fowler 2014. Trip via threshold em sliding
   window (deque + eviction baseada em `window_seconds`); recovery
   via cooldown + probe; backoff em probe failure (cooldown × 2
   capped em ×8 anti-DoS). 19 unit tests cobrem todas transições +
   2 thread-safety tests + 8 property tests Hypothesis.
2. **Thread-safety implementada com rigor** — `threading.Lock` interno
   protege todas as mutações de estado; **função wrapped roda FORA
   do lock** (decisão crítica anti-deadlock + anti-bloqueio durante
   chunk longo); lazy transition OPEN→HALF_OPEN economiza threads e
   elimina race de wakeup.
3. **Cooldown amplificado capped** em `base × 8` previne DoS contra
   o operador (sem cap, probe failures consecutivas escalariam
   exponencialmente). Test `test_amplified_cooldown_capped_at_8x_base`
   valida.
4. **`RetryPolicy` é frozen dataclass** com 7 env vars de override
   (factory `policy_from_env`) + fallback graceful (env malformada
   loga warning + cai em default — best-effort robustez). AC8
   satisfeita.
5. **Decorator + explicit API** ambas funcionando (`@with_circuit_breaker(breaker)`
   + `breaker.call(fn)`). Orchestrator usa explicit; decorator
   disponível para Epic 3 / public_api shadow.
6. **Fronteira `public_api/` PRESERVADA** — `CircuitOpenError` herda
   `DataDownloaderError` (ADR-011); JÁ ERA tipo público pre-Story
   2.6 (planejado em ADR-011); semântica de `download(...)` para
   chamadores externos inalterada (CircuitOpenError vira
   `failed_chunk` + segue próximo chunk; job result final
   `partial`/`failed`/`completed` inalterado). **SemVer impact = NONE.**
7. **`with_retry(fn)` sem `policy=` mantém comportamento Story 1.7a**
   (max_attempts=3, OSError/TimeoutError retryable, raise `RetryError`
   em exhaustion) — backward-compat preservado para chamadores
   internos legados.
8. **R10 (minimal deps) honrado** — implementação dependency-free
   (deque + threading.Lock + time.monotonic + StrEnum). Trade-off
   `pybreaker` documentado em COUNCIL-20 §D3 e rejeitado.
9. **R21 (cool path) honrado** — logger emite apenas em transições
   de estado / per-attempt; nunca per-trade. ADR-010 contextvar
   `correlation_id` herdado automaticamente.
10. **R7 (fail fast) honrado** — PERMANENT/UNKNOWN raise imediato
    sem retry; `should_retry` short-circuita `KeyboardInterrupt`/
    `SystemExit`.
11. **Lazy `_get_breaker` per (symbol, exchange)** prepara multi-symbol
    futuro (Epic 3) — cada par tem breaker isolado, sem contenção.
12. **Q02-E** tratado em layer correto (`download_primitive` Story 1.3
    + breaker NÃO conta porque NÃO recebe NL_* code para 99% repeats).
    Test sintético valida.

**Esta gate aprova design + implementação** dos 2 componentes
(`RetryPolicy`, `CircuitBreaker`) + integração no orchestrator. Os 4
checklists aplicados (`design_review` canônico + `thread_safety_review`
custom + `retry_policy_review` custom + `boundary_preservation_review`
custom) cobrem todas as preocupações arquiteturais relevantes para
state machines multi-threaded.
