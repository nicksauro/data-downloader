A Story 2.3 entrega o **Schema Migration Framework completo** que MIGRATIONS.md

checkpoint resumível + rollback documentado. **8/10 ACs PASS literal** + 2 ACs

**Highlights desta gate:**

1. **Schema canônico Parquet v1.0.0 INTACTO** (verificado por Sol audit + Aria
   design review). `SCHEMA_VERSION` em `storage/schema.py` e `pyarrow_schema()`
   preservados byte-a-byte.
2. **R4 (schema contrato perpétuo) preservada** com property test Hypothesis
   100 examples confirmando: `migrate(read_v1)` preserva 17 campos canônicos
   byte-a-byte; novo campo `liquidity_classification` sempre NULL.
3. **R5 (idempotência) preservada** em tripla defesa (checkpoint + applies_to
   + transform skip).
4. **INV-3 (atomicity) preservada** com pequena lacuna `fsync(parent_dir)`
   tracked como F-Q-3 LOW (mitigada por `.bak` + catálogo transacional).
5. **Two-phase commit reusado** (semântica Story 1.5): UPDATE de
   `partitions.schema_version` + `checksum_sha256` + `file_size_bytes` dentro
   de `with catalog._transaction()`, somente após `verify()` passar.
6. **Catálogo SQLite bumpado de forma controlada.** `CATALOG_VERSION 1.0.0 →
   1.1.0` aditivo puro (nova tabela `_migration_log` + index); tabelas
   existentes intactas; `_apply_migrations` aplica delta no boot
   (idempotente).
7. **Backup `.parquet.bak` por partição** criado **antes** de overwrite (AC7);
   rollback restaura via `os.replace` atômico; cleanup retention 30 dias
   (`migrate cleanup --older-than 30`).
8. **Fronteira `public_api/` intocada.** Aria APPROVED em `2.3-design-2026-05-04.md`.
   Diff `public_api/` = vazio. Framework é interno ao `storage/` (módulos com

   tests (Typer CliRunner) cobrem todos os comandos.

    Sol audit + Aria review formalizados.
