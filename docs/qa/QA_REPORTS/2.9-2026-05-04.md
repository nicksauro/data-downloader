**Justificativa:**

A Story 2.9 entrega exatamente o que prometeu — **pipeline structlog
formal (contextvars merge + redaction recursivo + JSON renderer +
cross-thread propagation), R21 hot path discipline preservado
integralmente, e cross-thread propagation via `contextvars.copy_context()`
cobrindo IngestorThread + ProgressMonitor + public_api/download worker**.

**7/8 ACs PASS literal + 1 PARTIAL** (AC8 `docs/dev/LOGGING.md` deferred
Story 2.12 docs sweep). ADR-010 amendment 2026-05-04 marca status
`accepted (implemented in Story 2.9)`.

**Highlights desta gate:**

1. **Pipeline structlog canônico ADR-010 V1** — `configure_logging`
   instala 8 processors em ordem (contextvars merge + thread name +
   log level + ISO 8601 UTC timestamp + redaction recursivo + stack
   info + dict_tracebacks + JSONRenderer/ConsoleRenderer). Chamada
   única no boot via `cli.py @app.callback()`.
2. **Cross-thread propagation via `contextvars.copy_context()`** —
   IngestorThread + ProgressMonitor + public_api/download worker
   capturam snapshot do parent thread no `__init__` e executam via
   `ctx.run()`. Preserva `bind_context(job_id=...)` do orchestrator
   em logs cross-thread. ADR-005 multi-thread compliance. Test
   integration valida 2 jobs concorrentes → contextvars isolados.
3. **Redaction defesa em profundidade** — substring matching
   case-insensitive contra `SENSITIVE_KEY_SUBSTRINGS` (10 substrings
   cobrem `PROFITDLL_KEY`/`nl_password`/`user_pass` sem enumerar cada
   combinação) + allow-list explícito + property test Hypothesis (100
   examples) garante INV-credenciais. `bind_context` também redacta
   kwargs antes do bind (defesa em profundidade — caso dev passe
   secret por engano).
4. **R21 hot path PRESERVADO integralmente** — `configure_logging` 1x
   boot; processors em cool path; `redact_secrets` NÃO no hot path;
   `copy_context()` snapshot 1x por thread. Pyro sign-off COUNCIL-19.
5. **CLI integration cross-platform** — `@app.callback()` global instala
   `configure_logging` UMA vez por boot; flags `--log-level` /
   `--log-format` + env vars `DATA_DOWNLOADER_LOG_*` + heurística TTY
   default (`isatty() → console` else `json`).
6. **Bind por escopo limpo** — Orchestrator bind `{job_id, correlation_id,
   symbol, exchange}` no início + `clear_context()` no `finally` global
   (lifecycle limpo, evita contaminação cross-job no mesmo thread).
   `_process_chunk` bind `chunk_id` + `unbind_context("chunk_id")` no
   `finally` (preserva job_id/symbol entre chunks).
7. **78 testes novos PASS, 0 regressão atribuível.** Lint ruff + mypy
   strict limpos. ADR-010 amendment 2026-05-04 marcado.
8. **Backwards compat preservada** — call sites existentes
   (`structlog.get_logger(__name__)`) continuam funcionando (apenas
   ganham campos automáticos via contextvars + redaction transparente).

**Esta gate desbloqueia:**

- **Epic 2 close (G-Quality-Final)** — uma das condições era
  "ADR-010 accepted + implementada"; **satisfeita**.
- **Epic 3 UI live log view** (Felix + Uma) consome formato JSON
  canônico parseable por Loki/ELK/CloudWatch.
- **Release V1 forense** — logs de produção auditáveis (todo evento
  traceable via `correlation_id` cross-thread) + seguros (zero
  credencial em log) + machine-parseable.
