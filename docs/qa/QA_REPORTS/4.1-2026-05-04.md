**Justificativa:**

A Story 4.1 entrega exatamente o que prometeu — **implementação fiel
ADR-015 Opção A**: broker no master + workers via `multiprocessing.Queue`
+ ACK protocol + pool persistente (H20 mitigação) + SQLite thread-bound
conn (Issue 1 Debug Log fix). **38 testes novos PASS, 0 regressão na
suite full (1055 PASS).**

**Highlights desta gate:**

1. **ADR-015 implementação fiel** — Aria APPROVED via audit
   `4.1-design-2026-05-04.md`. Sem reabertura de decisão arquitetural;
   D1 COUNCIL-25 preservada.
2. **Pool persistente confirmado** (H20 mitigação Pyro D2) — workers
   aquecidos reusados entre jobs; spawn cold-start uma vez por run.
   `_worker_main` top-level pickle-safe (Windows spawn).
3. **SQLite thread-bound conn no broker thread** (Issue 1 Debug Log
   fix crítico) — broker abre própria conn R/W na sua thread (mesmo
   db_path), preserva WAL semantics sem violar `check_same_thread`.
4. **ACK protocol garantia AC4** — worker bloqueia em ACK síncrono via
   UUID `request_id`; out-of-order ACK bufferizado e re-enfileirado
   (defensive FIFO); timeout 30s configurável raise `BrokerTimeoutError`.
5. **`BrokerCatalogClient` espelha contrato `Catalog`** (worker-side
   stub) — Orchestrator usa transparente; subset usado pelo
   Orchestrator (register_job, register_partition, register_gap,
   update_job_progress, get_completed_partitions). Methods não
   suportados (resume_job, get_pending_chunks) levantam
   `NotImplementedError` explícito.
6. **Q17-OPEN registrado** em `docs/dll/QUIRKS.md` — licença Nelogica
   multi-instância (probe humano futuro). Não bloqueia mock impl.
7. **Sem mudança fronteira public_api** (D6 COUNCIL-25) — multi-symbol
   apenas na CLI em V1.0; SemVer impact NONE.
8. **WAIVER smoke real estende COUNCIL-09 corretamente** (D3
   COUNCIL-25) — mesmo padrão 1.7b/1.8; sign-offs Aria+Pyro+Morgan
   implícitos; story-debt em `4.1-followup.story.md`; bloqueia release
   V1 não story.
9. **Bench mock 2.72x FAIL vs 3.2x WAIVED** — AC7 explicitamente
   "Pyro define exato após bench piloto"; bench mock é piloto;
   IPC overhead em payload mínimo (1 mut/job) explica gap; real
   downloads (multi-chunk × meses) terão IPC diluído. Re-validação
   4.1-followup.
10. **38 testes novos PASS, suite full 1055 PASS, 0 regressão**, lint
    ruff + mypy strict limpos.
