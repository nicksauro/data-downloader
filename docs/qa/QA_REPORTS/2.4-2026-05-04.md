**Justificativa:**

A Story 2.4 entrega o **Prometheus Observability Exporter V2**
antecipando a V2 deferred do ADR-013 (originalmente Epic 4 → Epic 2)
para satisfazer release readiness Story 1.7b smoke MVP. **5/8 ACs PASS
literal** + 3 ACs PASS parciais com tracking explícito (F-Q-1 auto-retry
de porta; F-Q-2 env var; F-Q-3 overhead bench Pyro; F-Q-4 OBSERVABILITY.md)
que **não** são bloqueantes — todas decididas em COUNCIL-15 §D5 yagni
V1 ou ADR-013 amendment ratificado.

**Highlights desta gate:**

1. **Protocol pattern correto (Aria fronteira preservada).** `MetricsEmitter`
   ABC `runtime_checkable` em `contracts/observability.py`;
   `NullMetricsEmitter` no-op default; orchestrator depende **apenas** do
   Protocol (verificado em `orchestrator.py:55`); `PrometheusExporter`
   implementa estruturalmente sem subclass. Aria APPROVED em
   `2.4-design-2026-05-04.md` com ADR-013 amendment ratificado.

2. **Hot path R21 REFORÇADO, não violado.** Validação multi-camada:
   - Design: COUNCIL-15 §D3 documenta tabela completa de call sites
     cool-path apenas.
   - Implementação: `orchestrator.py:683` incrementa `trades_received_total`
     1x por chunk batch (`if chunk_result.trades:`), nunca 1x por trade.
   - Test: `test_orchestrator_no_emitter_call_per_trade` valida explicitamente
     que job de N trades em 1 chunk gera **1** chamada (não N).

3. **Opt-in default OFF — zero overhead.** Sem flag `--metrics-port`,
   `NullMetricsEmitter` é o default. Métodos vazios `__slots__ = ()` —
   ~80ns dispatch Python sem alocações. Pyro endossou em COUNCIL-15 §D4.

4. **Public API intacto (extensão aditiva opcional).** `public_api/download.py`
   ganhou parâmetro `metrics_emitter: MetricsEmitter | None = None` em
   ambas funções públicas. Adição opcional com default `None` preserva
   backward compat. R-public-api (semver) NÃO exige bump major.

5. **8 counters + 5 gauges + 5 histograms canônicos** (ADR-013 §V1 +
   COUNCIL-05 §D3 finding `dll_drops_total`). Especificações declarativas
   single-source-of-truth via `_CounterSpec`/`_GaugeSpec`/`_HistogramSpec`
   em `prometheus_exporter.py` (decisão Dex+Aria COUNCIL-15 — `metrics.py`
   separado dispensado).

6. **MultiTargetEmitter alinhado com COUNCIL-15 §D6.** Composability futura
   (V1 structlog dump + V2 Prometheus HTTP) sem refactor do orchestrator.
   Implementa Protocol; delega para tuple imutável de targets.

7. **Sem novas deps além de `prometheus_client>=0.20`.** Autorizada
   explicitamente em ADR-013 §Decisão (Opção A). Lib é padrão da indústria
   (CNCF, used by Kubernetes, Grafana), thread-safe lock-free, mature.
   Sem `opentelemetry`, sem `flask`, sem `aiohttp` (`wsgiref.simple_server`
   é stdlib).

8. **+31 testes novos, 0 regressões.** Suite completa 622 PASS + 1 SKIP.
   Story 2.4 specific: 7 Protocol + 15 exporter + 4 CLI integration + 5
   orchestrator integration = 31/31 PASS em 8.89s.

9. **Lifecycle gerenciado em entrypoint (cli.py).** `start()` antes do
   download, `.stop()` em `try/finally`. Idempotência verificada por
   `test_lifecycle_start_stop_idempotent`. Daemon thread defensive layer
   (`prometheus_client` cria `ThreadingWSGIServer` daemon).

10. **Bind `127.0.0.1` por segurança** — exporter desktop local
    não-público. Sem opt-in explícito para `0.0.0.0` (segurança em design).

11. **Security:** `detect-secrets scan --baseline .secrets.baseline`
    Passed (exit 0). Sem credenciais em código novo. Sem print/log debug
    residual. Métricas nomeadas com prefixo `data_downloader_` previne
    colisão com outras métricas em prod compartilhada.

12. **ADR-013 amendment 2026-05-04 ratificado.** Implementação coerente
    com amendment ponto-a-ponto: Protocol em `contracts/`, PrometheusExporter
    em `observability/`, orchestrator aceita opcional, CLI flag, 8+5+5
    métricas canônicas, R21 reforçado, cardinality LRU deferred Epic 4.
    Aria audit §3.3 confirma alinhamento total.
