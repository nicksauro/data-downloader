# Story 0.3 — UX Foundation

## User Story

**As a** squad data-downloader prestes a consumir UX em Story 1.7b (CLI smoke MVP) e Epic 3 (UI desktop)
**I want** que Uma formalize 6 docs canônicos de UX em `docs/ux/` (PRINCIPLES, CLI_PATTERNS, MICROCOPY_CATALOG, THEME, FLOWS, WIREFRAMES placeholder) ANTES de Dex tocar em microcopy CLI ou Felix tocar em Qt
**So that** R17 (Uma reviewer obrigatório de fluxos/microcopy) deixe de ser teatro, MICROCOPY_CATALOG vire fonte única de NL_* → texto humano, e Epic 3 tenha base estética/funcional pronta.

---

## Acceptance Criteria

1. **AC1:** Existe `docs/ux/PRINCIPLES.md` documentando princípios UX do data-downloader:
   - Honestidade radical (microcopy nunca mente — ex: "99% reconectando" reflete realidade da DLL)
   - Foundation-first (UX prioriza clareza sobre estética até MVP)
   - Cross-platform CLI primeiro (toda funcionalidade exposta via CLI antes de Qt)
   - Falar a língua do quant (vocabulário: chunk, partição, baseline, gap, dedup — não "arquivo", "download")
   - Acessibilidade (NO_COLOR, fallback ASCII, screen-reader-friendly)
2. **AC2:** Existe `docs/ux/CLI_PATTERNS.md` com:
   - Layout Rich (Header, Body progress bars, Footer summary Panel)
   - Padrões de cor por severidade (info=cyan, warn=yellow, error=red, success=green)
   - Padrões de progress bar (chunk-level + global)
   - Política Ctrl+C: `KeyboardInterrupt` → graceful shutdown (drena dll_queue → drena write_queue → commit catálogo → exit code 130)
   - Microcopy do "99% reconectando" (vocabulário literal: "99% completo — reconectando à DLL (quirk Q-99-RECONNECT, manual §3.1). Aguarde até timeout de {N}s.")
   - Suporte a `NO_COLOR=1`, fallback ASCII, paleta dark mode
3. **AC3:** Existe `docs/ux/MICROCOPY_CATALOG.md` mapeando exaustivamente:
   - Cada `NL_*` (códigos de erro DLL conhecidos via Nelo `*errors-table`) → mensagem humana em PT-BR
   - Cada estado do orchestrator (`pending`, `in_progress`, `completed`, `failed`, `partial`, `cache_hit`) → label + descrição user-facing
   - Cada erro de validação/integridade → mensagem com sugestão de ação
   - Catálogo é máquina-consumível (YAML embedded ou tabela MD parseável) para Dex importar em runtime
4. **AC4:** Existe `docs/ux/THEME.md` com:
   - Paleta primária + secundária (light/dark)
   - Tokens de spacing, font-size (preparação para Qt QSS)
   - Hierarquia visual (H1/H2/Body/Caption/Label)
   - Ícones (placeholder por enquanto)
5. **AC5:** Existe `docs/ux/FLOWS.md` com diagramas (texto/Mermaid) dos 4 fluxos críticos:
   - Golden path: usuário roda `data-downloader download --symbol WDOJ26 ...` → progress → success
   - Resume flow: download interrompido → re-roda comando com `--resume {job_id}`
   - Cache hit flow: usuário re-roda mesmo comando → cache hit detectado
   - Error flow: erro NL_* → microcopy clara + sugestão de ação
6. **AC6:** Existe `docs/ux/WIREFRAMES.md` (placeholder para Epic 3):
   - Tela Download (golden path)
   - Tela Catálogo (lista de partições baixadas)
   - Tela Settings (env vars, paths)
   - Cada tela com nota "Detalhamento em Epic 3 / Story 3.x"
7. **AC7:** MICROCOPY_CATALOG cobre 99% dos códigos NL_* listados por Nelo (`*errors-table`). Códigos não cobertos têm fallback "Erro desconhecido NL_{code}: consulte equipe Nelogica."
8. **AC8:** Os 6 docs cross-reference entre si com links relativos.
9. **AC9:** Morgan + Quinn validam que cada AC de Story 1.7b (CLI smoke MVP) tem rastreabilidade para CLI_PATTERNS.md ou MICROCOPY_CATALOG.md.
10. **AC10:** Versão dos 6 docs é `v1.0.0`. Mudanças após esse marco em Epic 3 exigem bump.
