# Story 3.2 — CatalogScreen + SettingsScreen reais

## User Story

**As a** quant brasileiro sem terminal (target Epic 3) + Felix (frontend-dev)
**I want** as telas Catálogo (browse de partições baixadas com filtros, detail
panel e ações destrutivas com confirmação) e Configurações (4 seções: DLL /
Storage / Performance / About com persistência em `~/.data_downloader/config.toml`)
funcionais, substituindo os placeholders deixados pela Story 3.1, com microcopy
100% catalog-sourced (R17), padrões QT_PATTERNS aplicados (signals cross-thread
Queued, `DontUseNativeDialog`, `QSortFilterProxyModel`), e cross-screen wiring
via signals (Settings → Catalog data_dir; Settings → MainWindow DLL status)
**So that** o usuário pode efetivamente browsear o histórico baixado, validar
checksums, apagar partições com salvaguarda anti-erro (`MOD_DELETE_PERMANENT_*`),
configurar DLL/storage e ver status do app — fechando os 3 fluxos primários
prometidos para Epic 3 (download / browse / config).

## Acceptance Criteria

1. **AC1 — `CatalogScreen` 5 estados:** `QWidget` com `QStackedWidget` interno
   alternando entre `loading / normal / empty / empty_filtered / error`.
   - **loading:** texto `LBL_CATALOG_LOADING`.
   - **normal:** `QTableView` + `QSortFilterProxyModel` mostrando partições
     com colunas (Símbolo / Bolsa / Período / Trades / Tamanho MB / Atualizado /
     Schema) + detail panel embaixo + footer summary
     `LBL_CATALOG_FOOTER_SUMMARY`.
   - **empty:** ícone xl 📁 + `EMP_CATALOG_FIRST_RUN_*`.
   - **empty_filtered:** `EMP_CATALOG_FILTER_NO_MATCH_*` + `BTN_CLEAR_FILTERS`.
   - **error:** banner com microcopy `ERR_*` + RETRY + RECONCILIAR.

2. **AC2 — `CatalogScreen` filtros client-side:** Search box (`Ctrl+F` foca) +
   combo exchange (Todas / F / B). `_PartitionFilterProxy` override
   `filterAcceptsRow` combinando substring symbol + exchange exato. Esc limpa
   filtros (se algum ativo); senão no-op (`handle_escape`).

3. **AC3 — `CatalogScreen` ações:** Detail panel mostra pasta, schema, DLL,
   checksum (com prefix sha256:8), row count. Botões: `BTN_REVALIDATE_CHECKSUM`,
   `BTN_OPEN_FOLDER`, `BTN_DELETE` (variant=destructive). Modal delete exige
   user click no botão DestructiveRole (microcopy `MOD_DELETE_PERMANENT_BODY`
   + `MOD_DELETE_PERMANENT_HINT` + `BTN_DELETE_CONFIRM`).

4. **AC4 — `CatalogAdapter` em QThread:** `QObject` movido para `QThread`
   próprio (D2 COUNCIL-23 herdada). Slots `list_partitions`,
   `delete_partition`, `revalidate_checksum`, `reconcile`. Sinais tipados
   (`Signal(object)` carregando `tuple[Partition, ...]` ou `ReconcileReport`).
   Conexões cross-thread via `Qt.QueuedConnection` explícito. `Catalog`
   instanciado com `auto_reconcile=False` (D3 COUNCIL-24).

5. **AC5 — `SettingsScreen` 4 seções em `QScrollArea`:** Cada seção em
   `QGroupBox`:
   - **ProfitDLL:** status (label dinâmico via QSS property `status`),
     DLL path + browse, env vars (PROFITDLL_KEY/USER/PASS) com toggle
     show/hide, botões TESTAR CONEXÃO + ABRIR PASTA DLL.
   - **Storage:** data_dir + browse (`DontUseNativeDialog`), free space,
     status catálogo (count partições), botões ABRIR EXPLORER /
     VERIFICAR INTEGRIDADE / RECONCILIAR.
   - **Performance (read-only):** display de defaults (DLL queue size 8192,
     storage queue 2048, chunk 30 dias, max retries 3, SQLite profile do
     env var) + nota `LBL_PERF_NOTE_ADVANCED`.
   - **About:** versão app (de `__api_version__`), versão DLL (placeholder,
     atualiza após test connection), schema version (de `CATALOG_VERSION`),
     links docs/bug, lista de agentes squad (10 emojis).

6. **AC6 — `SettingsScreen` 5 estados:** `normal` / `loading` (durante test
   connection) / `error` (test failed) / `empty` (.env não configurado —
   passos `EMP_SETTINGS_DLL_FIRST_RUN_*`) / `success` (após salvar — toast
   `TST_SETTINGS_SAVED`).

7. **AC7 — `SettingsScreen` persistência TOML:** `_on_save_clicked` escreve
   em `~/.data_downloader/config.toml` (ADR-012) com `dll_path` e `data_dir`.
   Env vars NUNCA persistidos em TOML (segurança). `data_dir_changed` signal
   emitido após save.

8. **AC8 — `DontUseNativeDialog` em folder picker:** `_on_change_data_dir_clicked`
   passa `QFileDialog.Option.DontUseNativeDialog` (QT_PATTERNS §1, finding M9).

9. **AC9 — Microcopy 100% catalog-sourced (R17):** ~75 IDs novos em
   `microcopy_loader.MSG` para CatalogScreen + SettingsScreen
   (MICROCOPY_CATALOG.md §17b.2 e §17b.3). Test verifica que nenhum label
   mostra `<microcopy id not found>`.

10. **AC10 — Cross-screen wiring:** `MainWindow._build_screens` conecta
    `settings_screen.data_dir_changed → catalog_screen.set_data_dir` e
    `settings_screen.dll_status_changed → main_window._on_dll_status_changed`
    (statusbar reflete DLL status).

11. **AC11 — Tests headless pytest-qt:** 26 tests (13 CatalogScreen + 13
    SettingsScreen) PASS via `QT_QPA_PLATFORM=offscreen`. Sem regressão
    nos 24 tests da Story 3.1 (total 50/50 PASS).

12. **AC12 — Lint clean:** `ruff check src/data_downloader/ui/
    tests/integration/test_ui_*.py` All checks passed. Mypy strict pulado
    (ADR-003 — `data_downloader.ui.*` em `ignore_errors`).
