# Story 1.5b — read_continuous + queries DuckDB canônicas + property tests rollover

## User Story

**As a** projeto downstream (backtest engine, signal generator) que vai consumir os Parquets produzidos pelo data-downloader
**I want** uma função `read_continuous(symbol_root, start, end, exchange)` que abstrai rollover de contratos (ex: WDOH26 → WDOJ26 no fim de março), e queries DuckDB canônicas em `docs/storage/QUERIES.md` cobrindo casos comuns
**So that** projetos downstream não reinventam SQL toda vez, e rollover funciona sem race conditions ou gap silencioso.

---

## Acceptance Criteria

1. **AC1:** `src/data_downloader/storage/duckdb_reader.py` ganha método `read_continuous(symbol_root: str, start_ts_ns: int, end_ts_ns: int, *, exchange: str = 'F') -> pa.Table`.
2. **AC2:** `read_continuous` consulta tabela `contracts` (Story 1.6) para resolver lista de contracts vigentes no range, lê os Parquets de cada um, concatena com ordenação por `timestamp_ns`, retorna pa.Table único.
3. **AC3:** Quando rollover ocorre (ex: range [2026-03-25, 2026-04-05] cobre WDOH26 + WDOJ26), `read_continuous` adiciona coluna `current_contract: string` indicando qual contrato cada trade pertence (finding M16).
4. **AC4:** Existe `docs/storage/QUERIES.md` com 5+ queries canônicas DuckDB:
   - Query 1: "trades por dia" (group by date)
   - Query 2: "OHLC por minuto" (group by minute)
   - Query 3: "volume agregado por símbolo / range"
   - Query 4: "detect gaps" (gaps em sequência temporal)
   - Query 5: "rollover detection" (mudança de contract code)
5. **AC5:** Cada query em QUERIES.md tem exemplo executável (input → output) e nota de performance.
6. **AC6:** Property test (Hypothesis): para qualquer range que cubra rollover, `read_continuous(root, start, end)` retorna trades de ambos contratos sem gap nem duplicata na fronteira.
7. **AC7:** Property test: ordenação garantida — `result["timestamp_ns"]` é monotonic non-decreasing.
8. **AC8:** Test integration: smoke gated env baixa WDOH26 (1 dia) + WDOJ26 (1 dia), `read_continuous("WDO", start, end)` retorna trades dos dois.
9. **AC9:** Reconcile (Story 1.5 AC11) é chamado opcionalmente no início de `read_continuous` (flag `--reconcile`) — default false para não pagar custo em queries hot.
10. **AC10:** Aria valida que assinatura de `read_continuous` está em `public_api/` (será exposta para projetos downstream — Epic 4).
