# Story 1.6 — Contract calendar (WDO map vigente)

## User Story

**As a** orchestrator (Story 1.7) ou usuário CLI
**I want** uma função `vigent_contract(symbol_root, on_date)` que resolve "WDO" + data → "WDOJ26" (ou erro se não houver vigente), populada inicialmente para WDO 2025-2026 e validada via probe na DLL
**So that** Story 1.7 baixe contrato vigente correto sem chutar letras de mês (lei R9), e usuário não use `WDOFUT` (que retorna 0 trades — quirk Nelo).

---

## Acceptance Criteria

1. **AC1:** Sol cria `docs/storage/CONTRACTS.md` documentando: tabela de letras de mês CME/B3, regras de vigência por símbolo (WDO mensal, WIN trimestral H/M/U/Z), e mapa inicial de contratos validados.
2. **AC2:** `src/data_downloader/orchestrator/contracts.py` expõe função `vigent_contract(symbol_root: str, on_date: date, *, exchange: str = 'F') -> str`.
3. **AC3:** Função consulta tabela `contracts` no SQLite catalog (Story 1.5) — não calcula, faz lookup.
4. **AC4:** Levanta `InvalidContract(symbol_root, on_date)` se não houver contrato vigente.
5. **AC5:** Função auxiliar `populate_contracts_from_seed(catalog: Catalog, seed_path: Path) -> int` carrega tabela inicial de `docs/storage/CONTRACTS.md` (formato YAML embutido OU tabela Markdown parseável).
6. **AC6:** CLI command `data-downloader contracts validate {symbol_root} {contract_code}` faz probe via DLL (Story 1.2 init) chamando `GetHistoryTrades` em primeiro dia vigente; valida que retorna trades > 0 (delegação a Nelo `*probe-dll` via código).
7. **AC7:** `validate` atualiza tabela `contracts` com `validated_at = now()`, `validation_source = 'dll_probe'`.
8. **AC8:** Mapa inicial popula no mínimo: `WDOJ26` (vigente abr/26), `WDOK26` (mai/26), `WDOH26` (mar/26 — referência do passado próximo). Datas de vigência: hipótese inicial = penúltimo dia útil mês X-1 até penúltimo dia útil mês X (validar com Sol + Nelo via probe).
9. **AC9:** Property test: para todo `(root, date)` em range testado, `vigent_contract(root, date)` retorna contract com `vigent_from <= date <= vigent_until`.
10. **AC10:** Test integration: smoke test (gated por env) chama `validate WDO WDOJ26` e confirma probe retorna trades.

### Constraints CRÍTICAS
- **Não chutar:** mesmo que pareça óbvio que `WDOJ26` = abril 2026, validar via probe.
- **Lookup, não cálculo:** função `vigent_contract` consulta SQLite — não roda algoritmo. Algoritmo está implícito na seed inserida.
- **Probe usa Story 1.3:** depende de DLL inicializada. Smoke test gated por env.

    vigent_until: 2026-03-30
    validated_at: null  # preencher após probe

    month_letter: J
    year: 2026
    vigent_from: 2026-03-31
    vigent_until: 2026-04-29
    validated_at: null
    validation_source: hypothesized
