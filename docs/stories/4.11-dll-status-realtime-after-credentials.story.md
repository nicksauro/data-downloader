# Story 4.11 — DLL Status reativo após Save

## Contexto

Após Story 4.7 (credenciais persistem), próximo gap UX:

1. Usuário abre Settings com `.env` ausente → DLL Status =
   "Não configurado" + empty state hint.
2. Digita credenciais → clica Save → toast "OK".
3. **DLL Status continua "Não configurado"** até o usuário clicar
   "Testar Conexão" manualmente.

Esperado: depois do Save com credenciais válidas, DLL Status muda
automaticamente para "Configurado, não testado" (ou dispara um
auto-test silencioso se config flag permitir).

## User Story

**As a** usuário que acabou de digitar credenciais

**I want** ver o status DLL refletir imediatamente que está
configurado (ainda que não testado)

**So that** sei que o Save "pegou" sem precisar clicar Test Connection.

## Acceptance Criteria

1. **AC1 — `_on_save_clicked` chama `_set_dll_status` apropriado.**
   - Após gravar `.env` + `os.environ`, se as 3 vars estão
     populadas → `_set_dll_status("disconnected")` (configurado
     mas não testado, microcopy LBL_DLL_STATUS_DISCONNECTED).
   - Se vazias → mantém `not_configured`.

2. **AC2 — Empty state hint some quando configurado.**
   - `self._dll_empty_hint.setText("")` quando 3 vars setadas.
   - Reaparece se usuário esvaziar campos + Save.

3. **AC3 — MainWindow status bar reflete também.**
   - Signal `dll_status_changed` já é emitido — bar atualiza
     automaticamente (cross-screen wiring já cabeado).

4. **AC4 — Auto-test connection (opt-in flag, V1.0.3 default OFF).**
   - Config opcional `settings.auto_test_on_save: bool = False`.
   - Se True, dispara `_on_test_connection_clicked()` automaticamente
     após Save bem-sucedido.
   - V1.0.3 mantém OFF para evitar latência inesperada; V1.1+
     considera flip para ON com microcopy clara.

5. **AC5 — Smoke Pichau.**
   - .env vazio → abre Settings → digita 3 creds → Save →
     DLL Status muda de "Não configurado" para "Configurado,
     não testado" sem clique adicional.

## Out of Scope

- DLL probe automático em background (V1.1 polish).
- Notificação push quando DLL desconecta mid-session.
