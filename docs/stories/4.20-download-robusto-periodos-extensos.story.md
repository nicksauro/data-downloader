## Contexto / Problema

Pichau: "a DLL tem que traduzir todos os trades... lançar uma nova V mais robusta. Melhorar UX/download/interface, caçar bugs (zero bugs), refinar download ao máximo, garantir baixar períodos extensos (desde 2018), propor melhorias." Council de 6 agentes auditou; achados consolidados em `docs/qa/V1.2.0-PLAN.md`.

## Acceptance Criteria

- [x] **AC1 — `translate_failures` ≈ 0:** o callback V2 de trade histórico traduz o `handle` (`a_pTrade: Pointer`) **dentro do escopo do callback** (o handle só é válido ali — a DLL recicla o buffer) e enfileira o `TradeFields` copiado, não o handle. Antes ~0.01% dos trades eram perdidos por handle stale → `PopulateTradeV0` AV interna da DLL. R3/ADR-005 emendado. + retry de chunk se `completeness_pct < 99.99%` (máx. 2); métrica `download_chunk_completeness_pct`. (Q-DRIFT-40.)
- [x] **AC2 — Resume funcional:** `--resume <job_id>` na CLI (era no-op) + auto-resume (job incompleto pro mesmo `(symbol,exchange,range)` retoma automaticamente) + `download(symbol,start,end,*,resume_job_id=...)` na public API + fresh-path skip (re-rodar é idempotente, baixa só o que falta).
- [x] **AC3 — Granularidade diária do catálogo:** nova tabela `chunk_ledger` (1 row por dia útil baixado, status `completed`/`no_trades`/`failed`); `CATALOG_VERSION` 1.1.0→1.2.0 (migração automática); `resume`/cache-hit/fresh-skip operam por dia (fim da perda silenciosa do resume mensal — um mês "tocado" no dia 12 contava como completo).
- [x] **AC4 — Períodos extensos não abortam:** `_PARTITION_ROW_LIMIT` 5M→50M (paliativo; solução definitiva = ADR-025 parquet-por-dia, v1.2.1). Um mês de WDOFUT (~10-13M trades) não levanta mais `IntegrityError` por volta do 10º dia útil.
- [x] **AC5 — UX long-haul:** `DownloadProgress` += `elapsed_s`/`eta_s`/`trades_failed`/`retries`; ProgressCard com ETA / tempo decorrido / throughput / trades baixados / trades perdidos / chunks com retry + log view cap (2000 blocos) + "Abrir Pasta" durante download + "Calculando plano…" quando `total==-1`; PeriodPicker com presets ("Mês corrente"/"Último ano"/"Ano completo:YYYY"/"Tudo desde 2018"/"Personalizado") + aviso de duração para ranges grandes; banner "Retomar download de {symbol}?" ao reabrir o app (`Catalog.list_jobs`).
- [x] **AC6 — Multi-symbol footgun removido:** `download SYM1 SYM2 -p N>1` não usa mais o mock factory (que gravava dados falsos silenciosamente) → deprecado, cai no loop sequencial single-symbol com aviso. `orchestrator/broker/` (~2034 LOC) e 4 test files removidos.
- [x] **AC7 — Cleanup:** `busy_timeout=5000` no SQLite; `get_pending_chunks` `OR`→`UNION` + `idx_partitions_job`; `ui/shortcuts.py` deletado (folclore); `BTN_REPEAT_LAST` removido; waiver `test_pool_lifecycle_broker_dead_code.md` removido; `test_smoke_imports` deriva da fonte; `ruff UP047` → `ignore` (diferido v1.3.0).
- [x] **AC8 — Suite verde:** `ruff` 0 / `mypy --strict` 0 (87 source files) / `pytest` 1194 unit + 508 integration+property, 0 fail (3 skips ambientais).
