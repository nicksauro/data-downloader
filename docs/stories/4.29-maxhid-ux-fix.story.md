## Contexto / Problema

Em 2026-05-17, usuário Pichau reportou: "não está baixando, a DLL não loga nunca, conecta marketdata infinidamente no .exe". Investigação (relatório Aria 2026-05-17) confirmou causa raiz com 95% confiança: servidor Nelogica responde `ActivationResult=MaxHID` ("Todos os seus logins estão em uso") **178ms após login**. DLL marca `Profit Dll Valid=False`, NÃO emite `MARKET_CONNECTED=4`, `wait_market_connected` Python fica drenando queue até timeout 300s × 3 = ~990s.

Problemas de UX expostos:
1. **Spinner infinito** sem mensagem clara — usuário não sabe que é problema de licença
2. **Logs DLL escondidos** em `<install>/_internal/Logs/` (não ao lado do `.exe`) — usuário disse "DLL não loga nunca" porque procurou no lugar errado
3. **Mensagem de erro genérica** `ERR_DLL_MARKET_TIMEOUT` quando finalmente falha — não acionável

Solução: detectar `MaxHID` proativamente no LogDesktop, levantar exception específica com remedies prescritivos, expor botão "Abrir pasta de logs" para self-service.

Função que abre o `LogDesktop_YYYY_MM_DD.log` mais recente em `<install>/_internal/Logs/`, busca último bloco `TInfoClientProcessor.ProcessLoginResult`, extrai `ActivationResult`, `Mensagem`, `HardLogout`, `LoginResult`. Retorna dataclass `LoginResultSnapshot(activation_result: str, message: str, hard_logout: bool, login_result: str, timestamp: datetime)`. Retorna `None` se log ausente ou bloco não encontrado. Resiliente a logs incompletos (linha parcial no final).

- [x] **AC2 — `MaxHIDError(DLLInitError)` exception** em `src/data_downloader/public_api/exceptions.py`. Atributos: `activation_result`, `server_message`, `timestamp`. `humanized_message = "ERR_DLL_MAX_HID"`. `__str__` lista 3 remedies prescritivos:

## Notas de execução

**Modo autônomo:** Story criada e implementada na mesma sessão. Squad: Sol implementa, Quinn audita, Gage push. Sem pause manual entre fases.

**Convenção commits:** `tipo(escopo): descrição [Story 4.29]`. Scope sem caracteres especiais.

**Riscos:**
- **Log path em frozen mode** — `bundle_paths.exe_dir()` resolve para `<install>/` em frozen e `cwd` em dev. `_internal/Logs/` é relativo ao exe_dir em frozen. Validar com smoke real após implementação.
- **Parsing frágil** — formato do log Nelogica pode mudar entre versões DLL. Parser deve ser defensivo (regex tolerant a whitespace + encoding).
- **Race condition** — DLL pode estar escrevendo no log enquanto Python lê. Usar `open(path, 'r', errors='ignore')` + ler em chunks.
- **Performance** — `wait_market_connected` é hot path do boot. Detecção MaxHID deve ser proativa (<1s) e cacheada (não reler log múltiplas vezes).
