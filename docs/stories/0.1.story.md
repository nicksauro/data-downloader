# Story 0.1 — Environment Bootstrap

## User Story

**As a** squad data-downloader prestes a iniciar Story 1.1 (scaffolding)
**I want** que Gage inicialize git, configure branch protection, escreva `.gitignore` definitivo, crie `bootstrap-dll.ps1` (conforme ADR-008), e estabeleça primeiro commit limpo
**So that** R12 (git como fonte única) deixe de ser inoperante, R18 (secrets) seja enforced desde o commit zero, e DLL distribution não vire bagunça implícita.

## Acceptance Criteria

1. **AC1:** `git init` executado em `C:\Users\Pichau\Desktop\data-downloader\`. `git status` reporta repo inicializado, branch padrão = `main`.
2. **AC2:** `.gitignore` raiz contém no mínimo: `.env`, `.env.*`, `data/`, `dist/`, `build/`, `__pycache__/`, `*.pyc`, `.pytest_cache/`, `.coverage`, `htmlcov/`, `.mypy_cache/`, `.ruff_cache/`, `.venv/`, `venv/`, `*.tmp.*`, `*.parquet` (raiz — em data/ liberado), `*.duckdb`, `*.sqlite-wal`, `*.sqlite-shm`, `profitdll/dlls/*.dll` (conforme ADR-008 commit decision), `profitdll/dlls/*.dat` se aplicável.
3. **AC3:** Branch protection model definida em `docs/release/BRANCH_MODEL.md`:
   - `main` protegida (no force push, no direct commit, PR required)
   - Branch naming: `epic-{N}/story-{N.M}-{slug}` (ex: `epic-1/story-1.1-scaffolding`)
   - Branch isolation: Felix Epic 3 e Dex Epic 2 trabalham em branches independentes
4. **AC4:** Existe `scripts/bootstrap-dll.ps1` que:
   - Verifica presença de `profitdll/dlls/ProfitDLL.dll` (e companions necessárias conforme ADR-008)
   - Se ausente, mostra instrução clara (Nelogica EULA + caminho de download)

   - Retorna exit code 0 se OK, !=0 se ausente/incorreto
5. **AC5:** Primeiro commit `chore: initial bootstrap (story 0.1)` contém:
   - `.gitignore`
   - `docs/release/BRANCH_MODEL.md`
   - `scripts/bootstrap-dll.ps1`
   - Estrutura de diretórios já existente (docs/, agents/, etc.) sem secrets, sem DLL, sem `.env`
6. **AC6:** `.env.example` (já planejado em 1.1) referenciado e validado: deve estar versionado; `.env` real DEVE ser bloqueado.
7. **AC7:** Hook pre-push esqueleto criado em `scripts/git-hooks/pre-push.ps1` (instalação completa fica para Story 0.2 com framework pre-commit).
8. **AC8:** README raiz atualizada (seção Setup) referenciando `scripts/bootstrap-dll.ps1` e `docs/release/BRANCH_MODEL.md`.
9. **AC9:** Aria valida que ADR-008 (DLL Distribution Strategy) está accepted antes de finalizar AC4.
10. **AC10:** Smoke check: clonar repo em diretório limpo + rodar `bootstrap-dll.ps1` produz feedback claro (sem crash) em máquina sem DLL.
