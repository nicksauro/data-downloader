# Story 1.2 — DLL wrapper: init/finalize + state callback

## User Story

**As a** orchestrator do data-downloader
**I want** um wrapper Python da ProfitDLL que carrega a DLL Win64, valida companions, silencia o log nativo, inicializa em modo market-only com **todos os 11 callback slots** preenchidos (3 ativos + 8 NoopCallback), drena a sequência completa de connection states em thread separada, e finaliza limpamente preferindo `DLLFinalize` sobre `Finalize`
**So that** as stories 1.3 (history callbacks) e 1.7 (orchestrator) podem chamar `GetHistoryTrades` em uma DLL conectada **sem** que `SetHistoryTradeCallback` posterior corrompa por slots `None` deixados na inicialização (Q11-E, Sentinel §12).

---

## Acceptance Criteria

1. **AC1 — Surface pública**
   `src/data_downloader/dll/wrapper.py` expõe classe `ProfitDLL` com:
   - construtor `__init__(dll_path: Path | None = None)`
   - método `initialize_market_only(key: str, user: str, password: str) -> None`
   - método `wait_market_connected(timeout: int = 60) -> bool`
   - método `finalize() -> None`
   - context manager `__enter__/__exit__`
   - property `dll_version: str` (ver AC13)

2. **AC2 — Inicialização com 11 callback slots fixos**
   Wrapper usa `DLLInitializeMarketLogin` (assinatura **11 args**, não `DLLInitializeLogin` que é trading) — manual §3.1, §4 linha 4382. A assinatura do manual lista os slots na ordem: `state, trade, daily, priceBook, offerBook, histTrade, progress, tinyBook` mais 3 slots históricos hoje no-op (referência: `profit_dll.py` argtypes).
   - Esta story registra **ativos**: `state`, `histTrade`, `progress`.
   - Demais slots recebem **`NoopCallback`** (definida em `dll/callbacks.py` como `WINFUNCTYPE(c_void_p, ...)` com signature mínima compatível com cada callback type, fazendo `pass`/`return` sem tocar args).
   - **PROIBIDO** passar `None` em qualquer slot.
   - **Justificativa (Q11-E, Sentinel §12):** passar `None` em slots subsequentes corrompe internamente o registro e faz `SetHistoryTradeCallback` posterior (story 1.3) silenciosamente nunca disparar. Sintoma observado: 0 callbacks de histórico mesmo com requisição válida.
   - Logger emite `dll.initialize_call args={key:***, user, slots_active=['state','histTrade','progress'], slots_noop=8}`.

3. **AC3 — State callback APENAS enfileira**
   Wrapper registra `TStateCallback` com signature exata definida em `dll/types.py` (`WINFUNCTYPE(None, c_int, c_int)` — manual §3.2 linha 2738). O callback **EXCLUSIVAMENTE**:
   ```python
   def state_callback(conn_type, result):
       _state_queue.put_nowait((conn_type, result))
   ```
   Nada mais. Sem logs, sem prints, sem chamadas à DLL, sem acesso a self. Lei R3 / manual §4 linha 4382 ("não chamar DLL em callback").

4. **AC4 — `_cb_refs` global previne GC; finalize NÃO limpa**
   Lista global `_cb_refs: list[Any] = []` em `dll/callbacks.py` retém TODOS os callbacks (state ativo + NoopCallback × 8 + slots para 1.3) por toda a vida do processo.
   - Adicionar callback à lista IMEDIATAMENTE após criar o `WINFUNCTYPE`-wrapped object (regra ctypes).
   - **NÃO chamar `_cb_refs.clear()` em `finalize()`** — limpar a lista pode liberar callbacks ainda referenciados pela ConnectorThread interna da DLL e crashar o processo. Apenas anular `self._dll = None`.
   - Documentar este comportamento como comentário inline no `finalize()`.

5. **AC5 — `wait_market_connected(timeout=60)` aceita sequência completa**
   Método consume `_state_queue` em loop `while not timeout_expired` e aguarda a sequência canônica de pares `(conn_type, result)` per manual §3.2 linha 3317-3329:

   | conn_type | result | Significado |
   |-----------|--------|-------------|
   | 0 (`LOGIN`) | 0 (`CONNECTED`) | login OK |
   | 1 (`ROTEAMENTO`) | 2 (variável) | roteamento estabelecido |
   | 2 (`MARKET_DATA`) | 4 (`MARKET_CONNECTED`) | market data conectado |
   | 3 (`MARKET_LOGIN`) | 0 (`MARKET_LOGIN_OK`) | login market OK |

   **Decisão Q-AMB-01 (`MARKET_WAITING=2` vs `MARKET_CONNECTED=4`):**
   - **Aceitar AMBOS** result codes (2 ou 4) para `conn_type=2` como market data conectado.
   - Logar qual veio com structlog: `log.info("dll.market_state", conn_type=2, result=<2|4>, alias=<MARKET_WAITING|MARKET_CONNECTED>)`.
   - Documentar em `docs/dll/QUIRKS.md` como Q10-AMB.

   Retorna `True` ao receber MARKET_DATA conectado; `False` em timeout. Drenagem ocorre em **thread separada** (não no callback) — usa `_state_queue.get(timeout=remaining)`.

6. **AC6 — `finalize()` tenta `DLLFinalize` primeiro, fallback `Finalize`**
   Decisão Q-AMB-02:
   ```python
   def finalize(self) -> None:
       if self._dll is None:
           return
       try:
           ret = self._dll.DLLFinalize()
           method_used = "DLLFinalize"
       except AttributeError:
           ret = self._dll.Finalize()
           method_used = "Finalize"
       log.info("dll.finalized", code=ret, method=method_used)
       self._dll = None  # NÃO limpar _cb_refs (ver AC4)
   ```
   Resultado (qual método existiu) é registrado em `docs/dll/QUIRKS.md` Q09-AMB e atualizado quando smoke real validar.

7. **AC7 — `NL_*` errors viram `DLLInitError`**
   Retorno de `DLLInitializeMarketLogin < 0` levanta `DLLInitError(code: int, name: str, message: str)` com decode via mapa `dll/errors.py` (Nelo `*error-code`). Inclui `code` original, nome simbólico (`NL_INVALID_ARGS`, `NL_LICENSE_*`, etc.) e mensagem humana.

8. **AC8 — Logging structlog**
   Eventos (todos via `structlog.get_logger("data_downloader.dll")`):
   - `dll.companions_check` (paths verificados, status)
   - `dll.loading` (path)
   - `dll.native_log_silenced` (após `SetEnabledLogToDebug(0)`)
   - `dll.initialize_call` (args mascarados)
   - `dll.initialized` (return code)
   - `dll.market_state` (conn_type, result, alias)
   - `dll.connected` (timestamp)
   - `dll.finalized` (code, method)
   - `dll.error` (code, name, message)
   **NUNCA** logar dentro do callback (R3).

9. **AC9 — Path da DLL configurável**
   Path resolvido em ordem: (1) arg `dll_path`, (2) env `PROFITDLL_PATH`, (3) default `profitdll/DLLs/Win64/ProfitDLL.dll` relativo ao repo root. Path é resolvido para absoluto antes do `WinDLL()`.

10. **AC10 — Smoke test gated**
    `tests/smoke/test_dll_init.py` com `pytest.mark.skipif(not os.getenv("PROFITDLL_KEY"))` conecta de verdade e valida `wait_market_connected(timeout=60) is True`. Salva log estruturado em `docs/qa/SMOKE_EVIDENCE/1.2-{timestamp}.log` (formato definido por Quinn em `SMOKE_PROTOCOL.md`).

11. **AC11 — Silenciar log nativo da DLL ANTES do init**
    Chamar `self._dll.SetEnabledLogToDebug(0)` **ANTES** de `DLLInitializeMarketLogin` para evitar que a DLL escreva em stdout/stderr com formato próprio (poluiria structlog). Logger emite `dll.native_log_silenced`. Se `SetEnabledLogToDebug` falhar (não exposto em alguma versão), apenas warn — não bloquear init.

12. **AC12 — Validar companions DLLs/.dat ANTES do `WinDLL()`**
    `scripts/verify-dll-companions.py` é chamado (ou função importada `verify_dll_companions(dll_path) -> list[str]`) ANTES de `WinDLL(path)`. Verifica presença de:
    - Companions `.dll` esperados na mesma pasta (lista canônica em Nelo `*manual --section 4`)
    - `.dat` files exigidos pela DLL
    Se algum faltar, levanta `DLLInitError(code=-1, name='COMPANIONS_MISSING', message=<lista de paths faltantes>)` antes mesmo de tentar carregar a DLL principal. Logger emite `dll.companions_check status=ok|missing`.

13. **AC13 — `dll_version` exposta**
    Property `ProfitDLL.dll_version: str` retorna versão obtida via `GetDLLVersion()` se a função estiver exposta na DLL (consultar `profitdll/Exemplo Python/profit_dll.py` para confirmar signature; se ausente, hardcode placeholder `"unknown"` + emitir TODO no log + criar issue tracking). Property cacheada após primeira chamada. Usado por Sol em metadata Parquet (H19, H1).

14. **AC14 — Fixture pytest session-scoped**
    Testes smoke usam fixture **session-scoped**:
    ```python
    @pytest.fixture(scope="session")
    def initialized_dll():
        dll = ProfitDLL()
        dll.initialize_market_only(...)
        assert dll.wait_market_connected(60)
        yield dll
        dll.finalize()
    ```
    Justificativa M15: DLL **não é idempotente** em `init→finalize→init` na mesma sessão Python (segundo init pode crashar ou retornar estado corrompido). Re-init em sessão única é proibido em testes. Fixture session-scoped garante init exatamente UMA vez por `pytest` invocation.

15. **AC15 — Teste unit com `mock_calls == []` no callback**
    `tests/unit/test_callbacks.py::test_state_callback_does_not_call_dll`:
    - Cria `MagicMock` para a DLL e atribui a um `_dll` fake disponível ao callback (via closure).
    - Cria `_state_queue = Queue()` real.
    - Invoca o callback construído por `make_state_callback(_state_queue)` com `(0, 0)`.
    - Asserções:
      - `mock_dll.mock_calls == []` (callback NÃO chamou nenhum método da DLL — INV-1, finding C6).
      - `_state_queue.get_nowait() == (0, 0)` (callback enfileirou o evento).
      - Apenas `put_nowait` foi exercido (verificável também por substituir `_state_queue` por mock e assert `mock_calls == [call.put_nowait((0, 0))]`).

16. **AC16 — Política de queue declarada**
    `_state_queue = Queue(maxsize=1000)`. Política de overflow **`block`** (i.e., `put` blocking, NÃO `put_nowait`) **NÃO se aplica aqui**: o callback chama `put_nowait` (R3 — não pode bloquear ConnectorThread). State changes são **raras** (ordem de unidades por sessão), então `maxsize=1000` é >>> qualquer cenário realista — overflow é teoricamente impossível. Se `put_nowait` raise `Full`, log `dll.state_queue_full` (apenas via append no callback é proibido; usa-se atexit/diagnóstico) — esse cenário é tratado como bug, não fluxo normal. Documentar a decisão como comentário inline no `make_state_callback`.

### Constraints CRÍTICAS
- **R3 / INV-1:** callback APENAS `put_nowait`. Quinn audita via mock (AC15).
- **stdcall + WINFUNCTYPE:** manual §3.2 linha 2735.
- **`_cb_refs` global:** sem isso GC coleta callbacks → DLL crasha (Q07-V).
- **NoopCallback obrigatório:** todos os 11 slots do `DLLInitializeMarketLogin` preenchidos. `None` é PROIBIDO (Q11-E).
- **`finalize()` NÃO limpa `_cb_refs`:** ConnectorThread interna pode ainda referenciar callbacks pendentes; limpar = crash (AC4).
- **`SetEnabledLogToDebug(0)` ANTES do init:** evita poluir structlog (AC11).
- **Companions check antes do `WinDLL()`:** se faltar `.dll`/`.dat`, falha cedo com COMPANIONS_MISSING ao invés de erro críptico do Windows loader (AC12).

### Padrão proposto (esqueleto — Dex finaliza)

```python
# src/data_downloader/dll/wrapper.py
from ctypes import WinDLL, c_int, c_wchar_p, c_void_p, WINFUNCTYPE
from pathlib import Path
from queue import Queue, Empty
import os
import structlog

from .callbacks import register_state_callback, make_noop_callback, _cb_refs
from .errors import decode_nl_error, DLLInitError
from .types import (
    TStateCallback, TNoopTrade, TNoopDaily, TNoopPriceBook, TNoopOfferBook,
    TNoopHistTrade, TNoopProgress, TNoopTinyBook,
    LOGIN, ROTEAMENTO, MARKET_DATA, MARKET_LOGIN,
    MARKET_WAITING, MARKET_CONNECTED,
)
from ..scripts.verify_dll_companions import verify_dll_companions  # AC12

log = structlog.get_logger("data_downloader.dll")

DEFAULT_DLL_PATH = (
    Path(__file__).resolve().parents[3] / "profitdll" / "DLLs" / "Win64" / "ProfitDLL.dll"
)


class ProfitDLL:
    def __init__(self, dll_path: Path | None = None):
        env_path = os.getenv("PROFITDLL_PATH")
        self._dll_path = Path(env_path or dll_path or DEFAULT_DLL_PATH).resolve()
        self._dll: WinDLL | None = None
        self._state_queue: Queue = Queue(maxsize=1000)  # AC16
        self._dll_version: str | None = None

    def __enter__(self) -> "ProfitDLL":
        return self

    def __exit__(self, *exc) -> None:
        if self._dll is not None:
            self.finalize()

    def initialize_market_only(self, key: str, user: str, password: str) -> None:
        # AC12 — companions check
        missing = verify_dll_companions(self._dll_path)
        if missing:
            log.error("dll.companions_check", status="missing", paths=missing)
            raise DLLInitError(-1, "COMPANIONS_MISSING", f"missing: {missing}")
        log.info("dll.companions_check", status="ok")

        log.info("dll.loading", path=str(self._dll_path))
        self._dll = WinDLL(str(self._dll_path))
        self._configure_argtypes()  # reuse de profit_dll.py

        # AC11 — silenciar log nativo ANTES do init
        try:
            self._dll.SetEnabledLogToDebug(0)
            log.info("dll.native_log_silenced")
        except Exception as e:
            log.warning("dll.native_log_silence_failed", error=str(e))

        # AC2 — 11 callback slots (1 ativo + 8 noop + 2 placeholders deprecated)
        state_cb = register_state_callback(self._state_queue)  # ativo
        noop_trade = make_noop_callback(TNoopTrade)
        noop_daily = make_noop_callback(TNoopDaily)
        noop_price_book = make_noop_callback(TNoopPriceBook)
        noop_offer_book = make_noop_callback(TNoopOfferBook)
        noop_hist_trade = make_noop_callback(TNoopHistTrade)  # 1.3 vai substituir via SetHistoryTradeCallback
        noop_progress = make_noop_callback(TNoopProgress)     # 1.3 vai substituir
        noop_tiny_book = make_noop_callback(TNoopTinyBook)
        # 3 slots deprecated/no-op finais (consultar profit_dll.py para ordem exata)
        # Todos em _cb_refs (callbacks.py garante)

        log.info(
            "dll.initialize_call",
            key="***", user=user,
            slots_active=["state"],
            slots_noop=8,  # AC2
        )
        ret = self._dll.DLLInitializeMarketLogin(
            c_wchar_p(key), c_wchar_p(user), c_wchar_p(password),
            state_cb,
            noop_trade, noop_daily, noop_price_book, noop_offer_book,
            noop_hist_trade, noop_progress, noop_tiny_book,
            # + 3 deprecated noop slots se profit_dll.py confirmar
        )
        if ret < 0:
            err = decode_nl_error(ret)
            log.error("dll.error", code=ret, name=err.name)
            raise DLLInitError(ret, err.name, err.message)
        log.info("dll.initialized", code=ret)

    def wait_market_connected(self, timeout: int = 60) -> bool:
        # AC5 — drena _state_queue em thread separada (este método chama de fora do callback)
        import time
        deadline = time.monotonic() + timeout
        while True:
            remaining = deadline - time.monotonic()
            if remaining <= 0:
                log.warning("dll.connected_timeout", timeout=timeout)
                return False
            try:
                conn_type, result = self._state_queue.get(timeout=remaining)
            except Empty:
                return False
            alias = self._resolve_state_alias(conn_type, result)
            log.info("dll.market_state", conn_type=conn_type, result=result, alias=alias)
            if conn_type == MARKET_DATA and result in (MARKET_WAITING, MARKET_CONNECTED):
                log.info("dll.connected")
                return True

    @staticmethod
    def _resolve_state_alias(conn_type: int, result: int) -> str:
        names = {0: "LOGIN", 1: "ROTEAMENTO", 2: "MARKET_DATA", 3: "MARKET_LOGIN"}
        if conn_type == MARKET_DATA:
            if result == MARKET_WAITING:
                return "MARKET_WAITING"
            if result == MARKET_CONNECTED:
                return "MARKET_CONNECTED"
        return f"{names.get(conn_type, '?')}/{result}"

    def finalize(self) -> None:
        if self._dll is None:
            return
        # AC6 — Q-AMB-02
        try:
            ret = self._dll.DLLFinalize()
            method_used = "DLLFinalize"
        except AttributeError:
            ret = self._dll.Finalize()
            method_used = "Finalize"
        log.info("dll.finalized", code=ret, method=method_used)
        # AC4 — NÃO chamar _cb_refs.clear() (ConnectorThread pode ainda referenciar)
        self._dll = None
```
