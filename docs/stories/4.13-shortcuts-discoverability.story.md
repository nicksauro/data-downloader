# Story 4.13 — Atalhos descobríveis

## Contexto

Audit MainWindow + Screens revelou que os atalhos existem e funcionam
(verificado em main_window.py:274-291 e download_screen.py:296-305) mas
**não são descobríveis para usuário leigo**:

- Sidebar mostra "Ctrl+D" / "Ctrl+B" / "Ctrl+," abaixo do nome —
  ✓ bom.
- DownloadScreen footer tem `LBL_FOOTER_SHORTCUTS` — ✓ ok.
- **Catalog/Settings não tem footer equivalente.**
- Ctrl+R (refresh catalog), Ctrl+F (search), Delete (apagar), Ctrl+S
  (save settings), Ctrl+Q (sair com confirm) — não são listados em
  lugar nenhum acessível pelo usuário.

## User Story

**As a** usuário power que quer trabalhar mais rápido

**I want** ver lista de atalhos disponíveis em qualquer tela
(Ctrl+/ abre help, ou tooltip nos botões)

**So that** memorizo e uso o produto de forma fluida.

## Acceptance Criteria

1. **AC1 — Atalho global Ctrl+/ abre HelpDialog.**
   - Modal `QDialog` com tabela:
     | Atalho | Ação | Tela |
     | Ctrl+D | Ir para Download | Global |
     | Ctrl+B | Ir para Catálogo | Global |
     | Ctrl+, | Configurações | Global |
     | Ctrl+Q | Sair (confirm se download) | Global |
     | Ctrl+R | Refresh catálogo | Catálogo |
     | Ctrl+F | Buscar | Catálogo |
     | Delete | Apagar partição (confirm) | Catálogo |
     | Ctrl+S | Salvar | Settings |
     | Esc | Cancelar download / fechar modal | Context |

2. **AC2 — Tooltips em todos os botões nav + ações.**
   - `_nav_buttons` já tem texto `Ctrl+D` no label — adicionar
     `setToolTip("Atalho: Ctrl+D — Ir para tela de Download")`.
   - Botão "Salvar" → "Atalho: Ctrl+S".
   - Botão "Refresh" → "Atalho: Ctrl+R".

3. **AC3 — Footer LBL_FOOTER_SHORTCUTS replicado em
   Catalog + Settings.**
   - `LBL_FOOTER_SHORTCUTS_CATALOG`, `LBL_FOOTER_SHORTCUTS_SETTINGS`.

4. **AC4 — Smoke Pichau.**
   - Ctrl+/ em qualquer tela → modal help abre listando 9 atalhos.

## Out of Scope

- Re-mapping atalhos (configurável) — V2.
- Atalhos contextuais avançados (ex: Ctrl+Shift+R = reconcile) —
  V1.1 polish.
