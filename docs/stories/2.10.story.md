# Story 2.10 — Test strategy ADR-014 implementada

## User Story

**As a** Quinn (gatekeeper) responsável por suite de testes confiável + Aria (autora ADR-014) + Dex/Nelo (consumidores das fixtures)
**I want** **formalizar a estratégia de testes ADR-014** extraindo as fixtures hoje espalhadas (mock DLL no `conftest.py` Story 1.2, fake clock ad-hoc em alguns testes, Hypothesis usado pontualmente em Story 2.3/2.5) para uma **biblioteca compartilhada** `tests/_fixtures/` (mock_dll, fake_clock, layered fixtures por domínio) + **suite Hypothesis core** que cobre INV-1..INV-12 + finalizar `SMOKE_PROTOCOL.md` + protocolo formal de fixture session-scoped (M15 — DLL não-idempotente em re-init)
**So that** novas stories (Epic 3 UI, Epic 4 multi-asset) reusem fixtures consistentes, suite Hypothesis cresce de forma estruturada (não ad-hoc), gate de QA tenha cobertura formal de invariantes, e finding **F-S-4** (extrair mock DLL fixture compartilhada) do Sol audit Story 1.8 fica fechado.

**NOTA:** Esta story é **infra de testes**. Não muda código de produção. É o "trabalho invisível" que paga em todas as stories futuras. Sem 2.10, Epic 3/4 vão re-criar fixtures ad-hoc → drift → flaky tests.

## Acceptance Criteria

1. **AC1 — Pacote `tests/_fixtures/` formalizado:** Existe estrutura:
   - `tests/_fixtures/__init__.py` (exporta API pública)
   - `tests/_fixtures/mock_dll.py` (extraído de Story 1.2 conftest, agora reusável)
   - `tests/_fixtures/fake_clock.py` (NEW — substitui `time.time()`/`time.sleep()` para tests determinísticos)
   - `tests/_fixtures/sample_data.py` (workload sintético reproduzível, seed fixo, reusa Story 1.4.5 strategy)
   - `tests/_fixtures/layered_fixtures.py` (composição: dll → catalog → writer → orchestrator, cada layer com mock e real version)
   Doc README curto em cada arquivo (PT-BR — Quinn authority). Cada fixture tem typing + docstring com use case.

2. **AC2 — Mock DLL compartilhada:** `MockProfitDLL` (em `mock_dll.py`) com:
   - Implementa contrato `ProfitDLLProtocol` (referência a Story 1.2 wrapper).
   - Configurável via fixture params: número de trades, distribuição temporal, símbolo, error injection (probabilidade de NL_DISCONNECT/NL_INTERNAL_ERROR), latency simulation.
   - Reproduz quirks documentados: Q02-E (99% reconnect), Q11-99 (Q11 error rate).
   - Test fixture em si tem testes (meta-test) garantindo determinismo (seed fixo → mesmo output).
   - Migração: `tests/conftest.py` Story 1.2 importa de `_fixtures.mock_dll` (não duplicar).

3. **AC3 — Fake clock:** `FakeClock` (em `fake_clock.py`) com:
   - API: `clock.now() -> float`, `clock.sleep(s) -> None` (instantâneo, registra), `clock.advance(s) -> None`.
   - Substitui `time.time()` + `time.sleep()` via monkeypatch fixture.
   - Útil para retry/circuit_breaker tests (Story 2.6) sem esperar 30s real.
   - Integration: 2 fake clocks isolados em 2 fixtures (test parallelism safe).
   - Test meta: 100 iterações `advance(1.0)` → `now()` retorna start+100 exato.

4. **AC4 — Suite Hypothesis core (INV-1..INV-12):** Existe `tests/property/test_invariants_core.py` com pelo menos 1 property test por invariante listado em `docs/ARCHITECTURE.md` §Invariantes:
   - INV-1: callback NÃO chama DLL (mock_calls assertion).
   - INV-2: dedup idempotente (round-trip preserva).
   - INV-3: schema canônico Parquet preservado (sem drift por property).
   - INV-7: round-trip read(write(x)) == x.
   - INV-11: thread separation (OrchestratorThread != IngestorThread).
   - INV-12: fim de chunk = filas vazias + commit.
   - (INV-4..6 + 8..10 cobertura best-effort — pode deferred com tracking se complexo).
   Cada property com 100 examples mínimo. Hypothesis profile `ci` (max 5min total) e `dev` (mais rápido).

5. **AC5 — SMOKE_PROTOCOL.md finalizado:** `docs/qa/SMOKE_PROTOCOL.md` (já existe parcial em Story 1.7b) ganha §6 com:
   - Protocolo formal fixture session-scoped (M15 — DLL re-init failure).
   - Comando reprodutível para humano rodar smoke (env vars necessárias, output esperado).
   - Schema do `SMOKE_EVIDENCE/{story}-{ts}.md` (template com hash Parquet + log salvo + sanitização).
   - Lista de Stories que dependem de smoke real (1.7b, 1.8, futuras Epic 4).

6. **AC6 — Layered fixtures por domínio:** `layered_fixtures.py` expõe pyramid:
   - Layer 1 (atomic): `mock_dll`, `tmp_data_dir`, `fake_clock`.
   - Layer 2 (composto): `mock_catalog` (depends mock_dll + tmp_data_dir), `mock_writer` (depends tmp_data_dir).
   - Layer 3 (full stack): `mock_orchestrator` (depends layer 2).
   - Layer 4 (real): `real_dll_session` (gated por `PROFITDLL_KEY`, session-scoped — M15).
   Test demonstrativo: stories 1.4 (writer), 1.5 (catalog), 1.7a (orchestrator) podem usar fixtures sem re-criar boilerplate.

7. **AC7 — Cobertura formal CI gate:** Hypothesis profile `ci` é parte do `pytest` invocation default (não separado em command). CI badge ou métrica documenta cobertura de invariantes (X de 12 INV cobertos por property test). F-Q-1 (cobertura --cov bloqueada por duckdb) **não é resolvido nesta story** — apenas documentado o status atual em `docs/dev/COVERAGE_STATUS.md` (status: Hypothesis cobre invariantes; --cov line coverage continua deferred).

8. **AC8 — Documentação central:** `docs/qa/TEST_STRATEGY.md` (NEW) com:
   - Diagrama da pirâmide (unit → integration → property → smoke → e2e).
   - Tabela de fixtures disponíveis + use case.
   - Como criar nova property test (template referenciando ADR-014).
   - Política de mock vs real (quando usar cada, custo/benefício).
   - Referência a TEST_PYRAMID.md (existente) e SMOKE_PROTOCOL.md.
