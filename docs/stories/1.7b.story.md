# Story 1.7b — CLI typer + public_api mínima + smoke MVP gate

## User Story

**As a** usuário do data-downloader rodando via terminal Windows
**I want** um único comando `data-downloader download --symbol WDOJ26 --start 2026-03-01 --end 2026-03-30` que coordene tudo via orchestrator (Story 1.7a), use microcopy validada por Uma (Story 0.3), e respeite Ctrl+C com graceful shutdown
**So that** este comando funcionando = **gate de Epic 1** = squad confiante para construir Epic 2/3 em cima de fundação sólida.

---

## Acceptance Criteria

1. **AC1:** CLI typer `data-downloader download` em `src/data_downloader/cli.py`:
   - `--symbol` (required)
   - `--start YYYY-MM-DD` (required)
   - `--end YYYY-MM-DD` (required)
   - `--exchange F|B` (default F)
   - `--resume {job_id}` (opcional)
   - `--data-dir PATH` (default `./data`)
   - Inicialização DLL via env vars (PROFITDLL_KEY/USER/PASS)
2. **AC2:** Output Rich conforme `docs/ux/CLI_PATTERNS.md` (Story 0.3):
   - Header com symbol/range/exchange
   - Body: progress bar por chunk + global
   - Footer: summary Panel com chunks ok/failed/total trades
3. **AC3:** Microcopy 100% importada de `docs/ux/MICROCOPY_CATALOG.md` (Story 0.3) — Dex NÃO inventa textos. Suporte a `NO_COLOR=1`, fallback ASCII (finding M8).
4. **AC4:** AC novo (Uma — finding H15): Ctrl+C produz graceful shutdown:
   - SIGINT capturado
   - State machine transita `Running → DrainingDLL → DrainingWrite → Committed`
   - Mensagem clara: "Cancelando — aguarde drenagem ({N}s estimados)..."
   - Exit code 130 (POSIX SIGINT convention)
   - Re-rodar mesmo comando depois deve funcionar como resume
5. **AC5:** AC novo (Uma — finding H13): Microcopy 99% reconnect literal: "99% completo — reconectando à DLL (quirk Q-99-RECONNECT, manual §3.1). Aguarde até timeout de {N}s."
6. **AC6:** AC novo (Uma — finding H14): Erros NL_* resolvidos via MICROCOPY_CATALOG; código não mapeado mostra fallback "Erro desconhecido NL_{code}: consulte equipe Nelogica."
7. **AC7:** `src/data_downloader/public_api/download.py`:
   - Função `download(symbol, start, end, ...)` que constrói orchestrator e chama `.run()`
   - Retorna `DownloadHandle` (ADR-007a, finding H9/H10) com `.cancel()` e `.wait_result()`
   - Aria `*review-design` desta interface
8. **AC8:** `src/data_downloader/public_api/__init__.py`:
   - `__api_version__ = "0.1.0"`
   - Exports: `download`, `read_continuous` (de Story 1.5b), `DownloadHandle`, `DownloadResult`
9. **AC9:** Smoke test E2E `tests/smoke/test_mvp_gate.py` (gated por env `PROFITDLL_KEY`):
   - Roda `download WDOJ26 --start 2026-03-01 --end 2026-03-30`
   - Verifica: comando termina sem exception não-tratada
   - `data/history/F/WDOJ26/2026/03.parquet` existe e é Parquet válido
   - Catálogo SQLite tem ≥1 entrada em `partitions` para WDOJ26
   - DuckDB lê todos os trades (`SELECT COUNT(*)` > 0)
   - Re-rodar mesmo comando = no-op (cache hit, nenhuma chamada DLL)
   - Sol `*integrity-check --symbol WDOJ26` retorna clean (Story 2.1 já produz validador como código)
   - Hash do Parquet salvo em `docs/qa/SMOKE_EVIDENCE.md` (finding C5)
   - Log completo do smoke salvo em `docs/qa/smoke-{timestamp}.log`
10. **AC10:** Uma é reviewer obrigatório (finding H13). Sem GO de Uma, story não vai a InReview.

**REMOVIDO** (vai para Story 1.8): registro de baseline `bench_chunking` em BASELINES.md.
