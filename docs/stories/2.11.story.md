# Story 2.11 — Exception hierarchy ADR-011 implementada

## User Story

**As a** Aria (autora ADR-011) e Felix/Uma (consumidores em UI Epic 3) e Dex (implementador atual sem hierarquia formal)
**I want** **implementar a hierarquia de exceptions ADR-011** com 3 camadas claras: (a) **internals** podem usar exceptions livremente (`_ChunkRetryExhausted`, `_DLLStateError`, etc — prefixo `_` indica privado), (b) **public_api/** expõe APENAS hierarquia pública estável (`DataDownloaderError` raiz → `ConnectionError`, `IntegrityError`, `ContractError`, `ConfigError`, `CircuitOpenError` etc), (c) **UI** (Epic 3+) ramifica comportamento por tipo público (microcopy mapping `error_type → NL_*` via Uma)
**So that** caller `try: download(...) except ConnectionError:` funcione robusto, internals possam evoluir sem quebrar contrato público (R8 — public_api versionada SemVer), e Felix/Uma tenham mapa estável `exception type → microcopy NL_*` para UI Epic 3, fechando finding **H10** (`cancel()` não existe) e ground para finding **H11** (race shutdown propaga error mascarado).

**NOTA:** Esta story é **fronteira pura**. Não muda algoritmo, não muda performance, não muda dado. Muda **tipos**. É contrato. Por isso Aria audit é mandatory + property test "exception não vaza tipo interno em public_api" é mandatory.

## Acceptance Criteria

1. **AC1 — Hierarquia public_api documentada:** Existe `src/data_downloader/public_api/exceptions.py` com hierarquia formal:
   ```
   DataDownloaderError (base — abstract-ish, sempre via subclass)
   ├── ConfigError                  # validação de config (PydanticError wrapped)
   ├── ConnectionError              # NL_DISCONNECT, NL_LOST_AUTH, NL_NO_LOGIN
   ├── ContractError                # vigent_contract não encontrado, contrato inválido
   ├── DataIntegrityError           # checksum mismatch, dedup quebrado, schema drift
   ├── ChunkError (abstract)
   │   ├── ChunkTimeoutError        # 1800s sem progress (Q02-E hard timeout)
   │   ├── ChunkRetryExhaustedError # max_attempts atingido (Story 2.6)
   │   └── CircuitOpenError         # Story 2.6 circuit breaker
   ├── StorageError (abstract)
   │   ├── PartitionWriteError      # falha em fsync, replace, etc
   │   └── CatalogError             # SQLite WAL, lock, etc
   └── DLLError (abstract)
       ├── DLLNotInitializedError
       ├── DLLLicenseError          # NL_NO_LICENSE, NL_LICENSE_NOT_ALLOWED
       └── DLLInvalidArgError       # NL_INVALID_ARGS
   ```
   Todas com `__init__(self, *, code: str | None = None, context: dict | None = None)` para dados estruturados (consumível por UI sem parsing string).

2. **AC2 — Internals com prefixo `_`:** Internals em `src/data_downloader/{dll,storage,orchestrator}/` usam exceptions privadas (`_ChunkRetryExhausted`, `_DLLStateError`, etc). Prefixo `_` é **convenção forte** — auditoria mecânica (mini-script ou test) garante que NENHUMA exception sem prefixo `_` é raised diretamente em paths que cruzam fronteira `public_api/`. Adapter em `public_api/` traduz internals → pública.

3. **AC3 — Adapter pattern em public_api:** Cada função em `public_api/` (download, validate, etc) wraps internal calls em try/except + traduz:
   ```python
   try:
       return _internal_download(...)
   except _ChunkRetryExhausted as e:
       raise ChunkRetryExhaustedError(code=e.code, context=e.context) from e
   except _DLLStateError as e:
       raise DLLNotInitializedError(...) from e
   ```
   `from e` preserva chain (debug). Internals nunca aparecem no `__class__` da exception public.
   Test: invocar public api function em estado de erro → `assert isinstance(exc_info.value, PublicAPIError)` + `assert isinstance(exc_info.value.__cause__, _InternalError)`.

4. **AC4 — Microcopy mapping NL_* → error type → message:** `src/data_downloader/ui/error_microcopy.py` (NEW) mapeia:
   ```python
   ERROR_MICROCOPY: dict[type[DataDownloaderError], MicrocopyEntry] = {
       ConnectionError: NL_001,  # "Sem conexão com servidor Nelogica..."
       DLLLicenseError: NL_002,
       ...
   }
   def microcopy_for(exc: DataDownloaderError) -> MicrocopyEntry:
       return ERROR_MICROCOPY.get(type(exc), NL_DEFAULT)
   ```
   IDs `NL_*` reusam `docs/ux/MICROCOPY_CATALOG.md` (Story 0.3 — Uma authority). Nova entrada onde necessário (handoff via review).

5. **AC5 — `cancel()` finalmente implementado (finding H10):** `DownloadHandle` (ADR-007a) ganha método `cancel() -> None` que:
   - Aborta download em curso de forma cooperativa.
   - Sinaliza orchestrator → state machine transita para `Cancelled` (amendment ADR-005).
   - Rollback: parcial chunks são preservados em catalog como `incomplete` (não corrompido).
   - Raise `DownloadCancelledError` no caller que estava em `wait()`.
   - Test: thread B chama `handle.cancel()` enquanto thread A está em `handle.wait()` → A recebe `DownloadCancelledError` em < 5s.

6. **AC6 — Tests + property:**
   - **Unit:** Cada exception class tem test (init, attrs, str/repr).
   - **Unit:** Adapter pattern em public_api (mock internal raise → public raise correto).
   - **Property test (Hypothesis):** invariante "para qualquer raise em internals que cruza public_api, exception type pertence à hierarquia pública (NÃO `_*`)" — auditoria via `inspect` do call stack, 100 examples sintéticos.
   - **Integration:** `download(...)` com mock DLL que falha → exception correta (e.g., `ConnectionError`), com `__cause__` chain preservado.
   - **Integration cancel:** thread B cancela → thread A recebe `DownloadCancelledError` em SLA.

7. **AC7 — Aria audit fronteira (mandatory):** Sem APPROVED de Aria audit em `docs/qa/AUDIT_REPORTS/2.11-design-{date}.md`, story NÃO vai a InReview. Foco do audit:
   - Hierarquia stable (public types não mudam frequentemente).
   - Adapter pattern aplicado em todos os entry points public_api.
   - `from e` preservado (debug trail).
   - SemVer impact: classificar bump (MAJOR? MINOR?). Se MAJOR (existing types renamed/removed), exigir COUNCIL.

8. **AC8 — Documentação API + microcopy:** `docs/dev/EXCEPTIONS.md` (NEW) com:
   - Diagrama hierarquia (texto ASCII tree).
   - Tabela: exception → quando raised → microcopy NL_* → recovery suggestion.
   - Exemplos uso em caller (try/except patterns).
   - Política SemVer (mudança em hierarchy = MAJOR; nova subclass = MINOR).
   - Referência a ADR-011 + ADR-007a (public_api).
   `docs/ux/MICROCOPY_CATALOG.md` (Uma) atualizado com novas entradas NL_* para exceptions adicionadas.
