# Story 2.5 — Calendar B3 holidays.dat Integration

## User Story

**As a** sistema (DataValidator gap detection + IntegrityChecker `check_no_unexpected_gaps` + futuros validators de calendário)
**I want** ler o arquivo oficial `profitdll/DLLs/Win64/holidays.dat` distribuído pela Nelogica e usá-lo como fonte de calendário B3 em runtime
**So that** a tabela hardcoded de feriados 2025-2026 (`src/data_downloader/validation/calendar_b3.py`) seja substituída por fonte autoritativa e atualizável (cobre histórico ≥ 2020 + futuro até onde Nelogica documenta), eliminando finding **F-S-1** do Sol audit Story 2.1 e **COUNCIL-04 caveat**.

**NOTA:** Esta story executa o TODO documentado no docstring de `calendar_b3.py`: "integrar com `holidays.dat` distribuído pela Nelogica (PROFITDLL_KNOWLEDGE.md §X) — fonte autoritativa". Mantém fallback hardcoded para ambientes dev sem ProfitDLL instalado (CI, contributors externos).

## Acceptance Criteria

1. **AC1 — Investigação de formato:** Antes de implementar parser, Nelo (profitdll-specialist) confirma formato do `holidays.dat` consultando manual oficial (`Nelo *manual --section ?` se existir seção) ou via reverse engineering best-effort com hexdump + análise byte-a-byte. Resultado documentado em `docs/dll/HOLIDAYS_DAT_FORMAT.md` (ver AC7). Se formato é proprietário binário sem doc, registrar `validation_source: reverse_engineered` + lista de hipóteses testadas.

2. **AC2 — Parser dedicado:** Existe `src/data_downloader/validation/holidays_dat_parser.py` com:
   - Função `parse_holidays_dat(path: Path) -> frozenset[date]` — lê arquivo, retorna conjunto de feriados.
   - Tratamento de erros: arquivo ausente raise `HolidaysDatNotFoundError`; arquivo malformado raise `HolidaysDatParseError` com offset + bytes problemáticos.
   - Sem dependência nova (use só `struct` / `pathlib` / stdlib).

3. **AC3 — Integração transparente em `is_b3_business_day`:** `calendar_b3.py` agora consulta `holidays.dat` se disponível; fallback para tabela hardcoded se não. Lógica:
   ```
   if HOLIDAYS_DAT_PATH.exists():
       holidays = parse_holidays_dat(HOLIDAYS_DAT_PATH)  # cache módulo
   else:
       holidays = _B3_HOLIDAYS_2025_2026  # fallback
   ```
   API pública (`is_holiday`, `is_b3_business_day`, `b3_business_days_range`) **NÃO muda** — apenas fonte interna.

4. **AC4 — Cobertura mínima de feriados 2020-2030:** Após parsing de `holidays.dat` real (ou stub fixture com layout descoberto na AC1), conjunto retornado contém pelo menos os feriados nacionais brasileiros de 2020 até 2030 (≥ 13 feriados/ano × 11 anos = ≥ 143 entradas). Se cobertura efetiva é menor, log warning + complementar com hardcoded fallback dentro do range.

5. **AC5 — Refresh automático no boot + mtime check:** Ao primeiro uso, parser cacheia resultado + mtime do arquivo. Em invocações subsequentes, se mtime mudou, re-parse (Nelogica pode atualizar `holidays.dat` em update da DLL). Cache TTL adicional opcional (default 24h em background — Sol decide). Test verifica: alterar mtime artificialmente → próxima chamada re-parseia.

6. **AC6 — Test ground truth:** `tests/unit/test_holidays_dat_parser.py` valida contra dataset conhecido:
   - Feriados 2024 (passado, fonte oficial B3): 13 datas confirmadas.
   - Feriados 2025 (já hardcoded em `_B3_HOLIDAYS_2025_2026`): 13 datas confirmadas — devem bater 100%.
   - Feriados 2026 (já hardcoded): 13 datas confirmadas — devem bater 100%.
   - Caso edge: data não-feriado (ex: 2025-01-02) — `is_holiday(d)` retorna `False` em ambos os caminhos (parser + fallback).
   - Property test (Hypothesis): para qualquer date no range 2025-2026, `is_holiday(d)` via parser == `is_holiday(d)` via hardcoded.

7. **AC7 — Doc de formato:** `docs/dll/HOLIDAYS_DAT_FORMAT.md` documenta:
   - Localização do arquivo (`profitdll/DLLs/Win64/holidays.dat` ou outro path descoberto na AC1).
   - Formato byte-a-byte (header magic se existir, encoding de datas, alinhamento, endianness).
   - Validação contra fonte oficial Nelogica (`validation_source: nelogica_official` se manual confirmou; `reverse_engineered` se inferido).
   - Versão do `holidays.dat` analisada (sha256 + mtime).
   - Limitações conhecidas (range de anos coberto, ponto facultativo vs feriado oficial).
   - Nelo é reviewer obrigatório (DLL authority).

8. **AC8 — Graceful fallback:** Se `holidays.dat` não existe (ProfitDLL não instalado em CI ou contributor externo sem licença), API funciona normalmente com tabela hardcoded — log INFO `"holidays.dat não encontrado em {path}; usando tabela hardcoded 2025-2026"`. Test em CI confirma: sem `holidays.dat`, todos os tests existentes de `calendar_b3` continuam PASS sem mudança.
