## Acceptance Criteria

- [ ] **AC1 — ADR index update:** `docs/adr/README.md` + tabela §7 `docs/ARCHITECTURE.md` listam ADRs 018, 021, 023, 024, 025 (estão `accepted` em `docs/adr/` mas ausentes do índice canônico).
- [ ] **AC2 — ADR-004 superseded:** `docs/adr/ADR-004-partition-layout.md:3` marca `Status: superseded by ADR-025` (ADR-025 declara supersede mas ADR-004 não foi atualizado).
- [ ] **AC3 — MainWindow eventFilter wiring:** `MainWindow.__init__` chama `QApplication.instance().installEventFilter(self)` para ativar o Esc context-aware (override de `eventFilter` em `main_window.py:680-688` nunca recebe eventos hoje — grep `installEventFilter` retorna 0 hits).
- [ ] **AC4 — Package version dynamic:** `src/data_downloader/__init__.py:40` substitui literal `_PACKAGE_VERSION = "1.3.0"` por `importlib.metadata.version("data-downloader")` com fallback. Bug-class de v1.0.7 (drift `__init__` vs `pyproject.toml`) eliminado.
- [ ] **AC5 — RELEASES.md backfill:** `docs/release/RELEASES.md` recebe entries para v1.0.0, v1.0.1..v1.0.7, v1.1.0, v1.1.1, v1.2.0, v1.3.0 (R14 violado — vide `RELEASES.md:72` "(nenhuma release ainda...)").
- [ ] **AC6 — Pre-commit security hooks:** `.pre-commit-config.yaml` adiciona `bandit` (security lint) + `pip-audit` (CVE em deps). `gitleaks` adiado se já coberto por `detect-secrets v1.5.0`.
- [ ] **AC7 — Build timestamp determinístico:** `scripts/build_release.py` deriva `BUILD_TIMESTAMP` de `SOURCE_DATE_EPOCH` (`time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime(ctx.source_date_epoch))`) em vez de `datetime.utcnow()`. Bundle interno reproducível cross-build.
- [ ] **AC8 — Python pinning guard:** `scripts/build_release.py` falha cedo se `sys.version_info[:2] != (3, 12)` (R19 — build foi feito em 3.14 com `requires-python>=3.12`).
- [ ] **AC9 — StorageIndicator colorblind:** prefixos textuais `OK ·`, `! ·`, `!! ·` em `storage_indicator.py:64-67` antes do número de GB (cores verde/amarelo/vermelho mantidas, mas dual-encoded). Microcopy update + tooltip.
- [ ] **AC10 — Accessibility labels:** `setAccessibleName` em 6 botões principais (Download, Cancel, Save, Browse, Test Connection, Doctor) — habilita Narrator/NVDA em botões com ícone unicode (`⏹`, `✓`).
- [ ] **AC11 — Test cwd safety:** `tests/integration/test_ui_download_screen.py:150,160` troca `Path.cwd() / "data"` por `tmp_path`. Elimina dependência de cwd em test order.
- [ ] **AC12 — Monitor exception narrowing:** `orchestrator/orchestrator.py:1547-1553` troca `except Exception: continue` por `except Empty: continue` + log warning para outros tipos. Diagnóstico não fica cego.
- [ ] **AC13 — gc.freeze boot-only:** mover `gc.freeze()` de `Orchestrator.run()` para boot único (CLI/UI startup). Atual chama por job → heap permanente cresce monotonicamente em processo longo.
- [ ] **AC14 — Onboarding format_msg bug:** `src/data_downloader/ui/screens/onboarding_wizard.py:431-432` segundo `format_msg("WAR_ONBOARDING_SKIPPED", field="title")` corrige para `field="detail"` (atualmente título e corpo idênticos no diálogo "Pular").
- [ ] **AC15 — Dedup _VALID_EXCHANGES:** consolidar `_VALID_EXCHANGES` em `storage/partition.py` (single source-of-truth); `storage/schema.py:58` e `storage/_vectorized.py:45` importam dali. Drift entre 3 lugares eliminado.
