# Story 1.1 — Scaffolding do projeto

## User Story

**As a** desenvolvedor do squad data-downloader
**I want** um repositório Python 3.12 com lint, type-check, testes e estrutura de diretórios definida
**So that** todas as stories seguintes têm fundação consistente para implementar (sem retrabalho de setup).

---

## Acceptance Criteria

1. **AC1:** Existe `pyproject.toml` válido declarando: nome `data_downloader`, versão `0.1.0`, Python `>= 3.12`, dependências core (pyarrow, duckdb, structlog, rich, typer), test (pytest, pytest-cov, hypothesis), dev (ruff, mypy).
2. **AC2:** Estrutura `src/data_downloader/` criada conforme `docs/ARCHITECTURE.md` §5: `dll/`, `orchestrator/`, `storage/`, `public_api/`, `ui/`, `cli.py`, `__init__.py`.
3. **AC3:** Cada subpacote tem `__init__.py` mínimo (com docstring de propósito).
4. **AC4:** `tests/` contém subpastas `unit/`, `integration/`, `property/`, `smoke/`, `fixtures/`, cada uma com `__init__.py`.
5. **AC5:** `pytest` roda sem erro (mesmo que sem testes ainda) — saída "no tests collected" é OK.
6. **AC6:** `ruff check src/ tests/` retorna 0 findings.
7. **AC7:** `mypy src/` retorna 0 errors (config `strict = true`, `python_version = 3.12`).
8. **AC8:** `.gitignore` exclui: `data/`, `dist/`, `build/`, `__pycache__/`, `*.pyc`, `.pytest_cache/`, `.coverage`, `htmlcov/`, `.env`, `.env.*`, `.mypy_cache/`, `.ruff_cache/`.
9. **AC9:** `.env.example` lista variáveis esperadas (sem valores): `PROFITDLL_KEY`, `PROFITDLL_USER`, `PROFITDLL_PASS`, `DATA_DIR`, `LOG_LEVEL`.
10. **AC10:** README.md raiz tem seções mínimas: Visão, Pré-requisitos (Python 3.12 + Windows + DLL), Setup (venv + install -e), Como rodar testes, Link para docs/.
11. **AC11:** `pytest --collect-only` retorna 0 erros (garante imports válidos em todos os módulos do scaffolding — finding L1).
12. **AC12:** Wrapper de coleta de `dll_version` via `GetDLLVersion` em `src/data_downloader/dll/version.py` (placeholder se DLL não disponível em build); função usada no metadata Parquet conforme SCHEMA.md (Story 0.0). Coordenar com Sol antes de finalizar (finding H19).

### Pyproject.toml — seed proposto

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "data_downloader"
version = "0.1.0"
description = "Downloader de histórico de ativos via ProfitDLL — fundação para projetos futuros"
requires-python = ">=3.12"
authors = [{name = "Squad data-downloader"}]
dependencies = [
    "pyarrow>=15.0.0",
    "duckdb>=0.10.0",
    "structlog>=24.0.0",
    "rich>=13.0.0",
    "typer>=0.9.0",
]

[project.optional-dependencies]
test = [
    "pytest>=8.0",
    "pytest-cov>=4.1",
    "pytest-mock>=3.12",
    "hypothesis>=6.99",
]
dev = [
    "ruff>=0.3.0",
    "mypy>=1.9.0",
]
ui = [
    "PySide6>=6.6",
]
build = [
    "pyinstaller>=6.4",
]

[project.scripts]
data-downloader = "data_downloader.cli:app"

[tool.hatch.build.targets.wheel]
packages = ["src/data_downloader"]

[tool.ruff]
line-length = 100
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B", "A", "C4", "SIM", "RUF"]
ignore = []

[tool.mypy]
python_version = "3.12"
strict = true
warn_unused_configs = true
warn_redundant_casts = true
warn_unused_ignores = true

[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-v --strict-markers"

[tool.coverage.run]
source = ["src/data_downloader"]
omit = ["src/data_downloader/ui/*"]

[tool.coverage.report]
fail_under = 80
show_missing = true
```
