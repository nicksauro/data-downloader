# Story 0.4 — CodeRabbit adoption decision

## User Story

**As a** squad data-downloader que herdou referência a CodeRabbit em `agents/dev.md` (vinda do template AIOX) sem decisão explícita
**I want** que Gage + Quinn formalizem decisão GO/NO-GO/ADAPT sobre adoção do CodeRabbit, registrada em `docs/decisions/CODERABBIT_DECISION.md`
**So that** referências fantasma sejam removidas (ou a integração concretizada), eliminando ambiguidade de processo.

---

## Acceptance Criteria

1. **AC1:** Existe `docs/decisions/CODERABBIT_DECISION.md` com:
   - Contexto (porque está sendo decidido agora)
   - 3 opções avaliadas (ADOPT / ADAPT / DROP)
   - Critérios de decisão (custo, valor, integração com QA Loop, conflito com Quinn)
   - Decisão final + justificativa
2. **AC2:** Se ADOPT:
   - Documenta workflow (quando roda, gates, max iterations)
   - Atualiza `agents/dev.md` para descrever integração concreta
3. **AC3:** Se ADAPT:
   - Documenta diffs do workflow padrão AIOX
   - Atualiza `agents/dev.md`
4. **AC4:** Se DROP:
   - Remove referências em `agents/dev.md` (Gage faz PR)
   - Atualiza CLAUDE.md ou docs do squad para deixar explícito
5. **AC5:** Quinn valida que decisão não conflita com QA Gate (Quinn PASS é o gate canônico — CodeRabbit é assistência, não substituto).
6. **AC6:** Morgan registra a decisão no ROADMAP / CHANGELOG futuro.
7. **AC7:** Decisão é versionada: ADR-018 opcional se decisão tem implicações arquiteturais.
