# Story 4.17 — Windows installer (.exe Setup)

## Contexto

Hoje publicamos `data-downloader-vX.Y.Z-win64.zip` (~354 MB) via GitHub
Release. O usuário final precisa:

1. Baixar o zip.
2. Extrair manualmente (Explorer "Extract All...").
3. Navegar até `data_downloader.exe` dentro da pasta extraída.
4. Criar atalho no Desktop manualmente (clique-direito → Send to).
5. Não há entry em "Add/Remove Programs" → desinstalar = `rmdir /s` na
   mão.

Apps comerciais Windows entregam `Setup.exe` 1-clique com Start Menu +
Desktop shortcut + Add/Remove Programs registry + uninstaller. Nossa UX
de distribuição está abaixo do baseline esperado pelo usuário leigo.

**Pichau directive 2026-05-06:**
> "quero mais uma coisa, precisamos de um downloader nao arquiv ozipado,
> o quao dificil é desenvolver?"

## Decisão arquitetural (Aria, 2026-05-06)

**Tecnologia escolhida: InnoSetup 6.x** (Pascal-based, free, open source).

**3 razões:**

1. **Padrão de facto para apps Python/PyInstaller.** Comunidade
   PyInstaller documenta InnoSetup como caminho recomendado; templates
   `.iss` prontos abundantes; toolchain bem testada com bundles
   onedir (~350 MB) como o nosso.
2. **Code signing nativo (`SignTool=` directive)** — quando v1.2 trouxer
   cert EV, é flag de 1 linha sem refactor; e o instalador unsigned tem
   reputação histórica boa com Windows Defender (não trigger SmartScreen
   na mesma frequência que `--onefile` PyInstaller — ADR-016 Caminho B).
3. **Syntax `.iss` declarativa simples** — script de ~80-120 linhas
   resolve todos os 7 ACs; menor curva que NSIS (verbose) ou WiX (XML
   denso enterprise).

**Rejeitados:**

- **NSIS** — leve mas syntax verbose; sem ganho real para nosso caso
  (não precisamos de UI custom).
- **WiX Toolset** — overkill enterprise (.msi + Group Policy); curva XML
  alta para ROI baixo em single-app desktop.
- **MSIX** — moderno mas exige Win10+ e signing obrigatório; sandbox
  limita acesso a `~/.data-downloader/.env` e ProfitDLL paths; risco
  imaturo para apps Python tradicionais.

## User Story

**As a** usuário final do data-downloader (analista, trader)

**I want** um `Setup.exe` que instala o app em 1 clique e cria atalho
no Start Menu / Desktop

**So that** não preciso saber extrair zip, navegar pastas ou criar
atalhos manualmente — comportamento de app comercial Windows.

## Acceptance Criteria

1. **AC1 — Script InnoSetup versionado em repo.**
   - Arquivo `installer/data_downloader.iss` commitado (não gerado).
   - Tokens substituídos no build: `{{VERSION}}`, `{{BUILD_DATE}}`,
     `{{GIT_SHA}}`.
   - Sections obrigatórias: `[Setup]`, `[Files]`, `[Icons]`,
     `[Registry]`, `[UninstallDelete]`.

2. **AC2 — `scripts/build_release.py --with-installer` gera
   `Setup.exe`.**
   - Após empacotar zip determinístico, se flag presente, invoca
     `iscc.exe installer/data_downloader.iss` (caminho do Inno Setup
     Compiler configurável via `ISCC_PATH` env).
   - Output: `dist/data-downloader-Setup-vX.Y.Z.exe`.
   - Sem flag, build atual permanece intacto (backward compat).
   - SHA256 do Setup.exe entra em `build-manifest-v{version}.json`
     section `installer`.

3. **AC3 — Install path = `%LOCALAPPDATA%\Programs\data-downloader\`
   (default, sem admin prompt).**
   - `[Setup] PrivilegesRequired=lowest` + `DefaultDirName={localappdata}\Programs\data-downloader`.
   - Toggle opcional `/ALLUSERS` via `PrivilegesRequiredOverridesAllowed=dialog`
     para `C:\Program Files\data-downloader\` (com elevation).

4. **AC4 — Atalhos Start Menu + Desktop (toggle Desktop).**
   - `[Icons]` cria sempre `{group}\data-downloader` (Start Menu).
   - `[Tasks]` opcional `desktopicon` (default checked) → cria
     `{commondesktop}\data-downloader.lnk`.
   - Atalho aponta para `data_downloader.exe` (UI windowed, Story 4.8),
     não para `data_downloader-cli.exe`.

5. **AC5 — Add/Remove Programs registry entry.**
   - InnoSetup gera automaticamente:
     `HKCU\Software\Microsoft\Windows\CurrentVersion\Uninstall\data-downloader_is1`
     (ou `HKLM` se all-users).
   - Campos: `DisplayName=data-downloader`, `DisplayVersion=X.Y.Z`,
     `Publisher=data-downloader squad`, `URLInfoAbout=<repo URL>`,
     `UninstallString=<path>\unins000.exe`.

6. **AC6 — Uninstaller preserva user data por default.**
   - `unins000.exe` remove tudo em `<install_dir>\`.
   - **NÃO toca** `~/.data-downloader/` (env, profiles, configs) por
     default — diretório fora do escopo `[UninstallDelete]`.
   - Prompt opcional via `[Code]` Pascal: "Remover também configurações
     em `%USERPROFILE%\.data-downloader`? (default: NÃO)".

7. **AC7 — GitHub Release v1.1.0 inclui ambos artifacts.**
   - `data-downloader-v1.1.0-win64.zip` (advanced users / portable).
   - `data-downloader-Setup-v1.1.0.exe` (typical users).
   - Release notes destacam Setup.exe como recomendado.
   - `scripts/github_release.py` upload ambos via `gh release upload`.

## Out of Scope

- **Code signing (EV cert).** Defer v1.2 — aquisição cert EV
  ~3-5 dias úteis com vendor (DigiCert/Sectigo); custo ~$300-500/ano.
  Mitigação interim: documentar SmartScreen warning em INSTALL.md
  ("More info → Run anyway") — ADR-016 Caminho B.
- **Auto-update via Setup.** `tufup_stub` é V2 path (ADR-017 deferido).
  Setup.exe v1.1 é install-only; updates = baixar novo Setup manual.
- **MSI / WiX migration.** Reavaliar se enterprise customer demandar
  Group Policy deployment.
- **Localization installer (PT-BR, EN).** v1.2 — InnoSetup suporta
  `[Languages]` nativamente.

## Dependencies

- **1.7b-followup (Done)** — PyInstaller bundle determinístico
  `dist/data_downloader/` é input do `[Files]` section.
- **InnoSetup 6.x** instalado no build host (`iscc.exe` no PATH ou
  `ISCC_PATH` env). CI runner GitHub Actions: instalar via
  `chocolatey install innosetup` (1 linha extra em workflow).

## Risks

| Risk | Severidade | Mitigação |
|------|-----------|-----------|
| Instalador unsigned trigger SmartScreen warning | M | Documentar em INSTALL.md (ADR-016 Caminho B); v1.2 cert EV |
| `iscc.exe` não disponível em dev local | L | `--with-installer` opt-in; sem flag = build zip-only |
| Tamanho Setup.exe ~360-365 MB (~5-10 MB overhead) | L | Aceitável vs ZIP 354 MB; usuário typical não nota |
| Antivirus corporativo bloqueia .exe download | M | Manter ZIP em paralelo (AC7) como fallback |
| `%LOCALAPPDATA%` install permite múltiplas versões side-by-side | L | Aceitar (pro: dev/QA workflow); contra: usuário leigo confuso → docs orientam uninstall antes |
