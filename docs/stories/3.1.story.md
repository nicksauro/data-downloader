# Story 3.1 — Primeira tela real Epic 3 (MainWindow + DownloadScreen)

## User Story

**As a** quant brasileiro sem terminal (target Epic 3) + Felix (frontend-dev)
**I want** uma UI desktop nativa PySide6 com `MainWindow` (sidebar + status bar)
e `DownloadScreen` funcional consumindo `public_api.download()` via adapter em
`QThread`, com 5 estados (normal/loading/error/empty/success), microcopy 100%
catalog-sourced (R17), atalhos canônicos (Ctrl+D/B/,/Q + Esc context-aware),
e o quirk Q11-99 (banner amarelo com texto literal `WAR_99_RECONNECT`)
implementado fielmente
**So that** o golden path de download (1 clique no caso comum) finalmente
roda via UI real — promessa de produto inegociável (PRINCIPLES.md §1) — e
Epic 3 sai do prep COUNCIL-12 para in_progress.

---

## Acceptance Criteria

1. **AC1 — `app.py` real:** `python -m data_downloader.ui.app` instancia
   `QApplication`, configura HiDPI, aplica QSS de `assets/style.qss`, cria
   `MainWindow`, e entra em event loop. Logging via
   `data_downloader.observability.setup_logging` (Story 2.9).

2. **AC2 — `MainWindow` shell:** `QMainWindow` com sidebar `QFrame` esquerda
   (3 botões nav: Download/Catálogo/Settings), `QStackedWidget` central, e
   `QStatusBar` inferior (DLL status placeholder + versão app). Atalhos
   globais Ctrl+D / Ctrl+B / Ctrl+, registrados via `QShortcut` com
   `Qt.ApplicationShortcut`. Ctrl+Q dispara confirm modal se download ativo.

3. **AC3 — `DownloadScreen` 5 estados:** `QWidget` com `QStackedWidget` interno
   alternando entre normal/loading/error. Estados:
   - **normal:** form com `SymbolPicker` + `PeriodPicker` + folder picker +
     botão grande primário `BTN_DOWNLOAD_PRIMARY`.
   - **loading:** `ProgressCard` visível com barra, label `current_contract`
     (M16), subtitle ETA, log expansível, botão `BTN_CANCEL`.
   - **error:** card vermelho com microcopy `ERR_*` + botão `BTN_RETRY`.
   - **empty:** N/A (defaults inteligentes preenchem o form).
   - **success:** toast verde 5s + retorno ao normal.

4. **AC4 — `DownloadAdapter` em `QThread`:** `QObject` movido para `QThread`
   próprio. Sinais tipados (`Signal(object)` carregando `DownloadProgress`
   / `DownloadResult` / exceptions). Conexões cross-thread via
   `Qt.QueuedConnection` explícito. `start()` consome `handle.events()` em
   loop bloqueante, traduzindo cada evento para sinal. `cancel()` chama
   `handle.cancel()` (H10 fechado em Story 2.11). Story 2.9 propagation
   via `copy_context_to_thread`.

5. **AC5 — Microcopy 100% catalog-sourced (R17):** Toda string visível ao
   usuário vem de `microcopy_loader.MSG`. Texto canônico
   `WAR_99_RECONNECT` preservado byte-a-byte (literal Uma+Nelo, MICROCOPY §18).
   Test verifica que nenhum label mostra `<microcopy id not found>`.

6. **AC6 — `QFileDialog` com `DontUseNativeDialog`:** Folder picker usa
   `QFileDialog.Option.DontUseNativeDialog` (QT_PATTERNS §1, finding M9).

7. **AC7 — Atalhos por tela:** DownloadScreen registra Ctrl+D (start),
   Ctrl+C (cancel), Esc (cancel context-aware) via `QShortcut` com
   `Qt.WidgetWithChildrenShortcut`. Esc context-aware no MainWindow via
   `eventFilter` despachando para `screen.handle_escape()`.

8. **AC8 — Quirk 99% reconnect (Flow 4):** Quando `progress.is_99_reconnect`
   é `True`, `ProgressCard` muda barra para amarelo (state=reconnecting),
   mostra banner amarelo com texto LITERAL `WAR_99_RECONNECT`, e mantém
   posição da barra (não regride). NÃO interrompe download.

9. **AC9 — Tests headless pytest-qt:** 24 tests em
   `tests/integration/test_ui_main_window.py` + `test_ui_download_screen.py`
   PASS via `QT_QPA_PLATFORM=offscreen`.

10. **AC10 — Lint clean:** `ruff check src/data_downloader/ui/ tests/integration/test_ui_*.py`
    All checks passed. Mypy strict pulado (ADR-003 — `data_downloader.ui.*`
    em `ignore_errors`).
