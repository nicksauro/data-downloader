1. **AC1 — Smoke real executado.** Humano roda
   `pytest tests/smoke/test_mvp_gate.py -v` (com env vars setadas) OU executa
   manualmente o comando equivalente conforme `docs/qa/SMOKE_PROTOCOL.md`
   §4.2 (full smoke 30 dias WDOJ26).

2. **AC2 — Evidência produzida e sanitizada.** Test gera
   `docs/qa/SMOKE_EVIDENCE/1.7b-{YYYY-MM-DDTHHMMSSZ}.md` com:
   - Hash SHA256 dos Parquets gerados.
   - Output do log estruturado (linhas-chave).
   - Resultado de `data-downloader integrity check --symbol WDOJ26`.
   - Resultado do re-run (cache_hit observado).
   - Sanitização per `SMOKE_PROTOCOL.md` §11 (sem hostname, username em paths
     absolutos, conteúdo de .env).

3. **AC3 — Critérios PASS-1..6 do SMOKE_PROTOCOL.md §7.2 atendidos:**
   - [ ] PASS-1: >= 18 arquivos Parquet criados
   - [ ] PASS-2: total trades em [100.000 ; 50.000.000]
   - [ ] PASS-3: catálogo SQLite >= 18 entries com `schema_version` válido
   - [ ] PASS-4: re-run completo = no-op (todas partições `cache_hit=true`)
   - [ ] PASS-5: DuckDB lê 100% sem exception; range coerente com 2026-03-01..03-31
   - [ ] PASS-6: `integrity check` retorna 0 violations

4. **AC4 — Quinn `*qa-gate 1.7b-followup` emite PASS.** Quinn lê evidência,
   valida critérios PASS-1..6, e emite verdict `PASS` em
   `docs/qa/QA_REPORTS/1.7b-followup-{date}.md`.

5. **AC5 — Débito fechado.** Após PASS:
   - Atualizar `docs/qa/WAIVERS/1.7b-real-smoke-deferred-2026-05-04.md` →
     status `[X] Remediado` com link para o commit que rodou o smoke.
   - Atualizar `docs/stories/1.7b.story.md` Change Log com nota
     "AC9 fechada via 1.7b-followup em {date}".
