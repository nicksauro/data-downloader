## Acceptance Criteria

- [x] **AC1 вЂ” MГ©todo pГєblico `Catalog.recover_pending_commits() в†’ PendingRecoveryReport`** implementado em `src/data_downloader/storage/catalog.py`. - [x] **AC2 вЂ” PID liveness algorithm com defesa contra recycling.** Helper `_pid_alive(pid: int, started_at: datetime) в†’ bool`:
  1. Se `pid is None` в†’ False.
  2. Se `psutil` indisponГ­vel (ImportError) в†’ fallback: `True` apenas se `started_at > now - 1h` (timestamp-only).
  3. `psutil.pid_exists(pid)` в†’ se False в†’ False.
  4. `psutil.Process(pid).create_time()` (UTC epoch) в†’ comparar com `started_at`. Se `create_time > started_at` (PID reciclado, processo diferente) в†’ False.
  5. SenГЈo в†’ True.
  Helper unit-testГЎvel isoladamente (mockando psutil).

- [x] **AC3 вЂ” Recovery resolution table** para cada row de `_pending_commits`:
  | Estado | AГ§ГЈo |
  |--------|------|
  | PID vivo | skip (`pid_alive`) |
  | PID morto + arquivo ausente em `data_dir/final_path` | DELETE pending; `cleaned` (motivo `no_file`) |
  | PID morto + arquivo presente + sha256+size match | UPSERT em `partitions` (via helper interno `_register_recovered_partition`) + DELETE pending; `recovered` |
  | PID morto + arquivo presente + sha256 OR size mismatch | mover para `data/_quarantine/{utc_iso_compact}/{partition_path}`; DELETE pending; `quarantined` (motivo `sha_mismatch` ou `size_mismatch`) |

- [x] **AC4 вЂ” Quarantine convention.** Helper `_quarantine_partition(path: Path) в†’ Path` cria `data/_quarantine/{YYYYMMDDTHHMMSSZ}/{relative_partition_path}` preservando hierarquia interna. `os.replace` (atГґmico). Loga `catalog.recovery.quarantined` com path origem + destino. Se quarantine falhar (disco cheio, etc.), loga `catalog.recovery.quarantine_failed` e DELETE da pending row Г© abortada para retry no prГіximo boot.

- [x] **AC5 вЂ” IntegraГ§ГЈo em `__post_init__`.** Chamar `self.recover_pending_commits()` ANTES de `self.cleanup_orphans()` e `self.reconcile()` em `catalog.py:586-611`. - [x] **AC6 вЂ” Re-registro idempotente (`_register_recovered_partition`).** Helper interno lГЄ metadata do parquet on-disk (`pyarrow.parquet.read_metadata`) para extrair `row_count`, `first_ts_ns`, `last_ts_ns`, `schema_version`.
