# Story 4.5 — Test fixup (3 testes integration pré-existentes quebrados)

## Contexto

Durante a **sessão de sanity 2026-05-05** (post Story 1.7g schema integrity
+ Q17 broker DEAD-CODE + ProfitChart parity), executou-se a sanity suite
completa (`pytest tests/unit + tests/integration`). Foram detectados **3
testes integration quebrados**, todos validados como **PRÉ-EXISTENTES**
via `git stash + checkout 756a61d` (commit antes de qualquer mudança
desta sessão) — ou seja, o squad desta sessão **NÃO introduziu as
quebras**. Eles vinham herdados de sessões anteriores e ficaram
mascarados porque a sanity suite não estava sendo rodada ponta-a-ponta.

Esta story consolida o cleanup desses 3 testes em **1 entrega P2**
(non release-blocker — produção e schema integrity intactos), entregando
a sanity suite 100% verde para próximas iterações offline.

### Diagnóstico dos 3 testes (sessão 2026-05-05)

#### Teste 1 — `tests/integration/test_multi_asset_mock.py` (5/6 FAIL)

Testes afetados:
- `test_download_winh26_returns_trades`
- `test_download_petr4_with_bovespa_exchange_returns_trades`
- `test_download_wdoj26_regression_preserved`
- `test_download_winz26_quarterly_buffer`
- `test_download_multiple_equities_sequential`

**Causa raiz 1 — `subscribe_ticker` ausente no fake DLL.**
A classe `_MultiAssetFakeDLL` (definida no próprio arquivo de teste) não
implementa `subscribe_ticker(ticker, exchange) -> int` nem
`unsubscribe_ticker(ticker, exchange) -> int`. Esses métodos viraram
**pré-requisito** em **Story 1.7b-followup R7** (autoridade ProfitDLL —
sem subscribe a DLL aceita `GetHistoryTrades` mas não entrega trades em
tempo hábil). `download_chunk` chama `dll.subscribe_ticker`
defensivamente, captura `AttributeError`, loga warning, mas o teste
assume `result.status == "completed"` que falha porque sem subscribe não
chegam trades → status final = `"timeout"`.

**Causa raiz 2 — Progress callback signature drift (Q-DRIFT-05).**
`_emit_loop` no fake DLL chama `self._progress_cb(ticker, exchange, 0, p)`
(4 argumentos) mas a signature do progress callback foi consolidada em
**Q-DRIFT-05 fix (Story 1.7b-followup)** para `(TAssetID, c_int)` (2
argumentos). Resultado: `TypeError: this function takes 2 arguments
(4 given)` em thread emissora.

#### Teste 2 — `tests/integration/test_multi_symbol_mock.py::test_two_workers_two_symbols_both_complete`

**Sintoma:** TRAVA (não termina em <60s, nunca completa).

**Causa raiz:** depende do `orchestrator/broker/` package (broker process
do **ADR-015 REVOKED**). Pós **ADR-022 + Q17-CLOSED** (sessão 2026-05-05),
o broker é **DEAD-CODE annotated** (mantido apenas para evitar churn de
imports legados). O teste ainda exercita o broker diretamente, spawning
workers que travam aguardando IPC inexistente.

**Decisão pendente para mini-council:**
- (a) **Deletar** o teste (broker é DEAD-CODE pós-ADR-022 — ninguém
  exercita ou suporta).
- (b) **Refatorar** para single-process serial per ADR-022 (preserva
  intenção "2 símbolos completam" mas via single-process loop).

#### Teste 3 — `tests/integration/test_orchestrator.py::test_orchestrator_happy_path_single_chunk`

**Sintoma:** TRAVA (primeiro teste do arquivo, bloqueia o restante).

**Causa raiz desconhecida** — não exercita broker (não é o mesmo motivo
do teste 2). Comportamento **pré-existente** (também trava em commit
756a61d antes desta sessão). Requer investigação dedicada (debugger ou
`pytest -v --timeout=30 -s` para isolar onde o hang acontece).

## User Story

**As a** Quinn (@qa) responsável pela sanity suite verde da próxima
iteração offline + Dex (@dev) responsável por implementação

**I want** corrigir os 3 testes integration pré-existentes quebrados
(`test_multi_asset_mock.py` 5/6 FAIL, `test_multi_symbol_mock.py`
single test trava, `test_orchestrator.py` happy_path trava)

**So that** a sanity suite (`pytest tests/unit + tests/integration`)
fique 100% verde end-to-end, eliminando o ruído pré-existente que
mascarou regressões reais durante stories 1.7g/Q17/ProfitChart, e
estabelecendo baseline confiável para próximas stories Epic 4+.

## Acceptance Criteria

1. **AC1 — `test_multi_asset_mock.py` volta a 6/6 PASS.**
   - `_MultiAssetFakeDLL` recebe método `subscribe_ticker(ticker,
     exchange) -> int` retornando `0` (success default).
   - `_MultiAssetFakeDLL` recebe método `unsubscribe_ticker(ticker,
     exchange) -> int` retornando `0` (success default).
   - Signature do progress callback no `_emit_loop` ajustada de
     `self._progress_cb(ticker, exchange, 0, p)` (4 args) para
     `self._progress_cb(asset_id, c_int(p))` (2 args) per **Q-DRIFT-05**
     consolidação Story 1.7b-followup.
   - 6/6 testes voltam a PASS: `test_download_winh26_returns_trades`,
     `test_download_petr4_with_bovespa_exchange_returns_trades`,
     `test_download_wdoj26_regression_preserved`,
     `test_download_winz26_quarterly_buffer`,
     `test_download_multiple_equities_sequential` + o que já passava.
   - `pytest tests/integration/test_multi_asset_mock.py -v` retorna
     6 passed em <30s.

2. **AC2 — `test_two_workers_two_symbols_both_complete` resolvido
   (delete OU refactor).**
   - **Mini-council Aria + Pax** decide entre:
     - **(a) Delete** — remove o teste (justificativa: broker é
       DEAD-CODE pós-ADR-022 / Q17-CLOSED; teste exercita superfície
       morta sem cobertura útil).
     - **(b) Refactor** — reescrever para single-process serial per
       ADR-022 (preserva intenção "2 símbolos completam" mas sem
       broker IPC).
   - Decisão registrada em `docs/decisions/COUNCIL-{NN}-test-multi-symbol-broker-{date}.md`.
   - Se (a): `tests/integration/test_multi_symbol_mock.py` deletado OU o
     teste específico removido + comentário de pointer ao COUNCIL.
   - Se (b): teste refatorado, passa em <60s, exercita single-process
     download de 2 símbolos sequencialmente.
   - Sanity suite não trava mais nesse arquivo.

3. **AC3 — `test_orchestrator_happy_path_single_chunk` corrigido OU
   skipado com WAIVER.**
   - Investigação dedicada (`pytest -v --timeout=30 -s
     tests/integration/test_orchestrator.py::test_orchestrator_happy_path_single_chunk`)
     identifica root cause do hang.
   - **Caminho A (preferido)** — root cause corrigido, teste passa em
     <30s, restante do arquivo `test_orchestrator.py` também passa.
   - **Caminho B (fallback)** — se root cause é arquitetural / fora
     do escopo P2:
     - Teste marcado `@pytest.mark.skip(reason="...")` com pointer ao
       Q-DRIFT (criar novo Q-DRIFT-{NN} se necessário) e WAIVER em
       `docs/qa/WAIVERS/4.5-test-orchestrator-skip-{date}.md`.
     - WAIVER lista: root cause confirmado, próximo passo (story de
       follow-up OU Q-DRIFT entry para tracking), prazo de remediação.
   - Sanity suite não trava mais nesse arquivo.
