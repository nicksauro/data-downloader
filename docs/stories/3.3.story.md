# Story 3.3 — Theming refinado + Status bar metrics

## User Story

**As a** quant brasileiro (target Epic 3) + Felix (frontend-dev) + Pyro
(performance audit observable em runtime)
**I want** o tema QSS dark mode COMPLETO de THEME.md aplicado fielmente
(estados visuais para todos os widgets, gradiente em QProgressBar, header
dark + alternating rows + selection accent em QTableView, QGroupBox
transparente com label accent, focus ring accent.cyan em inputs, status
bar com text accent, scrollbars minimalistas, tabs styled), e a status
bar mostrando métricas runtime do `PrometheusExporter` (Story 2.4):
contador de downloads ativos, queue depths (DLL + write), trades
persisted total + indicador clickable de exporter port (URL copiada para
clipboard) — com graceful degradation quando exporter desabilitado
**So that** o app desktop tem a aparência polida prometida pela design
authority Uma e o usuário pode observar saúde do pipeline em tempo real
sem abrir CLI separada — fechando o gap visual + observability do MVP
Epic 3.

---

## Acceptance Criteria

1. **AC1 — QSS aplica paleta dark mode COMPLETA:** todos os 15 tokens
   canônicos de THEME.md §2 (`bg.primary` #0E0E10, `bg.surface` #17171A,
   `bg.elevated` #1F1F23, `bg.input` #26262B, `border.subtle` #2D2D33,
   `border.strong` #3D3D45, `text.*` 4 níveis, `accent.cyan` #3DD0E1,
   `primary` #4F8CFF, `success.green` #3FCB6F, `error.red` #F25656,
   `warning.yellow` #F2C94C, `info.blue` #5E9FFF) presentes no
   `style.qss`. Test `test_qss_contains_full_dark_palette` verifica.

2. **AC2 — Sem cores inventadas (R17 / Uma authority):** test
   `test_qss_no_unauthorized_colors` falha se cor hex fora da paleta
   canônica + 11 derivativos autorizados (hover/pressed/gradient end)
   aparecer no QSS.

3. **AC3 — QPushButton 4 estados:** normal / hover / pressed / disabled
   implementados para `QPushButton` base + cada variant (primary,
   destructive, link, success). Test verifica presença dos seletores.

4. **AC4 — QProgressBar gradiente + estados:** chunk usa
   `qlineargradient` em todos os 5 estados (normal cyan, reconnecting
   yellow, cancelling yellow, complete green, error red). Test
   `test_qss_progressbar_uses_yellow_during_reconnect` valida quirk 99%
   (Flow 4).

5. **AC5 — QTableView header dark + alternating + accent selection:**
   header `#17171A` + `alternate-background-color #131316` +
   `selection-background-color rgba(61,208,225,0.16)` (accent.cyan).

6. **AC6 — QGroupBox transparente com label accent:** background
   transparent + título `#3DD0E1` (accent.cyan) bold. Visual mais polido
   que o esqueleto inicial.

7. **AC7 — Inputs com border accent on focus:** QLineEdit / QComboBox /
   QDateEdit / QSpinBox border `#3DD0E1` 2px on focus (state.focus
   THEME §2.4).

8. **AC8 — QStatusBar dark + metrics styles:** padding, border-top
   subtle, `QStatusBar QLabel[role="metric"]` (default), `[role="metric-
   link"]` (accent + underline), `[role="metric-off"]` (text.muted +
   italic).

9. **AC9 — QScrollArea / QScrollBar / QTabBar / QMenu / QCheckBox
   styled:** todas as classes Qt usadas pelo MVP têm seletores explícitos
   no QSS (sem fallback para defaults Qt).

10. **AC10 — `MetricsPanel` widget compact:** ~360 LoC em
    `widgets/metrics_panel.py`. Display compact pronto para embed em
    status bar OU diálogo. Atualiza via slot `set_snapshot(MetricsSnapshot)`
    (signal target — desacoplado da fonte). Microcopy 100% catalog-sourced
    (R17). Estado inicial = "off".

11. **AC11 — `MetricsAdapter` em QThread separada:** QObject sem parent
    Qt (D2 COUNCIL-23), movido para QThread próprio em `start()`. QTimer
    de polling (default 1000ms, configurável) criado DENTRO da worker
    thread no slot `_on_thread_started`. Emite signals tipados
    (`metrics_updated(MetricsSnapshot)` + `exporter_unavailable()`) via
    QueuedConnection. `shutdown()` idempotente (D3 COUNCIL-23).

12. **AC12 — Consumer Protocol via introspecção (Aria audit):** adapter
    NÃO importa `PrometheusExporter` diretamente. Lê
    `exporter._gauges` / `exporter._counters` / `exporter.port` /
    `exporter.is_running` via duck-typing. Falhas absorvidas (campo
    fica `None`).

13. **AC13 — Status bar integrada:** `MainWindow._build_status_bar`
    embeda `MetricsPanel` entre DLL status e versão. `MetricsAdapter`
    instanciado e iniciado. `set_metrics_exporter(exporter|None)` método
    público para liga/desliga.

14. **AC14 — Graceful degradation sem exporter:** Sem
    `set_metrics_exporter` chamado OU com `None`, panel mostra
    `LBL_METRICS_OFF`, status bar continua exibindo DLL + versão. Pyro
    audit: zero overhead (adapter idle, sem polling).

15. **AC15 — Click no exporter URL copia para clipboard:** Quando
    exporter running, label do exporter URL é clickable (HTML
    `<a href="copy">`). Handler `_on_link_clicked` copia
    `http://localhost:{port}/metrics` para `QGuiApplication.clipboard()`
    + atualiza tooltip com microcopy `TST_METRICS_URL_COPIED`.

16. **AC16 — Microcopy 100% catalog-sourced (R17):** 8 IDs novos em
    `microcopy_loader.MSG` (§17b.6 — Metrics): `LBL_STATUSBAR_METRICS_PORT`,
    `LBL_METRICS_OFF`, `LBL_METRICS_ACTIVE_DOWNLOADS`,
    `LBL_METRICS_QUEUE_DEPTH`, `LBL_METRICS_TRADES_TOTAL`,
    `BTN_COPY_METRICS_URL`, `TST_METRICS_URL_COPIED`,
    `MOD_METRICS_DETAILS_TITLE`. Test verifica resolução (sem
    `<microcopy id not found>`).

17. **AC17 — `closeEvent` chama adapter shutdown (D3):** MainWindow
    override `closeEvent` para encerrar `_metrics_adapter.shutdown()`
    limpamente. Teste verifica que `adapter._thread` fica `None` após
    close.

18. **AC18 — Tests headless pytest-qt:** 36 tests novos (12 theming +
    14 metrics_panel + 10 status_bar) PASS via
    `QT_QPA_PLATFORM=offscreen`. Sem regressão nos 50 tests UI das
    Stories 3.1+3.2 (total 86/86 PASS).

19. **AC19 — Lint clean:** `ruff check src/data_downloader/ui/
    tests/integration/test_ui_theming.py
    tests/integration/test_ui_metrics_panel.py
    tests/integration/test_ui_status_bar.py` All checks passed. Mypy
    pulado (ADR-003 — `data_downloader.ui.*` em `ignore_errors`).
