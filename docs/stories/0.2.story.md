# Story 0.2 — Pre-commit Framework

## User Story

**As a** squad data-downloader que vai produzir muito código em paralelo (Dex em Epic 1/2, Felix em Epic 3)
**I want** um framework pre-commit instalado e configurado com hooks ruff/mypy/secrets-scan/large-files versionados em `.pre-commit-config.yaml`
**So that** R12, R18 e R19 sejam enforced ANTES do código sair da máquina do dev — sem depender de honor system.

---

## Acceptance Criteria

1. **AC1:** Existe `.pre-commit-config.yaml` versionado contendo no mínimo:
   - `pre-commit-hooks` (trailing-whitespace, end-of-file-fixer, check-yaml, check-json, check-toml, check-added-large-files com limit 1MB, check-merge-conflict)
   - `ruff` (lint + format)
   - `mypy` (strict mode em src/)
   - `detect-secrets` ou `gitleaks` (scan de secrets)
   - Custom hook que bloqueia commit de `.env` reais
2. **AC2:** Hook customizado `no-dll-in-commit`: bloqueia commit de `*.dll` exceto se em diretório explicitamente permitido pelo ADR-008.
3. **AC3:** Hook customizado `no-data-in-commit`: bloqueia commit de `*.parquet`, `*.duckdb`, `*.sqlite`, `*.csv` em qualquer lugar do repo (datasets nunca commitados).
4. **AC4:** Documentação em `docs/release/PRE_COMMIT.md` explicando:
   - Como instalar: `pip install pre-commit && pre-commit install`
   - Como rodar manual: `pre-commit run --all-files`
   - Como pular emergencialmente: `git commit --no-verify` (e por que NÃO fazer isso)
5. **AC5:** Hook pre-push (de Story 0.1) atualizado para rodar `pytest -q --no-header --tb=short` E `pre-commit run --all-files`.
6. **AC6:** Smoke: rodar `pre-commit run --all-files` em repo limpo após Story 1.1 (scaffolding) deve passar 100%.
7. **AC7:** CI cliente (futuro Gage Story 0.4 / Epic 2) consome a mesma config — single source of truth.
8. **AC8:** README raiz adiciona "Pre-commit instalado obrigatoriamente" em seção Setup.
9. **AC9:** Quinn valida que cada hook tem propósito ligado a uma lei de MANIFEST.md (R12, R18, R19, R20).
10. **AC10:** Versão do `.pre-commit-config.yaml` pinada (sem `rev: HEAD` — exige hashes específicos para reprodutibilidade R19).

- **Pinar versões:** todos os hooks têm `rev: <commit-sha>` ou `rev: v<X.Y.Z>` específico — nada de `HEAD`/`master`.
