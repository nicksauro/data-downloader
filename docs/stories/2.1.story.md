# Story 2.1 — Data integrity validators como código

## User Story

**As a** Quinn (QA) e Sol (storage) que dependem dos comandos `Sol *integrity-check` e `Quinn *data-validate` para gates honestos
**I want** esses comandos transformados em código real no subpacote `src/data_downloader/validation/` (não scripts ad-hoc)
**So that** circularidade do gate Epic 1 (smoke depende de integrity-check, mas integrity-check não existe — finding C4) seja eliminada definitivamente.

**NOTA:** Esta story foi MOVIDA do Epic 2 para Epic 1 conforme Plan Review 2026-05-03 finding C4. Sem ela, gate de close do Epic 1 não fecha.

---

## Acceptance Criteria

1. **AC1:** Existe `src/data_downloader/validation/__init__.py` expondo:
   - `IntegrityChecker` (Sol)
   - `DataValidator` (Quinn)
   - `RoundtripValidator` (vinda de Story 1.4 AC10 removido)
2. **AC2:** `IntegrityChecker(catalog, data_dir)` métodos:
   - `check_no_duplicates(symbol, exchange) -> list[Duplicate]`
   - `check_no_unexpected_gaps(symbol, exchange, *, exclude_weekends=True, exclude_holidays_brt=True) -> list[Gap]`
   - `check_schema_consistency(symbol, exchange) -> list[SchemaDrift]`
   - `check_catalog_filesystem_sync() -> ReconcileReport` (delega a Catalog.reconcile)
   - `check_all() -> IntegrityReport` (combina todos)
3. **AC3:** `DataValidator(reader)` métodos:
   - `validate_record(trade: TradeRecord) -> ValidationResult` (price>0, qty>0, exchange in {F,B}, etc.)
   - `validate_partition(parquet_path) -> PartitionValidationResult`
   - `validate_range(symbol, exchange, start_ts, end_ts) -> RangeValidationResult`
4. **AC4:** `RoundtripValidator` property tests Hypothesis: `read(write(L)) == sorted_dedup(L)` (INV-7) — extraído de Story 1.4 AC10.
5. **AC5:** CLI commands em `cli.py`:
   - `data-downloader integrity check --symbol WDOJ26 [--exchange F]`
   - `data-downloader validate --symbol WDOJ26 [--start ... --end ...]`
   - Output Rich conforme CLI_PATTERNS (Story 0.3)
6. **AC6:** Comandos retornam exit code 0 se clean, !=0 se issues encontrados.
7. **AC7:** Smoke gated env (`PROFITDLL_KEY`):
   - Após Story 1.7b smoke produzir dataset, rodar `integrity check --symbol WDOJ26` retorna clean
   - Rodar `validate --symbol WDOJ26 --start 2026-03-01 --end 2026-03-30` retorna clean
8. **AC8:** Quinn templates de output: `docs/qa/AUDIT_INTEGRITY_TEMPLATE.md`, `AUDIT_VALIDATION_TEMPLATE.md` — preenchidos automaticamente pelos comandos (finding M1).
9. **AC9:** WAIVED mecânica: se issue é aceitável, `--waiver-reason "..."` registra em `docs/qa/WAIVERS/{story}.md` (finding M2).
10. **AC10:** Sol + Quinn co-owners. Sem GO de ambos, story não vai a InReview.
