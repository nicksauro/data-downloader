# Story 1.4.5 — Synthetic perf baselines

## User Story

**As a** Pyro (perf-engineer) que precisa registrar baselines em `docs/perf/BASELINES.md` ANTES de Story 1.7b (smoke MVP) ser considerada gate honesto
**I want** rodar 3 benchmarks sintéticos (`bench_parquet_write`, `bench_dedup`, `bench_callback_to_disk`) usando mock DLL e fixtures de TradeRecord — sem precisar de código de produção real do orchestrator
**So that** o gate de Epic 1 não dependa de "palpites" de performance (V1 atual) e baseline real seja registrado quando smoke verde rodar (Story 1.8).

---

## Acceptance Criteria

1. **AC1:** Existe `benchmarks/bench_parquet_write.py` que:
   - Gera 1M trades sintéticos via fixture (Hypothesis ou randomização determinística com seed)
   - Mede tempo de `ParquetWriter.write(trades, partition)` (Story 1.4)
   - Reporta: trades/s, p50/p95/p99 latency, file size, compression ratio (Snappy vs ZSTD-1 — finding H5)
2. **AC2:** Existe `benchmarks/bench_dedup.py` que:
   - Gera N trades com 30% duplicatas (timestamp_ns/trade_id colisões)
   - Mede tempo de `dedup(L)` (Story 1.4 dedup function)
   - Reporta: trades/s dedup throughput, complexity (verificar O(n) ou O(n log n))
3. **AC3:** Existe `benchmarks/bench_callback_to_disk.py` que:
   - Mock DLL emite sequência de N callbacks (`HistoryTradeCallback`) via thread separada
   - Mede latência callback → put_nowait → drain queue → ParquetWriter.write
   - Reporta: p50/p95/p99 latency callback→disk
4. **AC4:** Cada benchmark é executável standalone (`python benchmarks/bench_parquet_write.py`) e via pytest-benchmark (`pytest benchmarks/ --benchmark-only`).
5. **AC5:** Existe `docs/perf/BASELINES.md` com seção "Synthetic baselines (Story 1.4.5)" — números registrados são MARCO INICIAL, sujeitos a revisão pós-Story 1.8 (real baselines).
6. **AC6:** Existe `docs/perf/REGRESSION_BUDGETS.md` com budgets propostos:
   - `bench_parquet_write`: regressão > 20% em trades/s = FAIL
   - `bench_dedup`: regressão > 30% = FAIL
   - `bench_callback_to_disk`: p99 latency > 100ms = FAIL
7. **AC7:** Mock DLL fixture em `tests/fixtures/mock_dll.py` é compartilhado com Story 1.2/1.3 testes (DRY).
8. **AC8:** Comparação Snappy vs ZSTD-1 documentada em `docs/perf/COMPRESSION_MATRIX.md` — Pyro recomenda baseado em (size, throughput, CPU).
9. **AC9:** Sol revisa que mock fixtures respeitam SCHEMA.md v1.0.0 (campos obrigatórios todos presentes).
10. **AC10:** CI hook (futuro) roda os 3 benchmarks e compara com `BASELINES.md` — regressão > budget = FAIL.
