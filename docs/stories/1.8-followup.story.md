# Story 1.8-followup — Real baselines (humano roda smoke + Pyro re-baseline)

## Contexto

Story 1.8 foi para Done em 2026-05-04 com verdict `CONCERNS deferred-real-baselines`
(WAIVER `docs/qa/WAIVERS/1.8-real-baselines-deferred-2026-05-04.md`).

Os baselines `v1.1.0-mock` registrados em `BASELINES.md` cobrem o pipeline
E2E mock (Orchestrator + ParquetWriter + Catalog reais com mock DLL inline)
mas não medem:

- Latência DLL real + reconnect 99% (Q11-99).
- Tempo total real download 1 mês WDOJ26 (rede + DLL).
- RSS / CPU steady state real.
- Comportamento real do quirk Q-RECON em download longo.

Esta story-followup rastreia a remediação do débito até Pyro registrar
baselines `v1.0.0-real` em `BASELINES.md` substituindo `v1.1.0-mock`.

## User Story

**As a** Pyro (perf-engineer) e Quinn (gatekeeper)
**I want** baselines reais derivados de smoke real (Story 1.7b-followup PASS)
para cada bench afetado (`bench_chunking`, `bench_callback_to_disk`,
`bench_multi_symbol`)
**So that** o gate de Epic 1 esteja **completamente** fechado e Stories de
Epic 2 (incluindo 2.2 perf-write-optimization) tenham referência **real**
para medir improvement (não só extrapolação mock).

## Acceptance Criteria

1. **AC1 — Story 1.7b-followup PASS pré-requisito.** Humano rodou smoke
   real per `SMOKE_PROTOCOL.md` §4.2 e Quinn emitiu PASS em
   `docs/qa/QA_REPORTS/1.7b-followup-{date}.md`. Esta story só pode iniciar
   após esse PASS.

2. **AC2 — Pyro re-roda `bench_chunking` em modo real.** Bench dispara
   download real WDOJ26 30 dias (2026-03-01 a 2026-03-31) via CLI canônica
   `data-downloader download --symbol WDOJ26 --start 2026-03-01 --end
   2026-03-31`. Métricas coletadas: tempo total, throughput trades/s, nº
   reconexões (eventos `download.99_reconnect_detected`), latência por chunk
   (p50/p95/p99). JSON salvo em `benchmarks/results/baselines/bench_chunking-1.0.0-real.json`.

3. **AC3 — Pyro re-roda `bench_callback_to_disk` em modo real.** Pipeline
   real exercitado durante o download de AC2 OU em sessão dedicada com
   contrato vigente. Captura flame graph via `py-spy` (preferido) ou
   `cProfile`. Top 5 hot paths identificados.

4. **AC4 — Pyro re-roda `bench_multi_symbol` em modo real (opcional).**
   Multi-process N=4 com 4 contratos vigentes distintos (WDOJ26 + WIN +
   PETR4 + VALE3 — ou 4 dias distintos do mesmo símbolo se rate-limit
   Nelogica restringir). Speedup vs N=1 medido. Crossover analysis (jobs
   curtos vs longos) opcional.

5. **AC5 — `BASELINES.md` atualizado:**
   - Seção `v1.0.0-real` adicionada (substitui `v1.1.0-mock` como canônico).
   - `v1.1.0-mock` movido para seção "Histórico mock baselines" (preservado
     para auditoria longitudinal, mas não mais canônico).
   - Comparação `v1.1.0-mock` vs `v1.0.0-real` documentada (delta % por bench
     + análise de causa raiz para deltas > 30%).

6. **AC6 — `TARGETS_V1.md` atualizado:**
   - Status `gap-tracked-by-2.2` → `measured-real` (com gap residual numérico
     atualizado se ainda > target).
   - Targets que já atingiam `measured-mock` → `measured-real` (confirmação).

7. **AC7 — `REGRESSION_BUDGETS.md` revisado:**
   - Override `bench_chunking = 30%` reavaliado contra stddev real (per
     promessa "Reduzir para 15% só após smoke real estabilizar").
   - Outros budgets confirmados ou ajustados.

8. **AC8 — Flame graphs entregues:**
   - `docs/perf/flame_chunking-real.svg`
   - `docs/perf/flame_callback_to_disk-real.svg`

9. **AC9 — Story 2.2 re-baseline.** Se Story 2.2 (perf-write-optimization)
   já estiver em desenvolvimento ou concluída, Pyro re-roda os benchs de
   Story 2.2 contra `v1.0.0-real` para confirmar que improvement esperado
   (>= +400% trades/s) se mantém no ambiente real. Se Story 2.2 ainda não
   começou, este re-baseline acontece como pré-requisito de merge da 2.2.

10. **AC10 — Quinn `*qa-gate 1.8-followup` emite PASS.** Quinn lê evidência,
    valida AC1-AC9, e emite verdict `PASS` em
    `docs/qa/QA_REPORTS/1.8-followup-{date}.md`. WAIVER 1.8 marcado como
    `Remediado`.
