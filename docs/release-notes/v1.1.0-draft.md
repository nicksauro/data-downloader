# v1.1.0 — Stable consolidated release

**Release date:** 2026-05-07
**Release type:** Single solid ship — supersedes v1.0.0 → v1.0.7.
**Owner:** @devops (Gage) publishes; @po (Pax) approves notes.

## Highlights

- 🎯 **Single solid release** — 8 hotfixes consolidados (v1.0.0 → v1.0.7 → v1.1.0) + sweep BIG COUNCIL pré-v1.1.0.
- 📦 **Bundle 55,6% menor** (886MB → **387.5MB**) sem perder funcionalidade — drop de Qt6WebEngineCore + qtwebengine_devtools_resources + **32 submodules** PySide6 não usados.
- ✅ **Primeiros testes subprocess do `.exe` real** — flag `--healthcheck` (Dex) + `tests/integration/test_binary_exe.py` (Quinn). v1.0.x dependiam apenas de import do source — bug de empacotamento passava silencioso até o usuário rodar.
- 🔧 **Q-DRIFT-37 mitigated** — `chunk_strategy=1d` para WINFUT previne saturação da queue do callback DLL (Nelo).
- 📅 **Chunks granulares 1d/dia** — todos os ativos baixam em chunks de 1 dia útil (ADR-023). Feedback per-day na UI + Q-DRIFT-37 FULLY mitigated.
- 🎨 **UI polish** — `CheatSheetDialog` (atalho **Ctrl+/**), onboarding banner em primeiro launch, deep-link "Abrir Settings" em toasts de erro.
- 🏛️ **Architecture** — ADR-018 (frozen-mode boundary) e ADR-021 (sys.frozen contract) ratificados; `bundle_paths.py` central elimina `_MEIPASS` espalhado.

## Install

Baixe **`data-downloader-Setup-v1.1.0.exe`** da release page e execute.
Caminho default de instalação: `%LOCALAPPDATA%\Programs\data-downloader\`.

Após instalação:

```powershell
# 1. Healthcheck — confirma instalação saudável (NEW v1.1.0)
data_downloader-cli.exe --healthcheck

# 2. Abrir UI desktop
data_downloader.exe

# 3. Smoke download (1 dia útil WDOFUT — chunk policy unificada 1d pós-ADR-023)
data_downloader-cli.exe download --symbol WDOFUT --start 2026-04-28 --end 2026-04-28
# Para janelas maiores (ex.: 5 dias úteis → 5 chunks 1d), basta estender --end:
# data_downloader-cli.exe download --symbol WDOFUT --start 2026-04-28 --end 2026-05-02
```

Configure credenciais ProfitDLL em `~\.data-downloader\.env` (preservado em
upgrades). Detalhes completos: [`docs/release/INSTALL.md`](../release/INSTALL.md).

## Mudanças relevantes (resumo)

### Adicionado

- `data_downloader-cli.exe --healthcheck` — self-test minimal (Dex).
- `bundle_paths.py` — resolução central de paths em frozen-mode (Aria, ADR-018/021).
- `CheatSheetDialog` (Ctrl+/) e onboarding banner (Uma).
- 4 suites de teste integration novas (Quinn): `test_binary_exe.py`, `test_frozen_assets.py`, `test_structlog_qt_bridge.py`, `test_cross_process_creds.py`.
- 2 scripts smoke real Pichau Windows local: `run_smoke_real.ps1`, `run_smoke_q-drift-37.ps1`.

### Mudado

- Bundle 886MB → **387.5MB** via PySide6 lean spec (Pyro).
- `CatalogScreen` recebe `@Slot(...)` em 5 métodos cross-thread (Felix-UI, fix do mesmo padrão de bug v1.0.7).
- `MetricsAdapter` usa `Qt.QueuedConnection` explícito (Felix-UI).
- `test_connection`, `integrity_check`, `reconcile` movidos para QThread workers — UI não freeza mais (Felix-UI).
- Version display em UI agora dinâmico via `importlib.metadata.version("data_downloader")` (Felix v1.0.8 RCA).
- Q-DRIFT-37 (HIGH/OPEN) → CLOSED-MITIGATED; Q-DRIFT-38 (NEW) → CLOSED-FILTERED (Nelo).
- Política de chunking unificada: 1 dia útil para todos os ativos (foi 5d para WDOFUT/INDFUT/equities, 1d para WINFUT). ADR-023.

### Corrigido

- `copy_context_to_thread` aceita `target=None` — elimina `TypeError` silenciado em catalog_adapter (Aria).
- `get_logger` retorna `BoundLogger` (cast) — mypy `--strict` 0 errors (Dex).
- Populate-seed em `_open_catalog*` agora loga warning em vez de silenciar (Dex).
- ruff: 7 → 0 errors em escopo Wave 1 (Dex).

Detalhes completos: [`CHANGELOG.md`](../../CHANGELOG.md) entry `[1.1.0]`.

## Compatibility

- **Windows 10** (build 19045+) ou **Windows 11** — x64.
- **ProfitDLL Nelogica** — single-session license; sem ProfitChart concorrente requerido (Q-DRIFT-02 refutado em 2026-05-05).
- **Schema v1.1.0** — Q-DRIFT-35 NEVER drops; upgrade automático seguro a partir de v1.0.x (storage backward-compatible).
- **Python 3.12** (para uso como library — bundle `.exe` é self-contained, não requer Python instalado).
